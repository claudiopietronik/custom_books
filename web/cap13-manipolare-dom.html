Capitolo 13

Manipolare il DOM

Nel capitolo scorso hai imparato a trovare un elemento e a cambiarne testo e stile. Ora facciamo un passo più ambizioso: costruire e smontare pezzi di pagina. Aggiungere una nuova riga a una lista quando l'utente scrive qualcosa, rimuovere un elemento quando clicca «elimina», accendere o spegnere una classe CSS per cambiare aspetto di colpo, leggere e impostare attributi (come l'indirizzo di un link o di un'immagine). Sono le operazioni con cui un'interfaccia diventa davvero dinamica: non più una pagina fissa che JS ritocca, ma una pagina che cresce e cambia mentre la usi. È la meccanica dietro ogni lista di cose da fare, ogni carrello, ogni feed che si allunga: creare elementi, riempirli, attaccarli, toglierli.

🗺 Dove siamo

Cambiare stili e classi con classList (§13.1), creare e rimuovere elementi (§13.2), leggere/impostare attributi (§13.3), scorrere una lista di elementi (§13.4). Nel laboratorio (§13.5) costruisci una lista che cresce dal vivo.

🔑 Perché ci tocca: le interfacce vere si costruiscono così

Cambiare il testo di un titolo è carino, ma le app vere generano contenuto: una chat che aggiunge messaggi, una tabella che si riempie coi dati, una galleria che carica altre foto. Tutto questo è creare nodi DOM e attaccarli. E gestire l'aspetto tramite classi (accendi/spegni una classe) invece di micro-stili sparsi è il modo pulito e professionale di farlo — CSS e JS restano al loro posto. Queste poche operazioni sono la cassetta degli attrezzi di ogni sviluppatore front-end.

Sezione 13.1Cambiare stili e classi

Nel Cap. 12 hai cambiato lo stile con el.style.color = …: va bene per un ritocco isolato, ma per cambiare più proprietà insieme il modo pulito è accendere una classe CSS già definita nel foglio di stile. Lo strumento è classList.

gestire le classi con classList
const box = document.querySelector("#box");

box.classList.add("attivo");       // aggiunge la classe "attivo"
box.classList.remove("attivo");    // la toglie
box.classList.toggle("attivo");    // se c'è la toglie, se manca la mette
box.classList.contains("attivo");  // true / false: c'è?
La squadra di classList add · remove · toggle · contains
classList.add("nome") / remove("nome")
aggiunge o toglie una classe CSS all'elemento. Tutto lo stile associato a .nome nel CSS si attiva o si spegne in blocco.
classList.toggle("nome")
l'interruttore: se la classe c'è la toglie, se manca la aggiunge. Perfetto per «apri/chiudi», «mostra/nascondi», «tema chiaro/scuro» con una sola riga. Il più usato dei quattro.
classList.contains("nome")
restituisce true/false a seconda che la classe sia presente. Utile per «decidere» in base allo stato attuale.
Perché è meglio che impostare tanti .style? Perché tiene lo stile nel CSS, dov'è il suo posto: JavaScript decide solo quando (accendi/spegni la classe), il CSS decide com'è fatto. Codice più pulito, e se un domani cambi l'aspetto tocchi solo il CSS. È la separazione dei ruoli che questo libro predica dal Cap. 1: struttura (HTML), aspetto (CSS), comportamento (JS).

Sezione 13.2Creare e rimuovere elementi

Il salto vero: non modificare elementi esistenti, ma fabbricarne di nuovi e inserirli nella pagina. Sono tre passi: crea l'elemento, riempilo, attaccalo a un genitore.

creare un elemento e aggiungerlo
// 1) CREA un nuovo <li> (esiste, ma è ancora "fuori" dalla pagina)
const voce = document.createElement("li");

// 2) RIEMPILO di contenuto
voce.textContent = "Comprare il latte";

// 3) ATTACCALO a un genitore già nella pagina (una <ul id="lista">)
const lista = document.querySelector("#lista");
lista.appendChild(voce);   // ora la voce COMPARE nella pagina

// RIMUOVERE un elemento:
voce.remove();   // sparisce dalla pagina
Fabbricare e togliere createElement · appendChild · remove
document.createElement("li")
crea un nuovo elemento del tipo indicato ("li", "div", "button"…). Esiste in memoria ma non è ancora visibile: è un pezzo staccato, che devi ancora attaccare.
riempirlo
gli dai contenuto e aspetto con ciò che già sai: .textContent, .classList.add(…), .style, attributi (§13.3).
genitore.appendChild(nuovo)
attacca il nuovo elemento come ultimo figlio del genitore. È questo il momento in cui l'elemento compare sullo schermo. Il genitore dev'essere un elemento già presente nella pagina.
elemento.remove()
stacca l'elemento dal DOM: sparisce dalla pagina. Semplice e diretto.
Il modello mentale: creare = fabbricare un pezzo; appendChild = montarlo sulla pagina; remove = smontarlo. Ripetendo questo ciclo costruisci qualsiasi interfaccia dinamica. Esiste anche una scorciatoia, lista.innerHTML += "<li>…</li>", comoda per casi semplici ma sconsigliata per contenuti che cambiano spesso (ricostruisce tutto e perde eventi/stato); per il lavoro serio, createElement + appendChild è la via robusta.

Sezione 13.3Attributi via JS

Gli attributi HTML (l'href di un link, il src di un'immagine, il disabled di un pulsante) si leggono e si cambiano da JavaScript come tutto il resto — così puoi, per esempio, cambiare dove punta un link o quale foto mostra un'immagine.

leggere e impostare attributi
const link = document.querySelector("a");

link.getAttribute("href");              // leggi: dove punta?
link.setAttribute("href", "https://example.com");  // cambia destinazione

// molti attributi hanno anche una proprietà diretta, più comoda:
const img = document.querySelector("img");
img.src = "gatto.jpg";              // = setAttribute("src", …)
img.alt = "Un gatto";
Attributi getAttribute · setAttribute · proprietà dirette
el.getAttribute("nome")
legge il valore di un attributo qualsiasi ("href", "src", "id", "data-…").
el.setAttribute("nome", "valore")
imposta (o cambia) un attributo. Funziona per ogni attributo, anche i tuoi data-* personalizzati.
proprietà dirette: el.src, el.href, el.id
per gli attributi più comuni, JavaScript offre una scorciatoia: img.src = "…" invece di setAttribute. Più corto e leggibile; nella pratica si usa quasi sempre questa forma quando esiste.
Un caso speciale utile: gli attributi booleani come disabled (di un pulsante) o checked (di una checkbox) si gestiscono con true/false: bottone.disabled = true lo disattiva. E ricordati la coppia del Cap. 12, input.value, per leggere cosa ha scritto l'utente in un campo: la userai proprio nel laboratorio.

Sezione 13.4Scorrere una lista di elementi

Spesso vuoi fare la stessa cosa a molti elementi: colorare tutte le voci, aggiungere una classe a tutte le card. Selezioni tutti con querySelectorAll e li scorri con un ciclo for … of.

un'azione su tutti gli elementi
// prendi TUTTE le voci della lista
const voci = document.querySelectorAll("#lista li");

// scorrile una a una e agisci su ciascuna
for (const voce of voci) {
    voce.classList.add("evidenziata");
}
🔁 for … of: scorrere una collezione

querySelectorAll restituisce una lista di elementi. Il ciclo for (const voce of voci) significa «per ogni voce nella lista, esegui questo blocco», dando ogni volta alla variabile voce l'elemento corrente. È il modo naturale di applicare un'operazione a un insieme intero — l'hai visto nell'idea generale nel Cap. 11, qui lo usi sugli elementi del DOM. (In alternativa vedrai voci.forEach(voce => { … }): stessa cosa, sintassi con una funzione arrow.)

Sezione 13.5Laboratorio: una lista dinamica

Mettiamo tutto insieme in una vera lista dinamica: scrivi una voce, premi «aggiungi» e compare nella lista; clicca «✕» accanto a una voce per rimuoverla. È l'ossatura di una to-do list — e usa ogni tecnica di questo capitolo.

Lista dinamica dal vivo crea e rimuove elementi veri
    La lista è vuota. Aggiungi qualcosa!
    🔑 Quello che devi portarti dietro dal Capitolo 13
    1. Classi con classList: add, remove, toggle (l'interruttore), contains. Meglio che tanti .style sparsi (§13.1).
    2. Creare: createElement → riempi → genitore.appendChild(…) (ora compare). Rimuovere: el.remove() (§13.2).
    3. Attributi: getAttribute/setAttribute, o le scorciatoie dirette img.src, link.href, bottone.disabled (§13.3).
    4. Scorrere molti elementi: querySelectorAll + for (const x of …) (§13.4).

    C'è però un pezzo che nel laboratorio abbiamo dato per scontato: come fa il pulsante «aggiungi» a sapere che è stato cliccato, e a far partire il codice? Finora i nostri esempi «partivano da soli». Ma le vere interfacce reagiscono all'utente: un click, un tasto premuto, un testo digitato. Il meccanismo che collega le azioni dell'utente al tuo codice si chiama evento — ed è il cuore del Capitolo 14. È lì che il DOM smette di aspettare i tuoi comandi e inizia ad ascoltare l'utente.

    Soluzione 13.1

    classList.toggle("nascosto"). È l'interruttore: a ogni click, se la classe c'è la toglie, se manca la aggiunge — esattamente il comportamento «alterna» che serve, in una sola riga, senza dover controllare prima lo stato.
