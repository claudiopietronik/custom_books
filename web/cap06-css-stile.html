Capitolo 6

CSS: dare stile

Finora le tue pagine erano corrette ma spoglie: font di sistema, testo nero su sfondo bianco, nessuna disposizione. Il CSS (Cascading Style Sheets) cambia tutto: è il linguaggio che decide come appare ogni elemento — colori, font, dimensioni, spazi, bordi, disposizione. La stessa identica pagina HTML, con un po' di CSS, si trasforma da bozza a sito vero: è la differenza tra un documento di testo e una pagina che invita a leggerla. In questo capitolo scopriamo cos'è il CSS, come collegarlo all'HTML, la sua semplice sintassi di regole e dichiarazioni, e i primi controlli — colori, font e dimensioni. È l'inizio di un mondo dove finalmente prendi il controllo dell'estetica.

🗺 Dove siamo

Cos'è il CSS e la separazione dei ruoli (§6.1), i tre modi di collegarlo all'HTML (§6.2), la sintassi — selettore, proprietà, valore (§6.3), e i primi controlli: colori, font, dimensioni (§6.4). Nel laboratorio (§6.5) trasformi una pagina spoglia in una con stile.

🔑 Perché ci tocca: dall'informazione al design

Il CSS è ciò che rende il web bello e usabile. Senza, ogni sito sembrerebbe un documento del 1995; con, nascono le interfacce curate che usi ogni giorno. Padroneggiarlo ti dà il potere di dare forma alle tue idee — e ti fa capire come sono fatti i siti che ammiri. È anche il naturale compagno dell'HTML semantico del Cap. 3: struttura pulita + CSS = pagine che si stilizzano con facilità.

Sezione 6.1Cos'è il CSS

Il CSS è il linguaggio dell'aspetto. Mentre l'HTML dice cosa c'è (un titolo, un paragrafo), il CSS dice come deve apparire (grande, blu, centrato).

📐 La separazione dei ruoli

Ricorda i tre linguaggi (Cap. 1): HTML = struttura, CSS = aspetto, JavaScript = comportamento. Questa separazione è un principio di design fondamentale: l'HTML descrive il significato, il CSS lo veste. I vantaggi sono enormi: puoi cambiare completamente l'aspetto di un sito senza toccare l'HTML (solo il CSS); puoi applicare lo stesso foglio di stile a mille pagine (coerenza, un posto solo da modificare); e mantieni il codice ordinato. È il motivo per cui, anche potendo mettere lo stile «dentro» l'HTML, si preferisce tenerlo separato in un file CSS a parte (§6.2). «Cascading» nel nome si riferisce al modo in cui le regole si combinano e «cascano» sugli elementi — lo capiremo con la specificità (Cap. 7).

Sezione 6.2Come collegare il CSS

Ci sono tre modi per applicare CSS a una pagina. Uno è nettamente il migliore per i progetti veri, ma è utile conoscerli tutti.

I tre modi esterno · interno · in linea
i tre approcci
<!-- 1. ESTERNO (il migliore): un file .css separato, collegato nell'head -->
<link rel="stylesheet" href="stile.css">

<!-- 2. INTERNO: un blocco <style> nell'head (ok per test rapidi) -->
<style>
    h1 { color: blue; }
</style>

<!-- 3. IN LINEA (da evitare): l'attributo style sull'elemento -->
<h1 style="color: blue">Titolo</h1>
1. Esterno — il modo giusto
scrivi il CSS in un file separato (es. stile.css) e lo colleghi nell'<head> con <link rel="stylesheet" href="...">. Separazione perfetta, riutilizzabile su più pagine, ordinato. È così che è fatto ogni sito serio (e questi libri!).
2. Interno — per prove rapide
un blocco <style>…</style> dentro l'<head>. Comodo per sperimentare su una singola pagina, ma non riutilizzabile.
3. In linea — da evitare
l'attributo style="..." direttamente sull'elemento. Sconsigliato: mescola struttura e stile, non si riusa, e diventa ingestibile. Usalo solo in casi eccezionali (o quando lo fa il JavaScript, Cap. 13).
La regola: nei progetti veri, sempre il file CSS esterno. In questo libro, per gli esempi brevi useremo spesso lo stile interno (più comodo da mostrare tutto insieme), ma sappi che il modo «professionale» è il file separato. Come per il codice C++/Python: separare le responsabilità in file diversi rende tutto più gestibile.

Sezione 6.3La sintassi: regole e dichiarazioni

Il CSS ha una sintassi semplice e regolare. Una regola dice «a questi elementi, applica questi stili». Impararla una volta vale per tutto il CSS.

Anatomia di una regola CSS selettore { proprietà: valore; }
stile.css
h1 {                    /* SELETTORE: a quali elementi (tutti gli h1) */
    color: navy;         /* DICHIARAZIONE: proprietà : valore ; */
    font-size: 32px;
    text-align: center;
}
il selettore — a chi
indica quali elementi stilizzare. Qui h1 = «tutti gli <h1> della pagina». I selettori sono un capitolo a sé (Cap. 7): si può selezionare per tag, classe, id e molto altro.
le graffe { } — cosa
racchiudono il blocco di dichiarazioni da applicare. Come in tanti linguaggi, delimitano.
la dichiarazione — proprietà : valore ;
ogni riga è proprietà: valore; — cosa cambiare (color) e come (navy). Attenzione ai due punti tra proprietà e valore, e al punto e virgola alla fine di ogni dichiarazione (dimenticarlo è l'errore più comune del CSS).
Puoi mettere quante dichiarazioni vuoi in una regola, e quante regole vuoi in un foglio. I commenti CSS si scrivono /* così */. La sintassi è tutta qui: selettore, graffe, coppie proprietà-valore. Il resto del CSS è imparare quali proprietà esistono e cosa fanno — e ce ne sono centinaia, ma poche decine coprono il 90% dei casi.

Sezione 6.4Colori, font, dimensioni

Le proprietà più immediate e gratificanti: quelle che cambiano colori, testo e dimensioni. Con queste poche, una pagina cambia già volto.

Le proprietà essenziali colore, font, spazio
colori: color e background-color
color = colore del testo; background-color = colore dello sfondo. I valori: nomi (red, navy), esadecimali (#7a5cff) o rgb(122, 92, 255). L'esadecimale è il più usato.
testo: font-size, font-family, font-weight
font-size (grandezza, es. 18px o 1.2rem), font-family (il carattere, es. Arial, sans-serif), font-weight (bold o un numero come 700), text-align (left/center/right).
spazio: padding e margin
padding = spazio interno (tra il bordo e il contenuto); margin = spazio esterno (tra l'elemento e i vicini). Sono al cuore del «box model» (Cap. 8), ma già ora danno «respiro» agli elementi.
bordi e angoli: border, border-radius
border: 2px solid gray (un contorno), border-radius: 10px (angoli arrotondati) — quest'ultimo, da solo, «ammorbidisce» qualsiasi elemento.

Sezione 6.5Laboratorio: una pagina con stile


  

    Prendiamo una pagina HTML spoglia e la trasformiamo con poche righe di CSS interno. Vedrai la differenza tra
    «documento» e «pagina».
  


  

    
Il file stile.html HTML + CSS interno

    

      
stile.html

      
<!DOCTYPE html>
<html lang="it">
<head>
    <meta charset="utf-8">
    <style>
        body {
            font-family: system-ui, sans-serif;
            background-color: #f4f2ff;
            color: #2a2a3a;
            max-width: 600px;
            margin: 2rem auto;     /* centra il contenuto */
            padding: 1rem;
        }
        h1 {
            color: #7a5cff;
            text-align: center;
        }
    </style>
</head>
<body>
    <h1>La mia pagina con stile</h1>
    <p>Che differenza fanno poche righe di CSS! 🎨</p>
</body>
</html>

    

    

      Apri e confronta con la stessa pagina senza il blocco <style>: la differenza è enorme.
      Poche regole — un font gradevole, sfondo tenue, contenuto centrato e con margini, titolo colorato — e la
      pagina sembra «vera». Nota il selettore body: gli stili applicati al body (font,
      colori) sono ereditati da tutti gli elementi dentro (i figli), a meno di essere sovrascritti — un
      comportamento comodo (imposti il font una volta per tutta la pagina). Sperimenta: cambia i colori, il
      max-width, aggiungi border-radius e background a un elemento. Il
      modo migliore di imparare il CSS è cambiare valori e guardare cosa succede.

Soluzione 6.1

      modificare per cambiare l'aspetto di tutto il sito).
