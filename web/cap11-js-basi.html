Capitolo 11

JavaScript: le basi

HTML dà la struttura, CSS l'aspetto. Ma finora le tue pagine sono statiche: mostrano, non fanno. JavaScript è il terzo linguaggio, quello che rende le pagine vive — reagire a un click, calcolare, aggiornare un testo senza ricaricare, chiedere dati a un server. È un vero linguaggio di programmazione, e gira dentro ogni browser del mondo. In questo capitolo impari le sue basi: le variabili (dove metti i dati), i tipi (numeri, testo, sì/no), le funzioni (blocchi di lavoro riutilizzabili) e le condizioni e i cicli (per decidere e ripetere). E scopri la console: la finestrella, nascosta in ogni browser, dove puoi provare codice al volo. Se hai fatto il libro di Python, ritroverai idee familiari con una sintassi leggermente diversa — qui è il «dialetto del web».

🗺 Dove siamo

Cosa aggiunge JS (§11.1), variabili e tipi (§11.2), funzioni (§11.3), condizioni e cicli (§11.4), la console del browser (§11.5). Nel laboratorio (§11.6) scrivi i tuoi primi script.

🔑 Perché ci tocca: senza JS il web è una brochure

Ogni cosa interattiva che hai mai usato sul web — un pulsante che apre un menu, un carrello che si aggiorna, una mappa che si trascina, una chat, questo stesso libro coi suoi widget — è JavaScript. È l'unico linguaggio che gira nativamente in ogni browser, e oggi anche fuori (sui server, con Node.js). Impararlo significa poter costruire, non solo descrivere. È il passaggio da «so fare una pagina» a «so fare un'applicazione».

Sezione 11.1Cosa aggiunge JavaScript

Prima di sintassi e dettagli, fissiamo cosa fa JavaScript e dove vive. Il modo di inserirlo in una pagina l'hai già intravisto: il tag <script>.

📖 I tre linguaggi, di nuovo — ma ora il terzo agisce

HTML = i contenuti (il «cosa»). CSS = l'aspetto (il «come appare»). JavaScript = il comportamento (il «cosa succede quando…»). JS può leggere e cambiare la pagina, reagire agli eventi dell'utente, fare calcoli, e comunicare con server — tutto mentre la pagina è aperta, senza ricaricarla.

Lo inserisci con <script>, di solito appena prima di </body> (così la pagina esiste già quando lo script parte), oppure in un file separato: <script src="app.js"></script> — esattamente come per il CSS esterno. In questo capitolo però useremo soprattutto la console (§11.5), dove puoi provare JS senza nemmeno una pagina.

Sezione 11.2Variabili e tipi

Una variabile è un contenitore con un nome, dove metti un dato per riusarlo. In JavaScript si dichiara con let (se il valore cambierà) o const (se resta fisso).

nella console del browser
let eta = 25;              // una variabile che può cambiare
const nome = "Anna";       // una costante: non cambierà
eta = 26;                // ok, eta era 'let'

console.log(nome, "ha", eta, "anni");   // → Anna ha 26 anni
Tre parole per le variabili let, const, (var)
const — costante
un valore che non cambierà più dopo l'assegnazione. È la scelta di default: usa const a meno che tu non sappia di dover cambiare il valore. Rende il codice più sicuro (nessuna modifica accidentale).
let — variabile
un valore che cambierà nel tempo (un contatore, un totale che cresce). Usa let solo quando serve la modifica.
var — la vecchia maniera
la parola storica, con regole più confuse. Nel codice moderno non si usa più: la citiamo solo perché la vedrai in codice vecchio. Tu usa let/const.
Nota il punto e virgola ; a fine riga: in JS chiude un'istruzione. Non è sempre obbligatorio, ma metterlo è un'abitudine chiara e sicura. E i nomi si scrivono in camelCase (numeroTotale, nomeUtente): prima lettera minuscola, ogni parola successiva con l'iniziale maiuscola — la convenzione di JavaScript.

Ogni valore ha un tipo. I tre fondamentali che userai subito:

I tipi di base number, string, boolean
number — numeri
25, 3.14, -7. In JS non c'è distinzione tra interi e decimali: è tutto number. Ci fai i conti: + - * /, e % (resto).
string — testo
tra virgolette: "Anna", 'ciao'. Le unisci con + ("ciao " + nome), o con i template: `ciao ${nome}` (apici inversi, e ${…} inserisce una variabile — comodissimo).
boolean — vero/falso
solo due valori: true o false. È la base delle decisioni (§11.4): nasce da confronti come eta > 18 (che vale true o false).
Esistono altri tipi (array [1,2,3], oggetti { }, null, undefined) che incontrerai presto — gli array e gli oggetti soprattutto quando lavorerai col DOM e con i dati. Per ora questi tre bastano per iniziare a ragionare.

Sezione 11.3Funzioni

Una funzione è un blocco di istruzioni con un nome, che puoi richiamare quando vuoi. Serve a non ripetere codice e a dare un nome a un'operazione. È il mattone di ogni programma.

definire e chiamare una funzione
// definizione: una funzione 'saluta' che riceve un 'nome'
function saluta(nome) {
    return `Ciao, ${nome}!`;
}

// chiamata: usiamo la funzione con un valore concreto
console.log( saluta("Anna") );   // → Ciao, Anna!
console.log( saluta("Luca") );   // → Ciao, Luca!
Anatomia di una funzione function nome(parametri) { return … }
function saluta(nome)
function annuncia una funzione; saluta è il nome che le dai; nome tra parentesi è il parametro: un dato che la funzione riceve dall'esterno, un «ingresso».
{ … } — il corpo
tra graffe c'è il lavoro che la funzione fa quando viene chiamata.
return — il risultato
return è il valore che la funzione «restituisce» a chi la chiama, il suo output. Qui restituisce una stringa. Non tutte le funzioni ritornano qualcosa: alcune fanno solo azioni (es. aggiornare la pagina).
saluta("Anna") — la chiamata
«esegui saluta con nome = "Anna"». Il nome tra le parentesi ("Anna") è l'argomento: il valore concreto che passi al parametro.
Vedrai spesso anche la forma arrow, più corta: const saluta = (nome) => `Ciao, ${nome}!`; — stessa cosa, sintassi compatta molto usata nel web (soprattutto per funzioncine passate ad altre funzioni, come vedrai con gli eventi nel Cap. 14). Le due forme convivono.

Sezione 11.4Condizioni e cicli

Due strumenti per dare «intelligenza» al codice: decidere (fare qualcosa solo se una condizione è vera) e ripetere (fare qualcosa più volte). Se conosci Python, è la stessa idea con parentesi graffe.

if / else e un ciclo for
// DECIDERE: if / else
const eta = 20;
if (eta >= 18) {
    console.log("Maggiorenne");
} else {
    console.log("Minorenne");
}

// RIPETERE: un ciclo che conta da 1 a 3
for (let i = 1; i <= 3; i++) {
    console.log("Giro numero", i);
}
Decidere e ripetere if / else, for
if (condizione) { … } else { … }
esegue il primo blocco se la condizione è true, altrimenti il blocco else. La condizione usa i confronti: >, <, >=, <=, === (uguale), !== (diverso).
=== — occhio: tre uguali!
In JS l'uguaglianza si scrive === (tre segni), non = (che è l'assegnazione) né == (una versione «permissiva» da evitare). Usa sempre === e !==.
for (let i = 1; i <= 3; i++)
il ciclo classico, in tre parti: inizio (let i = 1), condizione di continuazione (i <= 3: ripeti finché è vera), passo (i++: aumenta i di 1 a ogni giro). Qui: stampa per i = 1, 2, 3.
C'è anche while (condizione) { … } (ripeti finché la condizione è vera, senza contatore predefinito) e, per scorrere le liste, for (const x of lista) { … } — che userai spesso col DOM per ciclare sugli elementi. Ma if e for sono i due che incontri per primi ovunque.

Sezione 11.5La console del browser

Non serve un progetto per provare JavaScript: ogni browser ha una console integrata, un piccolo interprete dove scrivi una riga e vedi subito il risultato. È il tuo banco di prova e il tuo strumento di debug numero uno.

🛠 Aprire la console

Premi F12 (o tasto destro → «Ispeziona») e scegli la scheda Console. Lì puoi digitare qualsiasi JS e premere Invio: 2 + 2 → 4; "ciao".toUpperCase() → "CIAO". È il modo migliore per imparare: prova, sbaglia, correggi, subito.

Dal codice, invece, usi console.log(…) per «stampare» valori nella console: è così che sbirci cosa sta succedendo dentro un programma («qui eta quanto vale?»). Ogni programmatore usa console.log di continuo per capire cosa fa il proprio codice. Se qualcosa non va, la console mostra anche gli errori in rosso, con la riga colpevole — leggerli è metà del mestiere.

⚠ console.log non «mostra» nulla all'utente

Attenzione a non confondere: console.log scrive nella console per sviluppatori, che l'utente normale non apre mai. Serve a te per controllare. Per mostrare qualcosa all'utente devi cambiare la pagina — ed è esattamente ciò che imparerai dal prossimo capitolo in poi, con il DOM. Per ora, console.log è il tuo compagno di apprendimento.

  1. JavaScript è il linguaggio del comportamento: rende le pagine vive. Si inserisce con <script> (§11.1).

Quella porta tra il tuo codice e ciò che l'utente vede si chiama DOM — ed è il cuore del Capitolo 12.

Sezione 11.8Soluzioni

Soluzione 11.1

const per un valore che non cambia (es. const pi = 3.14159, o il nome di un utente appena letto). let per un valore che cambierà nel tempo (es. let punteggio = 0 che poi aumenta, o un contatore in un ciclo). Regola pratica: parti da const, passa a let solo se ti serve modificarlo.

Soluzione 11.2
JS
function doppio(n) {
    return n * 2;
}
console.log( doppio(7) );   // → 14

In forma arrow: const doppio = (n) => n * 2; — identica.

Soluzione 11.3

Stampa 0, 1, 2 (tre righe). Parte da i = 0, continua finché i < 3, e aumenta i di 1 ogni giro: quindi 0, 1, 2, e a i = 3 la condizione è falsa e si ferma. Attenzione al classico «parte da 0 e non include il 3»: sono comunque tre giri.

Soluzione 11.4
JS
if (voto >= 6) {
    console.log("Promosso");
} else {
    console.log("Bocciato");
}
