Capitolo 12

Il DOM: la pagina come oggetti

Nel capitolo scorso JavaScript parlava solo alla console. Ora facciamo il salto che rende tutto interessante: collegare JS alla pagina. Quando il browser carica il tuo HTML, non lo tiene come testo: lo trasforma in un albero di oggetti — uno per ogni tag — che JavaScript può leggere e modificare. Questo albero si chiama DOM (Document Object Model). È il ponte tra il tuo codice e ciò che l'utente vede: attraverso il DOM puoi trovare un elemento (un titolo, un pulsante, un paragrafo), leggere cosa contiene e cambiarlo al volo — senza ricaricare la pagina. Capire il DOM è capire come ogni sito interattivo, dal più semplice al più complesso, aggiorna ciò che hai davanti agli occhi.

🗺 Dove siamo

Cos'è il DOM (§12.1), l'albero degli elementi (§12.2), selezionare con querySelector (§12.3), leggere e cambiare il contenuto (§12.4). Nel laboratorio (§12.5) cambi davvero una pagina dal vivo.

🔑 Perché ci tocca: è qui che JS «tocca» la pagina

Tutto ciò che un sito fa di dinamico — mostrare un messaggio, aggiornare un contatore, nascondere un menu, inserire i risultati di una ricerca — passa dal DOM. Senza DOM, JavaScript sarebbe una calcolatrice chiusa in una stanza: potente ma isolata. Con il DOM diventa il regista della pagina. È il concetto cardine di tutta la programmazione web lato browser: i prossimi tre capitoli (manipolare, eventi, fetch) sono tutti costruiti su di esso.

Sezione 12.1Cos'è il DOM

Quando scrivi HTML, scrivi testo. Ma il browser, per disegnare la pagina, quel testo lo interpreta e ne costruisce una rappresentazione «viva» in memoria: un insieme di oggetti, uno per ogni elemento. Quella rappresentazione è il DOM.

📖 DOM = Document Object Model

Scomponiamo il nome: Document (il documento, cioè la pagina), Object (a oggetti: ogni tag diventa un oggetto con proprietà e metodi), Model (un modello, cioè una rappresentazione strutturata). Il DOM è la tua pagina HTML, ma vista come una collezione di oggetti che JavaScript può manipolare.

La cosa magica: il DOM è collegato in tempo reale a ciò che vedi. Se JavaScript cambia un oggetto del DOM (per esempio il testo di un titolo), il browser ridisegna subito la pagina di conseguenza. Non modifichi il file HTML sul disco: modifichi la copia viva in memoria, e lo schermo si aggiorna all'istante. Questo è il meccanismo dietro ogni pagina che «cambia da sola».

Il punto di ingresso è una parola speciale che JavaScript ti mette a disposizione: document. Rappresenta l'intera pagina, e da lì parti per raggiungere ogni elemento.

Sezione 12.2L'albero degli elementi

Gli elementi HTML sono annidati uno dentro l'altro (il <body> contiene un <div> che contiene un <h1>…). Questa nidificazione forma un albero, con relazioni di genitore, figlio e fratello — esattamente la struttura che il DOM rispecchia.

Dall'HTML all'albero ogni tag è un nodo
questo HTML…
<body>
    <div class="scheda">
        <h1>Titolo</h1>
        <p>Un paragrafo</p>
    </div>
</body>
…diventa questo albero
document
└─ body
   └─ div.scheda
      ├─ h1   "Titolo"
      └─ p    "Un paragrafo"
nodo (node)
ogni elemento dell'albero. div, h1, p sono nodi. Anche il testo dentro un tag è, tecnicamente, un nodo.
genitore / figlio (parent / child)
il div è il genitore di h1 e p; loro sono i suoi figli. body è il genitore del div. Le stesse relazioni di annidamento dell'HTML, ora navigabili da JavaScript.
fratelli (siblings)
h1 e p, avendo lo stesso genitore, sono fratelli.
Riconosci questa struttura? È la stessa dell'albero delle cartelle di un computer, o della struttura di una rete (Cap. 6 di «Reti»). L'annidamento «a scatole dentro scatole» è ovunque in informatica — e nel DOM è ciò che ti permette di dire «prendi il paragrafo dentro quella scheda».

Sezione 12.3Selezionare elementi

Per cambiare un elemento, prima devi trovarlo nel DOM. Lo strumento moderno è document.querySelector(…), che accetta gli stessi selettori CSS che hai imparato nel Cap. 7 — quindi già sai usarlo.

selezionare elementi
// per id (il #): restituisce quell'unico elemento
const titolo = document.querySelector("#titolo");

// per classe (il .): il PRIMO elemento con quella classe
const scheda = document.querySelector(".scheda");

// per tag: il primo <button> della pagina
const bottone = document.querySelector("button");

// TUTTI gli elementi che combaciano (una lista):
const tutti = document.querySelectorAll(".scheda");
Trovare elementi querySelector / querySelectorAll
document.querySelector("selettore")
cerca nella pagina e restituisce il primo elemento che combacia con quel selettore CSS (#id, .classe, tag, e tutte le combinazioni del Cap. 7). Il selettore è identico a quello che scriveresti nel CSS — un solo linguaggio di «puntamento», riusato.
querySelectorAll("selettore")
restituisce tutti gli elementi che combaciano, in una lista che puoi scorrere con un ciclo for … of. Utile quando vuoi cambiare molti elementi insieme.
se non trova nulla?
querySelector restituisce null (il «niente»). Provare a usare un elemento null è la causa dell'errore più comune per chi inizia («Cannot read properties of null»): quasi sempre significa che il selettore è sbagliato, o che lo script è partito prima che la pagina esistesse (per questo si mette lo <script> in fondo).
Esiste anche il più vecchio document.getElementById("titolo") (senza il #): fa la stessa cosa del querySelector("#titolo") ed è velocissimo. Lo vedrai spesso in giro e in questi stessi widget. Ma querySelector è più flessibile (accetta qualsiasi selettore), quindi è il nostro default.

Sezione 12.4Leggere e cambiare il contenuto

Una volta che hai un elemento in mano (in una variabile), puoi leggerne le proprietà e — la parte magica — modificarle. La più usata è textContent: il testo dentro l'elemento.

leggere e scrivere il contenuto
const titolo = document.querySelector("#titolo");

// LEGGERE: cosa c'è scritto adesso?
console.log(titolo.textContent);   // → "Titolo"

// SCRIVERE: cambiamo il testo → la pagina si aggiorna subito!
titolo.textContent = "Nuovo titolo!";

// cambiare anche lo stile, via la proprietà .style
titolo.style.color = "crimson";
Le proprietà che userai di più textContent, innerHTML, style
.textContent
il testo dentro l'elemento. Lo leggi (a destra dell'=) o lo scrivi (a sinistra): el.textContent = "ciao" sostituisce il contenuto testuale. È il modo più semplice e sicuro per cambiare cosa mostra un elemento.
.innerHTML
come textContent, ma interpreta l'HTML dentro la stringa: el.innerHTML = "<b>ciao</b>" mette la parola in grassetto. Potente, ma attenzione: non metterci mai testo scritto da un utente sconosciuto (rischio di sicurezza — lo vedrai). Per solo testo, preferisci textContent.
.style.qualcosa
cambia una proprietà CSS al volo: el.style.color = "red", el.style.display = "none" (per nascondere). Le proprietà CSS con il trattino diventano camelCase: background-color → el.style.backgroundColor.
.value (per gli input)
per i campi <input>, .value è ciò che l'utente ha digitato. Fondamentale per leggere i form da JavaScript — lo userai molto.
Il concetto chiave da fissare: leggi con = elemento.proprietà (a destra), scrivi con elemento.proprietà = (a sinistra). La stessa proprietà serve per entrambi, a seconda di quale lato dell'uguale sta. E ogni volta che scrivi, la pagina si aggiorna da sola. Nel prossimo capitolo andremo oltre il testo: creare e rimuovere interi elementi, gestire classi. Ma leggere/scrivere textContent e style è già abbastanza per fare cose vive — come nel laboratorio.

🔑 Quello che devi portarti dietro dal Capitolo 12
  1. Il DOM è la pagina vista come albero di oggetti: il ponte tra JavaScript e ciò che l'utente vede. Il punto di ingresso è document (§12.1–12.2).
  2. Selezionare: document.querySelector("selettore CSS") (il primo), querySelectorAll(…) (tutti). Se non trova, dà null (§12.3).

Ora sai trovare un elemento e cambiarne il contenuto e lo stile. È già moltissimo. Ma finora cambiamo elementi che esistono già nell'HTML. E se volessi creare nuovi elementi (una nuova riga in una lista), rimuoverli, o gestire in modo pulito le classi CSS invece di impostare stili uno a uno? È il tema del Capitolo 13: manipolare il DOM per davvero — costruire e smontare pezzi di pagina.

Sezione 12.7Soluzioni

Soluzione 12.1

Il DOM è la rappresentazione viva, a oggetti, che il browser costruisce dal tuo HTML per disegnare e gestire la pagina. Il file HTML è testo statico su disco; il DOM è la sua versione «in memoria», un albero di oggetti che JavaScript può leggere e modificare in tempo reale — e ogni modifica si riflette subito sullo schermo, senza toccare il file originale.

Soluzione 12.2
JS
const msg = document.querySelector("#messaggio");

Oppure, equivalente: const msg = document.getElementById("messaggio"); (qui senza il #).

Soluzione 12.3
JS
msg.textContent = "Fatto!";
Soluzione 12.4

«null» significa che querySelector non ha trovato l'elemento, quindi textContent viene chiesto al «niente». Le due cause tipiche: (1) non esiste un elemento con id="box" (id sbagliato o assente nell'HTML); (2) lo <script> è eseguito prima che quell'elemento sia stato creato dal browser — soluzione: metti lo script in fondo al <body> (o attendi il caricamento della pagina).
