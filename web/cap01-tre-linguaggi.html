Capitolo 1

Il web e i suoi tre linguaggi

Ogni pagina web che apri — un giornale, un social, questo stesso libro — è costruita con tre linguaggi che collaborano, ognuno con un ruolo preciso: HTML le dà la struttura (cosa c'è: titoli, testi, immagini), CSS l'aspetto (com'è: colori, font, disposizione), JavaScript il comportamento (cosa fa: reagire ai click, aggiornarsi). Impararli significa poter costruire il web, non solo navigarlo. E c'è una notizia bellissima: per iniziare ti bastano un editor di testo e un browser — niente da installare, niente da compilare. Scrivi, salva, apri: la tua pagina è viva. In questo capitolo conosciamo i tre linguaggi, i loro ruoli, e creiamo la tua primissima pagina.

🗺 Dove siamo

Cosa succede quando apri una pagina (§1.1), i tre ruoli di HTML, CSS e JavaScript (§1.2), la tua prima pagina (§1.3), gli strumenti che servono (§1.4). Nel laboratorio (§1.5) crei e apri il tuo primo file HTML.

🔑 Perché ci tocca: da utente a creatore

Il web è il mezzo più diffuso al mondo: saperlo costruire ti apre infinite possibilità — un sito personale, un'interfaccia per un tuo programma, una piccola app che chiunque può usare dal browser. È anche la via d'accesso più immediata alla programmazione visuale: vedi subito il risultato di ciò che scrivi. E getta le basi per il libro sul backend: il web ha un «davanti» (ciò che vedi, questo libro) e un «dietro» (i server che lo alimentano). Cominciamo dal davanti.

Sezione 1.1Cosa succede aprendo una pagina

Quando apri un sito, il tuo browser (Chrome, Firefox, Safari…) scarica dei file di testo e li trasforma nella pagina colorata e interattiva che vedi. Capire questo passaggio è il primo passo.

📐 Il browser come interprete

Un browser riceve, dal server, dei file di testo: un documento HTML (il contenuto e la struttura), uno o più fogli CSS (lo stile) e degli script JavaScript (il comportamento). Il suo lavoro è leggerli e «renderizzarli»: costruire la pagina visibile, applicare gli stili, ed eseguire il codice interattivo. La cosa sorprendente: sono solo file di testo che tu potresti scrivere in un editor. Non c'è magia — c'è una grammatica precisa che il browser sa interpretare, e che stai per imparare. (Come i dati arrivino dal server al browser è la storia del libro «Reti e Internet»; qui ci concentriamo su cosa succede dentro il browser.)

Sezione 1.2HTML, CSS, JavaScript: i tre ruoli

I tre linguaggi non competono: collaborano, ognuno responsabile di un aspetto. Tenere distinti i loro ruoli è il concetto più importante di tutto il libro.

💡 L'analogia: costruire una casa

Pensa a una pagina come a una casa. L'HTML è la struttura: muri, stanze, porte — cosa c'è e come è organizzato (un titolo, un paragrafo, un pulsante). Il CSS è l'arredamento e la pittura: colori delle pareti, mobili, disposizione — come appare. Il JavaScript è l'impianto elettrico e la domotica: ciò che fa qualcosa quando interagisci (premi l'interruttore, la luce si accende). Puoi avere una casa solo con la struttura (HTML nudo, brutta ma funzionante); aggiungere l'arredo (CSS, diventa bella); e infine l'interattività (JS, diventa viva). Il widget qui sotto costruisce una pagina esattamente così, strato dopo strato.

Sezione 1.3La tua prima pagina

Vediamo com'è fatto il più piccolo documento HTML completo. Non serve capire ogni riga ora — la smonteremo nel Capitolo 2 — ma riconoscerne la forma.

Il documento minimo lo scheletro di ogni pagina
index.html
<!DOCTYPE html>              <!-- "questo è HTML moderno" -->
<html lang="it">
<head>                       <!-- informazioni sulla pagina (non visibili) -->
    <title>La mia pagina</title>
</head>
<body>                       <!-- il contenuto visibile -->
    <h1>Ciao, web!</h1>
    <p>La mia prima pagina.</p>
</body>
</html>
i tag tra parentesi angolari
l'HTML è fatto di tag come <h1> (un titolo) e <p> (un paragrafo). Quasi sempre vanno in coppia: uno di apertura (<p>) e uno di chiusura con la barra (</p>), che racchiudono il contenuto.
la struttura: head e body
<head> contiene informazioni sulla pagina (il titolo nella scheda del browser, i collegamenti a CSS…), non visibili; <body> contiene tutto ciò che vedi nella pagina. Li vedremo bene nel Cap. 2.
salvalo come .html
salva questo testo in un file con estensione .html, poi fai doppio clic: si apre nel browser, che lo renderizza. È tutto qui: nessuna installazione, nessuna compilazione.

Sezione 1.4Gli strumenti: editor e browser

Per fare sviluppo web ti servono solo due cose, entrambe gratuite e che probabilmente hai già: un editor per scrivere il codice e un browser per vederlo.

Il tuo kit da sviluppatore web gratis, subito
un editor di testo
tecnicamente basta il Blocco note, ma molto meglio VS Code (gratuito): colora il codice, segnala errori, completa i tag. È lo strumento standard degli sviluppatori web.
un browser
ne hai già uno. Sarà sia il modo di vedere la pagina sia, con gli strumenti per sviluppatori (tasto F12), un potentissimo laboratorio per ispezionare e provare — lo useremo tantissimo.
il ciclo di lavoro
scrivi nell'editor → salva → ricarica la pagina nel browser (F5). Modifica, salva, ricarica: vedi subito il risultato. Immediato e senza attese — uno dei motivi per cui il web è così divertente da imparare.
Bonus: gli strumenti per sviluppatori (F12) sono un mondo. La scheda Elements ti mostra l'HTML «vivo» della pagina e ti fa modificarne lo stile al volo; la Console esegue JavaScript (Cap. 11); la scheda Network mostra i file scaricati (il legame col libro di Reti!). Su qualsiasi sito, premi F12 e curiosa: è il modo migliore per imparare guardando come sono fatti i siti che ami.

Sezione 1.5Laboratorio: ciao web

Il rito di passaggio: crei, salvi e apri la tua prima pagina web. Cinque minuti, e avrai fatto sviluppo web per davvero.

Dal file alla pagina scrivi · salva · apri
1. scrivi
apri l'editor (VS Code o Blocco note) e scrivi il documento della §1.3 — o parti anche solo da questo, ancora più minimale:
ciao.html
<!DOCTYPE html>
<html>
<body>
    <h1>Ciao, web!</h1>
    <p>Questa è la mia prima pagina. 🎉</p>
</body>
</html>
2. salva
salvalo come ciao.html (attenzione all'estensione .html, non .txt!). Ricordati la cartella.
3. apri
trova il file e fai doppio clic: si aprirà nel browser, mostrando il titolo grande «Ciao, web!» e il paragrafo sotto. Complimenti: hai creato una pagina web.
Ora sperimenta: cambia il testo, aggiungi un altro <p>…</p>, salva e ricarica (F5) — vedi il cambiamento all'istante. Poi premi F12 e guarda la scheda «Elements»: ritroverai il tuo HTML, «vivo». Questo ciclo modifica-salva-ricarica è tutto ciò che serve per costruire il web: nei prossimi capitoli riempiremo questa pagina di contenuti (HTML), la renderemo bella (CSS) e interattiva (JS).
🔑 Quello che devi portarti dietro dal Capitolo 1
  1. Il browser trasforma file di testo (HTML, CSS, JS) nella pagina che vedi: li «renderizza» (§1.1).
  2. Tre linguaggi, tre ruoli: HTML = struttura (cosa c'è), CSS = aspetto (com'è), JavaScript = comportamento (cosa fa). Tenerli distinti è la chiave (§1.2).
  3. Un documento HTML ha una forma fissa (<!DOCTYPE>, <html>, <head>, <body>) fatta di tag in coppia; si salva .html e si apre nel browser (§1.3).
  4. Ti bastano un editor (VS Code) e un browser (con F12, gli strumenti sviluppatore). Ciclo: scrivi → salva → ricarica (§1.4-1.5).

Hai creato la tua prima pagina — ma era spoglia. È ora di imparare a strutturare il contenuto per bene. Nel Capitolo 2 smontiamo l'HTML pezzo per pezzo: l'anatomia completa di un documento, la differenza tra elementi, tag e attributi, i ruoli di <head> e <body>, e il concetto fondamentale dell'annidamento — le scatole dentro le scatole che danno struttura a ogni pagina. Sono le fondamenta su cui poggeranno CSS e JavaScript: partiamo da qui.

Sezione 1.6Esercizi

Esercizio 1.1

Associa ogni linguaggio al suo ruolo: HTML, CSS, JavaScript ↔ struttura, comportamento, aspetto. Fai un esempio concreto per ciascuno (es. «il colore di sfondo è…»).

Esercizio 1.2

Cosa fa il browser con i file HTML/CSS/JS che riceve? Con parole tue, cosa significa «renderizzare» una pagina?

Esercizio 1.3

(Pratico) Crea ciao.html come nel laboratorio, aprilo, poi aggiungi una tua frase in un nuovo <p>, salva e ricarica. Ha funzionato?

Esercizio 1.4

(Esplorativo) Vai su un sito che ti piace, premi F12 e apri la scheda «Elements». Riconosci qualche tag (<h1>, <p>, <img>)? Prova a modificare un testo dal pannello (non salva nulla, è solo la tua copia locale).

Sezione 1.7Soluzioni

Soluzione 1.1

HTML → struttura (es. «c'è un titolo e sotto un paragrafo»); CSS → aspetto (es. «il colore di sfondo è azzurro, il testo è grande»); JavaScript → comportamento (es. «cliccando il pulsante appare un messaggio»).

Soluzione 1.2

Il browser legge i file di testo e li trasforma nella pagina visibile: costruisce gli elementi dall'HTML, applica gli stili del CSS ed esegue il JavaScript. «Renderizzare» significa proprio questo: produrre, a partire dai file, la pagina grafica e interattiva che appare sullo schermo.

Soluzione 1.3

Aggiungendo <p>La mia frase.</p> dentro il <body>, dopo aver salvato e ricaricato, la nuova frase appare sotto le altre. Se non compare: controlla di aver salvato (con estensione .html) e di aver ricaricato la pagina giusta.

Soluzione 1.4

Nella scheda «Elements» vedrai l'albero dell'HTML del sito: quasi certamente riconoscerai tag come <div>, <a> (link), <img>, <p>. Modificando un testo lì, cambia solo nella tua visualizzazione (una copia locale nel browser): al ricaricamento torna l'originale. È un laboratorio sicuro per curiosare e imparare.
