Capitolo 7

I selettori

Finora abbiamo stilizzato interi tipi di elemento — «tutti gli h1». Ma il vero controllo arriva quando puoi dire «solo questo elemento» o «solo questo gruppo»: colorare di rosso un prezzo, evidenziare un pulsante, applicare uno stile a una certa categoria di elementi. È il lavoro dei selettori — la parte del CSS che decide a chi si applicano gli stili. In questo capitolo padroneggiamo l'arsenale: oltre ai tag, le classi e gli id (i «ganci» intravisti col form), come combinarli, le magiche pseudo-classi come :hover (che reagiscono al mouse!), e la specificità — la regola che decide chi vince quando più stili si contendono lo stesso elemento. Selezionare con precisione è metà del mestiere del CSS.

🗺 Dove siamo

I selettori di tag (§7.1), classi e id (§7.2), come combinare i selettori (§7.3), le pseudo-classi come :hover (§7.4), e la specificità (§7.5). Nel laboratorio (§7.6) selezioni con precisione.

🔑 Perché ci tocca: dire esattamente «a chi»

I selettori sono ovunque nel CSS: ogni regola inizia con uno. Saperli usare bene è la differenza tra stilizzare «a martellate» (tutti gli elementi uguali) e con precisione chirurgica (esattamente ciò che vuoi). E le classi, in particolare, sono anche il gancio principale che il JavaScript userà per trovare e manipolare gli elementi (Cap. 12-13). Padroneggiare i selettori è la base sia per stilizzare sia per rendere interattivo.

Sezione 7.1Selettori di tag

Il selettore più semplice, che già conosci: il nome del tag. Seleziona tutti gli elementi di quel tipo nella pagina.

Selettore di tag tutti gli elementi di un tipo
stile.css
p { color: gray; }        /* TUTTI i paragrafi */
a { color: green; }       /* TUTTI i link */
li { font-weight: bold; } /* TUTTE le voci di lista */
Semplice e potente per gli stili «di base» che vuoi ovunque: il font del body, il colore dei link, l'aspetto dei titoli. Ma è «tutto o niente»: p colpisce ogni paragrafo, senza distinzioni. Quando ti serve selezionare solo alcuni elementi, entrano in gioco le classi (§7.2).

Sezione 7.2Classi e id

Ecco i selettori più usati in assoluto. La classe è un'etichetta che puoi mettere su quanti elementi vuoi; l'id è un'etichetta unica per un solo elemento. Sono i «ganci» per stile e JavaScript.

Classi (.) e id (#) gruppi e singoli
HTML + CSS
<!-- nell'HTML, assegno class e id agli elementi -->
<p class="importante">Attenzione!</p>
<p>Un paragrafo normale.</p>
<p class="importante">Anche questo!</p>
<h1 id="titolo-principale">Home</h1>

/* nel CSS, li seleziono così: */
.importante { color: red; }          /* punto (.) = classe: i DUE p con class="importante" */
#titolo-principale { font-size: 40px; } /* cancelletto (#) = id: quell'UNICO h1 */
la classe: .nome
nell'HTML metti class="nome"; nel CSS la selezioni con il punto: .nome. Puoi mettere la stessa classe su tanti elementi (per stilizzarli insieme) e più classi su un elemento (class="grande rosso"). È il selettore più usato: flessibile e riutilizzabile.
l'id: #nome
nell'HTML id="nome" (unico nella pagina!); nel CSS col cancelletto: #nome. Per un elemento specifico e singolo (un'intestazione principale, un contenitore preciso).
classe o id?
regola pratica: usa quasi sempre le classi (flessibili, riutilizzabili); riserva l'id a elementi davvero unici. Un motivo in più: gli id hanno un'alta «specificità» che complica le cose (§7.5). Nel dubbio, classe.

Sezione 7.3Combinare i selettori

I selettori si combinano per essere più precisi: «i paragrafi dentro quel contenitore», «gli elementi con entrambe le classi». Poche combinazioni coprono quasi tutto.

Combinare i selettori discendenza, gruppi, condizioni
stile.css
nav a { color: white; }        /* i link DENTRO un nav (spazio = discendente) */
h1, h2, h3 { color: navy; }    /* h1 E h2 E h3 (virgola = gruppo) */
p.importante { color: red; }    /* i p che HANNO la classe importante */
discendente: nav a (spazio)
uno spazio tra due selettori significa «dentro»: nav a = «i link contenuti in un <nav>». Utilissimo per stilizzare elementi solo in un certo contesto (i link del menu diversi dai link del testo).
gruppo: h1, h2, h3 (virgola)
la virgola applica lo stesso stile a più selettori: «tutti gli h1, gli h2 e gli h3». Evita di ripetere le stesse dichiarazioni.
e: p.importante (attaccati)
senza spazio, «un <p> che ha anche la classe importante» (non gli altri p, non gli altri elementi con quella classe). Restringe con precisione.
Con queste tre combinazioni — dentro (spazio), e anche (virgola per il gruppo), con questa caratteristica (attaccato) — copri la maggior parte dei casi. Esistono selettori più avanzati (figlio diretto >, fratelli +…), ma questi bastano per iniziare a lavorare con precisione.

Sezione 7.4Pseudo-classi (:hover…)

Le pseudo-classi selezionano gli elementi in un particolare stato — quando ci passi sopra il mouse, quando un campo è a fuoco, il primo di una lista. La più iconica: :hover, il primo tocco di interattività fatto col solo CSS.

Le pseudo-classi stati: :hover, :focus…
stile.css
a:hover { color: orange; }          /* quando ci passi SOPRA col mouse */
button:hover { background: #5a3ce0; }  /* il pulsante si "illumina" al passaggio */
input:focus { border-color: blue; } /* quando il campo è selezionato */
:hover — al passaggio del mouse
applica gli stili solo quando l'utente ci passa sopra col mouse. È il modo più semplice per rendere un elemento «reattivo» senza JavaScript: link che cambiano colore, pulsanti che si illuminano, card che si sollevano. Provalo nel widget!
:focus — quando è a fuoco
per i campi di input: applica lo stile quando il campo è selezionato (ci stai scrivendo). Ottimo per evidenziare il campo attivo di un form.
altre utili
:first-child / :last-child (il primo/ultimo figlio), :nth-child(odd) (le righe dispari, per tabelle a righe alternate). Ce ne sono molte, ma :hover è quella che userai da subito.
Selettori dal vivo: chi viene colpito? scegli un selettore e guarda cosa evidenzia (poi passa il mouse sull'ultimo)

Sezione 7.5La specificità


  

    Domanda cruciale: se due regole danno stili diversi allo stesso elemento, chi vince? La risposta è
    la specificità — e capirla risolve il misterioso «ho scritto la regola ma non funziona!».
  


  

    
📐 Chi è più «specifico» vince

    

      Quando più regole si applicano allo stesso elemento e sono in conflitto, il browser sceglie la più
      specifica. La scala, dal meno al più forte: selettore di tag (p) < classe
      (.nota) < id (#titolo) < stile in linea
      (style="..."). Quindi .nota { color: red } batte p { color: gray }
      su un paragrafo con classe nota: la classe è più specifica del tag. A parità di specificità, vince
      l'ultima regola scritta (l'ordine conta). E c'è !important, che forza la vittoria — ma è
      un martello da usare pochissimo, perché rompe la logica normale e rende il CSS difficile da mantenere.
    

  


  

    
💡 Il segreto per non impazzire

    

      La maggior parte dei «bug CSS» del principiante è specificità: «ho scritto p { color: blue } ma
      il paragrafo resta rosso!» — perché c'è una regola più specifica (una classe, un id) che vince. Il
      consiglio d'oro: usa soprattutto le classi, con specificità bassa e uniforme, evita id e
      !important per lo stile, e tieni il CSS «piatto». Così le regole si comportano in modo prevedibile
      e i conflitti sono rari. Quando qualcosa «non prende», apri F12 → Elements: il browser ti mostra tutte
      le regole applicate a un elemento e quale vince (barrando quelle sovrascritte). È lo strumento
      diagnostico definitivo.
    

  



  
  
Sezione 7.6Laboratorio: selezionare con precisione


  

    Mettiamo in pratica: una piccola pagina con classi ben scelte, stilizzata con precisione — inclusa
    un'interazione :hover.
  


  

    
Il file selettori.html classi + hover

    

      
selettori.html (estratto)

      
<style>
    .card {
        border: 1px solid #ddd;
        border-radius: 12px;
        padding: 1rem;
        transition: transform .2s;   /* animazione morbida */
    }
    .card:hover {
        transform: translateY(-4px);  /* si "solleva" al passaggio */
        box-shadow: 0 8px 20px rgba(0,0,0,.1);
    }
    .prezzo { color: #e0654f; font-weight: bold; }
</style>

<div class="card">
    <h3>Prodotto</h3>
    <p>Prezzo: <span class="prezzo">9,99 €</span></p>
</div>

    

    

      Apri e passa il mouse sulla card: si solleva con un'ombra, con un'animazione morbida — e non abbiamo
      scritto una riga di JavaScript! È tutta la potenza di :hover + transition. Il
      <span class="prezzo"> mostra come una classe su una porzione di testo (il prezzo) la
      stilizzi senza toccare il resto della frase (il Cap. 3 torna utile). Nota il selettore combinato
      .card:hover: «una card, quando ci passi sopra». Sperimenta: cambia i colori, aggiungi altre card,
      prova :hover su altri elementi. Con classi ben scelte e :hover, il tuo CSS diventa preciso
      e persino animato.
    

  


  

    
🔑 Quello che devi portarti dietro dal Capitolo 7

    
    
      
  1. Selettori base: tag (p, tutti), classe (.nome, su quanti vuoi —
          la più usata), id (#nome, unico) (§7.1-7.2).
    2. 
      
  2. Combina: dentro (spazio, nav a), gruppo (virgola, h1, h2),
          e anche (attaccato, p.nota) (§7.3).
    3. 
      
  3. Le pseudo-classi selezionano stati: :hover (mouse sopra — interattività col solo CSS!),
          :focus (§7.4).
    4. 
      
  4. La specificità decide chi vince: tag < classe < id < inline; a parità, l'ultima regola.
          Usa le classi, evita id/!important, e F12 per diagnosticare (§7.5).
    5. 
    

  


  

    Sai selezionare con precisione e applicare stili. Ma per disporre gli elementi nello spazio — dimensioni,
    spaziature, allineamenti — serve capire come il browser «pensa» ogni elemento. Nel Capitolo 8
    affrontiamo il box model: l'idea che ogni elemento è una scatola fatta di strati —
    contenuto, padding, border, margin. Capire questi strati (e la sottigliezza del
    box-sizing) è il prerequisito per qualsiasi layout — ed è dove molti principianti si bloccano,
    finché non «vedono» le scatole. Con un widget interattivo, le vedrai chiaramente.

      (c) l'elemento con id menu?
    

  


  

    
Esercizio 7.2

    

      Qual è la differenza tra classe e id? Perché nella pratica si preferiscono le classi?
    

  


  

    
Esercizio 7.3

    

      Cosa fa il selettore nav a (con lo spazio)? E a:hover?
    

  


  

    
Esercizio 7.4

    

      Hai p { color: blue } e .avviso { color: red }, e un paragrafo
      <p class="avviso">. Di che colore sarà, e perché?
    

  



  
  
Sezione 7.8Soluzioni


  

    
Soluzione 7.1

    

      (a) li { … } (selettore di tag); (b) .evidenza { … } (classe, col punto); (c)
      #menu { … } (id, col cancelletto).
    

  


  

    
Soluzione 7.2

    

      La classe (.nome) si può applicare a tanti elementi (ed è riutilizzabile);
      l'id (#nome) è unico per un solo elemento. Si preferiscono le classi perché sono
      flessibili, riutilizzabili e hanno una specificità bassa e uniforme, il che rende il CSS più prevedibile e
      facile da mantenere.
    

  


  

    
Soluzione 7.3

    

      nav a seleziona i link (<a>) contenuti dentro un <nav>
      (lo spazio = discendente): solo quelli, non tutti i link della pagina. a:hover seleziona un link
      quando ci passi sopra col mouse (pseudo-classe di stato): applica lo stile solo in quel momento.
    

  


  

    
Soluzione 7.4

    

      Sarà rosso. Entrambe le regole si applicano, ma la classe (.avviso) è più
      specifica del selettore di tag (p), quindi vince: color: red. È la regola
      della specificità: classe > tag.
