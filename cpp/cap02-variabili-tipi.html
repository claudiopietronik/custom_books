Capitolo 2

Variabili e tipi

Qui il C++ mostra il suo carattere. In molti linguaggi una variabile può contenere «qualsiasi cosa»; in C++ ogni variabile ha un tipo dichiarato — un intero, un decimale, un carattere — deciso al momento della dichiarazione e immutabile. È la tipizzazione statica, e il compilatore la fa rispettare con rigore: è più «severa» all'inizio, ma cattura una montagna di errori prima che il programma giri, e permette al C++ di essere velocissimo. In questo capitolo conosciamo i tipi fondamentali, impariamo a dichiarare e inizializzare le variabili nel modo giusto, e leggiamo l'input dell'utente con cin. Capire i tipi è capire la mentalità del C++: dì al compilatore esattamente cosa vuoi, e lui ti ripagherà con velocità e sicurezza.

🗺 Dove siamo

La tipizzazione statica e perché conta (§2.1), i tipi fondamentali int, double, char, bool (§2.2), come dichiarare e inizializzare (§2.3), le parole chiave const e auto (§2.4), e leggere l'input con cin (§2.5). Nel laboratorio (§2.6) costruisci una calcolatrice.

🔑 Perché ci tocca: la disciplina che protegge

La tipizzazione statica è la caratteristica che più distingue il C++ da linguaggi come Python. Sembra una scocciatura, è in realtà una rete di sicurezza: dichiarando i tipi, il compilatore trova al volo errori (assegnare un testo a un numero, usare male una variabile) che altrove esploderebbero solo a runtime. È anche il segreto delle prestazioni: sapendo in anticipo che x è un intero di 4 byte, il compilatore genera codice macchina ottimale. Capire i tipi è il primo passo per pensare in C++.

Sezione 2.1Tipizzazione statica

In C++ dichiari il tipo di ogni variabile, e quel tipo non cambia mai. Il compilatore lo verifica a ogni uso. È l'opposto dei linguaggi «dinamici», dove una variabile può cambiare natura al volo.

📐 Statico vs dinamico

In un linguaggio dinamico (Python) puoi scrivere x = 5 e poi x = "ciao": la stessa variabile cambia tipo, e nessuno controlla nulla finché il programma non gira. In C++, invece, dichiari il tipo una volta: int x = 5; — e da quel momento x è per sempre un intero. Provare a metterci del testo dà un errore di compilazione, subito. Il tipo è un contratto che il compilatore fa rispettare: più rigore in cambio di più sicurezza e velocità. Non è una limitazione fastidiosa: è ciò che rende il C++ affidabile su programmi enormi, dove un errore di tipo scoperto tardi costerebbe carissimo.

Sezione 2.2int, double, char, bool

I quattro tipi fondamentali che userai di continuo. Ognuno rappresenta una categoria di dato e occupa una quantità precisa di memoria — un dettaglio che in C++ conta.

I tipi base interi, decimali, caratteri, verità
int — numeri interi
senza virgola: 42, -7. Occupa di solito 4 byte, quindi ha un intervallo limitato (circa ±2 miliardi). Per contare, indicizzare, quantità intere.
double — numeri decimali
con la virgola (anzi, il punto): 3.14, -0.5. «Double» = doppia precisione, lo standard per i decimali. (Esiste float, meno preciso: usa double.)
char — un singolo carattere
una sola lettera o simbolo, tra apici singoli: 'A', '?'. Occupa 1 byte. Attento: 'A' (apici, un char) è diverso da "A" (virgolette, un testo).
bool — vero/falso
solo true o false (minuscoli). La base delle decisioni (Cap. 4).
Perché in C++ conta «quanti byte»? Perché il C++ ti fa lavorare vicino alla memoria: un int ha un intervallo finito, e se lo superi vai in overflow (Cap. 3). Per il testo (più caratteri) c'è il tipo std::string, che vedremo nel Cap. 9 — un char da solo è un singolo carattere. Per numeri interi più grandi c'è long long; per ora int e double coprono quasi tutto.

Sezione 2.3Dichiarazione e inizializzazione

Dichiarare una variabile significa dire al compilatore il suo tipo e nome. Inizializzarla significa darle un valore di partenza. In C++ è importante fare entrambe le cose — e farle bene.

Dichiarare e dare un valore tipo nome = valore;
esempio.cpp
int eta = 25;            // dichiara un int di nome eta, valore 25
double prezzo = 3.5;     // un double
char voto = 'A';         // un char (apici singoli!)
bool promosso = true;    // un bool

int contatore;              // DICHIARATO ma NON inizializzato: valore "spazzatura"!
la struttura: tipo nome = valore;
prima il tipo, poi il nome, poi = e il valore, e il ; finale. Da lì in poi usi nome come in ogni linguaggio.
la trappola: variabili non inizializzate
in C++, dichiarare int contatore; senza valore non la mette a zero: contiene un valore casuale («spazzatura», ciò che c'era in quella cella di memoria)! Usarla così è un bug classico e insidioso. Regola d'oro: inizializza sempre le variabili quando le dichiari.
lo stile moderno con le graffe
C++ moderno preferisce int eta{25}; (inizializzazione con graffe): è più sicura, perché impedisce conversioni pericolose. Entrambe le forme vanno bene; l'importante è inizializzare.

Sezione 2.4const e auto

Due parole chiave molto usate: const per valori che non devono cambiare, e auto per far dedurre il tipo al compilatore. Entrambe rendono il codice più sicuro e leggibile.

const e auto immutabile · tipo dedotto
esempio.cpp
const double PI = 3.14159;   // costante: non può essere modificata
// PI = 4;   →  ERRORE di compilazione: PI è const!

auto x = 42;              // auto: il compilatore deduce "int" dal valore
auto nome = "Anna";         // deduce il tipo testo
const — non toccare!
rende una variabile immutabile: tentare di cambiarla è un errore di compilazione. Usalo per valori fissi (π, un'aliquota, una configurazione). Dichiarare const ciò che non deve cambiare previene bug ed esprime l'intenzione — è considerato buono stile: «const per default».
auto — deduci tu il tipo
chiedi al compilatore di dedurre il tipo dal valore iniziale. auto x = 42; rende x un int. Non è «tipizzazione dinamica»: il tipo è comunque fisso e statico, solo scritto dal compilatore. Utilissimo con tipi lunghi e complicati (li vedrai con la STL, Cap. 13). Va sempre inizializzato (senza valore, non c'è nulla da dedurre).

Sezione 2.5cin: leggere dall'utente

Speculare a cout: std::cin («character input») legge ciò che l'utente digita e lo mette in una variabile. Usa l'operatore >>, che «estrae» dallo stream.

Leggere con cin >> estrae nella variabile
eta.cpp
#include <iostream>

int main() {
    int eta;                          // dichiaro la variabile che riceverà l'input
    std::cout << "Quanti anni hai? ";
    std::cin >> eta;                // legge un intero e lo mette in eta
    std::cout << "Tra un anno ne avrai " << eta + 1 << std::endl;
    return 0;
}
std::cin >> eta;
legge dall'input e lo estrae nella variabile eta. Nota la freccia opposta a cout: >> «tira dentro» (dall'input alla variabile), << «spinge fuori» (dalla variabile allo schermo).
la conversione è automatica per tipo
bella comodità del C++: siccome eta è un int, cin legge direttamente un numero — non serve convertire come in altri linguaggi. Il tipo della variabile guida la lettura. Puoi anche leggere più valori: cin >> a >> b;.
Attenzione: se l'utente scrive testo dove ti aspetti un numero, cin «fallisce» silenziosamente (la variabile resta a un valore indefinito e le letture successive saltano). La gestione robusta dell'input la vedremo con le eccezioni (Cap. 14). Per ora, dai per scontato input corretti. E per leggere un'intera riga con spazi (non una sola parola) servirà std::getline, che incontreremo con le stringhe (Cap. 9).

Sezione 2.6Laboratorio: una calcolatrice

Uniamo tipi, dichiarazioni, cin e cout in un programma utile: una calcolatrice che chiede due numeri e mostra somma, differenza, prodotto e quoziente. Attenzione ai tipi!

Il programma calcolatrice.cpp leggi · calcola · stampa
calcolatrice.cpp
#include <iostream>

int main() {
    double a, b;                    // double per gestire anche i decimali
    std::cout << "Primo numero: ";
    std::cin >> a;
    std::cout << "Secondo numero: ";
    std::cin >> b;

    std::cout << "Somma:     " << a + b << "\n";
    std::cout << "Prodotto:  " << a * b << "\n";
    std::cout << "Quoziente: " << a / b << std::endl;
    return 0;
}
Esecuzione (dopo g++ calcolatrice.cpp -o calc e ./calc):
Terminale
Primo numero: 7
Secondo numero: 2
Somma:     9
Prodotto:  14
Quoziente: 3.5
Scelta di tipo cruciale: abbiamo usato double, non int. Se a e b fossero int, 7 / 2 darebbe 3 (divisione intera, i decimali scartati!) invece di 3.5. Il tipo determina il comportamento — lo approfondiremo nel Capitolo 3. Prova a cambiare double in int e osserva il quoziente «tagliato»: è una lezione che vale più di mille parole.
🔑 Quello che devi portarti dietro dal Capitolo 2
  1. Il C++ è a tipizzazione statica: ogni variabile ha un tipo dichiarato e fisso, controllato dal compilatore — più sicurezza e velocità (§2.1).
  2. Tipi base: int (interi), double (decimali col punto), char (un carattere, apici singoli), bool (true/false) (§2.2).
  3. tipo nome = valore; per dichiarare e inizializzare. Inizializza SEMPRE: una variabile non inizializzata contiene spazzatura (§2.3). const = immutabile, auto = tipo dedotto (§2.4).
  4. std::cin >> var; legge l'input (converte in base al tipo della variabile) (§2.5). E la scelta int vs double cambia i risultati (§2.6).

Ora i tuoi programmi ricordano dati con tipi precisi e dialogano con l'utente. Ma abbiamo appena intravisto un dettaglio spinoso — 7 / 2 che fa 3 tra interi! — che merita attenzione. Come si comportano davvero gli operatori del C++? Cosa succede se un int supera il suo limite? Come si convertono i tipi tra loro? Nel Capitolo 3 studiamo operatori ed espressioni: l'aritmetica e le sue insidie (divisione intera, overflow), i confronti e gli operatori logici, e le regole di conversione — tutte cose che in un linguaggio «vicino alla macchina» hanno conseguenze molto concrete.

Sezione 2.8Soluzioni

Soluzione 2.1

Nella statica (C++) il tipo di ogni variabile è dichiarato e fisso, controllato dal compilatore prima dell'esecuzione; nella dinamica il tipo è associato al valore e può cambiare, controllato solo a runtime. Vantaggio della statica: gli errori di tipo si scoprono in compilazione (prima di eseguire), e il codice è più veloce.

Soluzione 2.2

(a) int (numero intero); (b) double (prezzo con decimali); (c) char (un solo carattere); (d) bool (vero/falso).

Soluzione 2.3

Perché una variabile non inizializzata non vale zero: contiene un valore casuale (la «spazzatura» che c'era in quella cella di memoria). Usarla porta a comportamenti imprevedibili e bug difficili da trovare. Regola: inizializza sempre alla dichiarazione (es. int contatore = 0;).

Soluzione 2.4
media.cpp
#include <iostream>

int main() {
    double a, b, c;
    std::cout << "Tre voti: ";
    std::cin >> a >> b >> c;
    std::cout << "Media: " << (a + b + c) / 3 << std::endl;
    return 0;
}

Le parentesi in (a + b + c) / 3 sono essenziali: senza, per la precedenza si dividerebbe solo c per 3 (Cap. 3).
