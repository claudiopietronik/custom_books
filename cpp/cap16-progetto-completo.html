Capitolo 16 · Il gran finale

Un progetto completo

Ci sei arrivato. Hai imparato tutti i mattoni del C++ — tipi, funzioni, puntatori, memoria, classi, template, STL, eccezioni, file. È il momento di costruire qualcosa di vero, dall'inizio alla fine. Realizzeremo un gestore di attività da terminale: una lista di cose da fare che ricorda tra un avvio e l'altro, salvando su file. Non è un frammento isolato: è un'applicazione con una sua architettura — una classe per modellare i dati, la STL per gestirli, i file per la persistenza, le eccezioni per la robustezza — dove ogni pezzo che hai imparato trova il suo posto. È qui che «conoscere il C++» diventa «saper costruire in C++»: il salto da studente a programmatore. E ti renderai conto di quanta strada hai fatto dal primo cout << "Ciao, mondo!".

🗺 Dove siamo

Dall'idea al piano (§16.1), progettare le classi (§16.2), scrivere il codice pezzo per pezzo (§16.3), compilare e provare (§16.4), migliorare (§16.5) e infine dove andare (§16.6). Prima, prova la versione finita nel widget!

🔑 Perché ci tocca: il metodo, non solo il codice

Questo capitolo insegna il metodo per affrontare un progetto reale: scomporre, progettare, costruire un pezzo alla volta, compilare, provare, migliorare. È la competenza che trasforma la conoscenza in capacità: non «sapere la sintassi», ma saper portare un'idea dal nulla a un programma funzionante e ben strutturato. È lo stesso metodo che userai per qualsiasi cosa costruirai in C++ e oltre.

Sezione 16.1L'idea e il piano

Ogni progetto parte da un'idea e da qualche domanda. Prima di scrivere codice, chiarisci cosa deve fare: è metà del lavoro.

📐 L'idea: un gestore di attività

Vogliamo un programma che permetta di: vedere le attività, aggiungerne, segnarle come fatte, e — cruciale — ricordarle tra un avvio e l'altro. Domande da porsi subito: come rappresento un'attività? (una classe con testo e stato «fatto»). Come tengo la lista? (un std::vector di attività). Dove salvo i dati? (un file di testo, Cap. 15). Come interagisce l'utente? (un menu che si ripete in un ciclo, Cap. 5). Rispondere a queste domande prima di programmare evita di riscrivere tutto a metà strada. Un piano anche approssimativo vale più di mille righe scritte a caso.

Sezione 16.2Progettare le classi

Con il piano in mano, decidiamo l'architettura: quali classi e funzioni servono. In C++, pensare per oggetti (Cap. 10) e per funzioni piccole (Cap. 6) rende tutto più semplice.

📐 Scomporre in classi e funzioni

Il cuore è una classe Attivita: incapsula un testo e un bool fatto, con metodi per completarla e mostrarla. Attorno, delle funzioni:

  • carica() — legge le attività dal file in un vector<Attivita> (o vuoto se il file non c'è);
  • salva(...) — scrive il vector sul file;
  • mostra(...) — stampa l'elenco numerato;
  • un ciclo menu che chiama queste funzioni in base alla scelta dell'utente, con gestione dell'input.

Ogni pezzo fa una cosa sola e bene: la classe modella il dato, le funzioni orchestrano. Così puoi scriverli e provarli uno alla volta, e se qualcosa non va sai dove guardare. È il «dividi e conquista» applicato al C++: un programma grande è tante parti piccole e chiare che collaborano.

Sezione 16.3Il codice, pezzo per pezzo

Ecco il programma completo. Leggilo notando come ogni concetto del libro trova il suo posto — dalla classe al vector, dai file alle eccezioni.

Il progetto attivita.cpp tutto il libro, in un file
attivita.cpp
#include <iostream>
#include <fstream>
#include <vector>
#include <string>

class Attivita {                     // la classe (Cap. 10)
public:
    std::string testo;
    bool fatto = false;
    Attivita(std::string t, bool f = false) : testo(t), fatto(f) {}
};

std::vector<Attivita> carica() {           // legge dal file (Cap. 15)
    std::vector<Attivita> v;
    std::ifstream in("attivita.txt");
    std::string riga;
    while (std::getline(in, riga)) {
        bool f = (riga[0] == '1');     // formato: "1|testo" o "0|testo"
        v.push_back(Attivita(riga.substr(2), f));
    }
    return v;
}

void salva(const std::vector<Attivita>& v) {   // const& : niente copia (Cap. 6)
    std::ofstream out("attivita.txt");
    for (const Attivita& a : v)
        out << (a.fatto ? "1|" : "0|") << a.testo << "\n";
}

void mostra(const std::vector<Attivita>& v) {
    for (std::size_t i = 0; i < v.size(); i++)
        std::cout << "  " << i + 1 << ". ["
                  << (v[i].fatto ? "x" : " ") << "] " << v[i].testo << "\n";
}

int main() {
    auto lista = carica();                  // auto = vector<Attivita> (Cap. 2)
    std::string scelta;
    while (true) {                        // ciclo menu (Cap. 5)
        std::cout << "\n--- ATTIVITÀ ---\n";
        mostra(lista);
        std::cout << "[a]ggiungi [f]atto [q]esci: ";
        std::cin >> scelta;

        if (scelta == "a") {
            std::cin.ignore();
            std::string t;
            std::cout << "Cosa? ";
            std::getline(std::cin, t);
            if (!t.empty()) { lista.push_back(Attivita(t)); salva(lista); }
        } else if (scelta == "f") {
            int n; std::cin >> n;
            if (n >= 1 && n <= (int)lista.size()) {   // controllo (Cap. 8)
                lista[n - 1].fatto = true; salva(lista);
            }
        } else if (scelta == "q") {
            break;
        }
    }
    std::cout << "Ciao!\n";
    return 0;
}
Fai il conto di quanti concetti collaborano: una classe con costruttore e lista di inizializzazione (Cap. 10), un vector (Cap. 8), file con ifstream/ofstream e getline (Cap. 15), const& per non copiare (Cap. 6), auto (Cap. 2), substr e l'operatore ternario (Cap. 9, 4), il ciclo menu con while/ if (Cap. 4-5), il controllo dei limiti (Cap. 8). Nessun pezzo, da solo, è difficile — ma insieme formano un'applicazione vera. È così che si costruisce il software: componendo tante idee semplici che hai imparato una alla volta.

Sezione 16.4Compilare e provare

Il codice scritto non è finito: va compilato (il C++ te lo ricorda!) e provato — specie nei casi in cui le cose vanno storte.

📐 Compila, poi metti alla prova i casi «cattivi»
Terminale
$ g++ -std=c++17 attivita.cpp -o attivita
$ ./attivita
  • Il caso normale: aggiungi un'attività, segnala fatta, esci e riapri — c'è ancora? (la persistenza su file funziona).
  • I casi limite: lista vuota all'avvio; testo vuoto; segnare fatto un numero inesistente (5 su 3 attività). Il controllo if (n >= 1 && n <= size) previene l'accesso fuori limite — senza, sarebbe un bug pericoloso (Cap. 8).
  • Se il compilatore protesta, leggi la prima riga d'errore (Cap. 14) e correggi. È il ciclo del C++: scrivi, compila, correggi, riprova.

Provare pensando «come potrebbe rompersi?» è ciò che distingue il codice fragile da quello robusto.

Sezione 16.5Migliorare

Un programma che funziona è l'inizio, non la fine. Con un occhio critico, si trovano sempre modi per renderlo più utile, più robusto, più «C++ moderno».

💡 Idee per far crescere il progetto
  • Funzionalità: rimuovere un'attività (lista.erase(...)); una priorità o una scadenza; contare quante sono fatte / da fare.
  • Qualità del codice: spostare la classe e le funzioni in file separati (header + cpp, Cap. 6); rendere i campi di Attivita privati con metodi (Cap. 10); usare gli algoritmi STL (Cap. 13) per contare o cercare.
  • Robustezza: gestire un input non-numerico nella scelta «fatto» con try/catch o controllando cin (Cap. 14); un file corrotto.

Il codice migliore nasce dalla riscrittura: la prima versione fa funzionare le cose, le successive le fanno funzionare bene. Non aver paura di rimettere mano a ciò che hai scritto — è lì che si impara di più, e dove un progetto diventa davvero tuo.

Sezione 16.6Dove andare da qui

Hai finito il libro — e sai programmare in C++. Non è un traguardo da poco: il C++ è uno dei linguaggi più profondi in circolazione, e ne hai le fondamenta solide. Ecco alcune direzioni.

🧭 Il cammino continua
  • Costruisci. La cosa più importante: fai progetti tuoi. Un gioco a caratteri, uno strumento da riga di comando, un piccolo motore fisico. Il C++ si impara scrivendolo (e compilandolo!).
  • Approfondisci il C++ moderno. C'è molto oltre le basi: move semantics, template avanzati, concorrenza (thread), le novità di C++20/23. Ma ora hai il vocabolario per affrontarle.
  • Esplora dove regna il C++. Motori di gioco (SFML per iniziare, poi Unreal), grafica, sistemi embedded, calcolo ad alte prestazioni: scegli un ambito che ti appassiona e tuffati.
  • La collana continua. Dopo i linguaggi, la parte web (HTML, CSS, JavaScript) e il backend (server, database): mondi diversi, dove porterai il rigore che il C++ ti ha insegnato.
🔑 Quello che devi portarti dietro da tutto il libro
  1. Le basi: tipi statici, operatori (e le loro insidie), decisioni, cicli, funzioni con passaggio per valore/riferimento.
  2. Il cuore del C++: puntatori e memoria (stack/heap), e l'idea che rende tutto sicuro — RAII e gli smart pointer.
  3. L'OOP: classi, incapsulamento, ereditarietà e polimorfismo; e la potenza di template e STL per fare tanto con poco.
  4. Gli strumenti pratici: eccezioni e file. E soprattutto: sai portare un'idea dal nulla a un programma C++ funzionante e ben strutturato. Il resto è pratica.

Complimenti: hai imparato il C++. Ricorda com'eri all'inizio del Capitolo 1 — davanti a un g++ da installare e a un cout << "Ciao, mondo!" da compilare — e guardati ora, capace di progettare classi, gestire la memoria e costruire un'applicazione completa che ricorda i suoi dati. Hai domato il linguaggio con la fama più intimidatoria che ci sia, e con esso hai capito come funziona davvero un computer: memoria, tipi, prestazioni. Questa comprensione ti renderà un programmatore migliore in ogni linguaggio che toccherai. La collana continua — col web, col backend — ma tu hai già fatto il salto più difficile. Ora vai, compila, e costruisci qualcosa di tuo. Buon codice! ⊕

Sezione 16.8Soluzioni

Soluzione 16.1

Le domande chiave: cosa deve fare (funzionalità)? Come rappresento i dati (quali classi, quali campi)? Come li tengo insieme (vector? map?)? Dove li salvo (file? formato?)? Come interagisce l'utente (menu, input)? Quali casi limite gestire? Rispondere prima rende il codice molto più diretto.

Soluzione 16.2

Per esempio: classe con costruttore (Cap. 10), std::vector (Cap. 8), file ifstream/ofstream/getline (Cap. 15), passaggio const& (Cap. 6), ciclo while + if/else if (Cap. 4-5), ternario e substr (Cap. 4, 9), controllo dei limiti (Cap. 8). Tutto il libro converge qui.

Soluzione 16.4
Attivita incapsulata
class Attivita {
private:
    std::string testo_;
    bool fatto_ = false;
public:
    Attivita(std::string t, bool f = false) : testo_(t), fatto_(f) {}
    void completa() { fatto_ = true; }
    bool e_fatta() const { return fatto_; }
    std::string get_testo() const { return testo_; }
};

Poi nel codice usi lista[n-1].completa() invece di lista[n-1].fatto = true, e a.e_fatta() / a.get_testo() per leggere. È il progetto «cresciuto» con incapsulamento pieno.
