Capitolo 8

Array e vector

I programmi veri lavorano su tanti valori insieme: i voti di una classe, i pixel di un'immagine, i punteggi di un gioco. Il C++ offre due modi per tenere sequenze di dati. Gli array «grezzi», eredità del linguaggio C: velocissimi e vicini alla memoria (sono, in fondo, dei puntatori — il capitolo scorso torna utile!), ma rigidi e pericolosi. E std::vector: l'array «intelligente» che cresce da solo, conosce la propria dimensione e ti protegge dagli errori più comuni. Nel C++ moderno la regola è semplice: usa std::vector — è la struttura dati che maneggerai ogni giorno. In questo capitolo li conosciamo entrambi, capendo perché il vector ha (quasi sempre) ragione.

🗺 Dove siamo

Gli array del C e i loro limiti (§8.1), std::vector e perché è meglio (§8.2), come aggiungere, accedere, scorrere (§8.3), il legame tra array e puntatori (§8.4), e le matrici (§8.5). Nel laboratorio (§8.6) calcoli statistiche su una lista di numeri.

🔑 Perché ci tocca: gestire tanti dati

Le sequenze sono ovunque. std::vector è probabilmente la struttura dati che userai di più in C++: flessibile, sicura, efficiente. Capire la differenza con gli array «grezzi» ti fa apprezzare cosa il vector fa per te (crescere, ricordare la dimensione, evitare accessi fuori limite), e ti fa capire perché tanto codice C legacy è pieno di trappole. È anche il tuo primo incontro con la STL, la ricchissima libreria di strutture e algoritmi del C++ (Cap. 13).

Sezione 8.1Gli array del C

L'array classico: una sequenza di elementi dello stesso tipo, contigui in memoria, di dimensione fissa. Semplice e velocissimo — ma con spigoli pericolosi.

Un array C dimensione fissa, indici da 0
esempio.cpp
int voti[4] = {7, 8, 6, 9};   // array di 4 int

std::cout << voti[0];      // 7  (indice da 0!)
voti[2] = 10;              // modifica il terzo elemento

// voti[7] = 1;   →  NESSUN controllo! Scrive fuori dall'array: DISASTRO
int voti[4] — dimensione fissa
dichiari il tipo, il nome e quanti elementi tra quadre. La dimensione è fissata alla creazione e non può cambiare: 4 elementi, per sempre. Accesso per indice (da 0), come le stringhe.
contigui in memoria
gli elementi vivono in caselle adiacenti (Cap. 7): voti[0], poi subito voti[1], ecc. È ciò che li rende velocissimi (e legati ai puntatori, §8.4).
la trappola mortale: nessun controllo di limite
l'array ha 4 elementi (indici 0-3), ma voti[7] non dà errore: scrive in memoria fuori dall'array, corrompendo altri dati o facendo crashare il programma. L'array grezzo si fida ciecamente di te. È la causa di innumerevoli bug e falle di sicurezza. Per questo esiste std::vector.

Sezione 8.2std::vector: l'array che cresce

Ecco la soluzione moderna. std::vector è un array che cresce e si accorcia da solo, sa quanti elementi contiene, e offre metodi comodi. È la struttura dati «di default» del C++.

std::vector l'array intelligente
esempio.cpp
#include <vector>          // serve questo include

std::vector<int> voti = {7, 8, 6};   // un vector di int

voti.push_back(9);       // AGGIUNGE in fondo → {7,8,6,9}  (cresce da solo!)
std::cout << voti.size();   // 4  (conosce la propria dimensione)
std::cout << voti[0];        // 7  (accesso per indice come l'array)
std::vector<int>
«un vector di int». Il tipo degli elementi va tra parentesi angolari < > (è un template, Cap. 13): vector<double>, vector<std::string>, ecc. Richiede #include <vector>.
.push_back(x) — cresce
aggiunge x in fondo, allargando il vector automaticamente. Non c'è una dimensione fissa: parti anche da vuoto e riempi. È ciò che l'array grezzo non sa fare.
.size() — quanti elementi
il vector ricorda la propria dimensione: niente più bisogno di portartela dietro in una variabile separata (fonte classica di bug con gli array).
std::vector: cresce, si accorcia, ricorda la size usa i metodi e guarda il vector

Sezione 8.3Aggiungere, accedere, scorrere

Le operazioni quotidiane sul vector: aggiungere e togliere elementi, accedere in sicurezza, e scorrerlo — spesso col comodo for range-based del Capitolo 5.

Le operazioni sul vector push · at · for range
esempio.cpp
std::vector<int> v = {7, 8, 6};

v.push_back(9);      // aggiunge in fondo
v.pop_back();        // toglie l'ultimo

std::cout << v[0];       // accesso rapido (ma NON controlla i limiti)
std::cout << v.at(0);    // accesso SICURO: .at() lancia un errore se fuori limite

for (int x : v) {        // scorrere col for range-based (Cap. 5)
    std::cout << x << " ";
}
v[i] vs v.at(i)
entrambi accedono all'elemento i. v[i] è veloce ma non controlla i limiti (come l'array grezzo); v.at(i) controlla e lancia un'eccezione se l'indice è fuori dai limiti. Usa .at() quando l'indice potrebbe sbagliare, per un crash controllato invece di un bug silenzioso.
scorrere
il for range-based (for (int x : v)) è il modo più pulito; se ti servono gli indici, usa il for classico da 0 a v.size(). Con int& x puoi anche modificare gli elementi.
altre operazioni utili
v.empty() (è vuoto?), v.clear() (svuota), v.front()/ v.back() (primo/ultimo). Il vector è ricchissimo.

Sezione 8.4Array e puntatori

Ecco dove il Capitolo 7 torna: in C++ (eredità del C) un array è, quasi, un puntatore al suo primo elemento. È un legame profondo che spiega molte cose — e molte trappole.

📐 Il nome dell'array è un indirizzo

Quando scrivi il nome di un array «nudo», in molti contesti esso «decade» nell'indirizzo del suo primo elemento — cioè si comporta come un puntatore. Per questo voti e &voti[0] sono di fatto lo stesso indirizzo, e *(voti + 1) è equivalente a voti[1] (l'indice è «spostati di tanto dall'inizio»). Gli elementi contigui + l'aritmetica dei puntatori spiegano perché l'accesso per indice è così veloce: è una semplice somma di indirizzi. Il rovescio: passando un array a una funzione, perdi la sua dimensione (arriva solo il puntatore al primo elemento), da cui i tanti bug del C. std::vector, invece, si porta dietro la sua dimensione: un altro motivo per preferirlo. Non devi padroneggiare l'aritmetica dei puntatori ora — ti basta l'idea: array e puntatori sono parenti stretti.

Sezione 8.5Matrici (vector di vector)

Per dati a due dimensioni (una griglia, una tabella, una scacchiera) servono strutture bidimensionali. La via moderna: un vector di vector.

Una matrice vector di vector
esempio.cpp
std::vector<std::vector<int>> griglia = {
    {1, 2, 3},
    {4, 5, 6}
};

std::cout << griglia[1][2];   // 6  (riga 1, colonna 2)

for (auto& riga : griglia) {      // scorri le righe...
    for (int x : riga) {           // ...e le colonne (cicli annidati, Cap. 5)
        std::cout << x << " ";
    }
    std::cout << "\n";
}
Un vector<vector<int>> è «una lista di righe, ognuna una lista di numeri»: l'accesso è griglia[riga][colonna]. La sintassi dei template annidati (<< >>) spaventa all'inizio, ma l'idea è semplice — è la stessa struttura «lista di liste» dei linguaggi ad alto livello. Per scorrerla, cicli annidati (Cap. 5). Nota auto& nel ciclo esterno: il & evita di copiare ogni riga (che è un vector intero!) — efficienza, come nel Cap. 6.

Sezione 8.6Laboratorio: statistiche su una lista

Uniamo tutto: leggiamo dei numeri in un vector e calcoliamo somma, media e massimo — scorrendo con un ciclo e accumulando. Un classico che mostra il vector al lavoro.

Il programma stat.cpp vector + ciclo + accumulatori
stat.cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> voti = {7, 8, 6, 10, 9};

    int somma = 0;
    int massimo = voti[0];
    for (int v : voti) {
        somma += v;                       // accumulo la somma
        if (v > massimo) massimo = v;    // tengo il più grande
    }

    double media = static_cast<double>(somma) / voti.size();   // cast! (Cap. 3)
    std::cout << "Media: " << media << ", massimo: " << massimo << std::endl;
    return 0;
}
Output: «Media: 8, massimo: 10». Vedi convergere tante idee: il vector tiene i dati e conosce la sua .size(), il for range-based li scorre, gli accumulatori sommano e cercano il max, e il cast a double (Cap. 3!) evita la divisione intera nella media. Nota static_cast<double>(somma): senza, 40 / 5 sarebbe intero e con numeri diversi perderebbe i decimali. Compila con g++ stat.cpp -o stat. (Bonus: la STL, Cap. 13, ha std::accumulate e std::max_element che fanno questo in una riga — ma capire lo schema a mano è essenziale.)
🔑 Quello che devi portarti dietro dal Capitolo 8
  1. L'array C (int a[4]) è a dimensione fissa, contiguo e velocissimo, ma non controlla i limiti (bug pericolosi) (§8.1).
  2. std::vector (#include <vector>) è l'array «intelligente»: cresce (push_back), conosce la .size(), offre l'accesso sicuro .at(). È la scelta di default (§8.2-8.3).
  3. Un array «decade» in un puntatore al primo elemento (Cap. 7): parenti stretti; ma passandolo a una funzione perdi la dimensione — il vector no (§8.4).
  4. Le matrici = vector<vector<T>>, accesso [riga][col], scorse con cicli annidati (§8.5).

Sai gestire sequenze di numeri. Ma i programmi lavorano tantissimo con il testo: nomi, messaggi, contenuti. In C++ il testo ha una storia doppia — le vecchie stringhe «grezze» ereditate dal C (array di char, spinosi) e la moderna std::string, comoda e sicura. Nel Capitolo 9 ci dedichiamo alle stringhe: std::string, concatenazione, sottostringhe, i metodi utili, e come leggere righe intere. Vedrai che, con le classi giuste (come vector e string), il C++ moderno è molto più amichevole della sua fama.

Sezione 8.8Soluzioni

Soluzione 8.1

L'array C ha dimensione fissa e non controlla i limiti; il std::vector cresce e si accorcia dinamicamente (push_back/pop_back), conosce la propria dimensione (.size()), offre accesso sicuro (.at()) e si porta dietro i dati passandolo alle funzioni. Vantaggi del vector: flessibilità, sicurezza, comodità.

Soluzione 8.2

L'array di 4 elementi ha indici validi 0-3; voti[7] scrive/legge fuori dall'array, in memoria che non gli appartiene, senza alcun errore: corrompe altri dati o fa crashare (bug insidioso). Il vector con .at(7), invece, rileva l'accesso fuori limite e lancia un'eccezione (crash controllato e chiaro) invece di corrompere la memoria.

Soluzione 8.3
spesa.cpp
std::vector<std::string> spesa = {"pane", "latte"};
spesa.push_back("uova");
std::cout << spesa.size();   // 3
Soluzione 8.4
conta.cpp
std::vector<int> v = {3, 4, 7, 8, 10};
int pari = 0;
for (int x : v) {
    if (x % 2 == 0) pari++;
}
std::cout << pari << " numeri pari\n";   // 3
