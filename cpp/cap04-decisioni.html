Capitolo 4

Prendere decisioni

È il momento di dare un cervello ai tuoi programmi: la capacità di scegliere in base alle condizioni. I confronti del Capitolo 3, che producono true/false, diventano qui dei bivi che cambiano il corso dell'esecuzione. Il C++ ha gli strumenti classici — if, else if, else — più il costrutto switch per quando i casi sono molti, e l'elegante operatore ternario. E c'è una differenza di forma rispetto a linguaggi come Python: qui i blocchi si segnano con le graffe { }, non con l'indentazione. Con le decisioni, i tuoi programmi cominciano a reagire in modo intelligente al mondo.

🗺 Dove siamo

Il costrutto if / else if / else (§4.1), il ruolo delle graffe e dei blocchi (§4.2), lo switch per molti casi (§4.3) e l'operatore ternario (§4.4). Nel laboratorio (§4.5) costruisci un classificatore.

🔑 Perché ci tocca: la logica di ogni programma

Le decisioni sono ovunque: validare un input, controllare un permesso, reagire a una scelta, gestire i casi limite. Senza if, un programma esegue sempre la stessa sequenza fissa; con if, si adatta. Insieme ai cicli (Cap. 5), le decisioni formano il «controllo di flusso», il cuore della programmazione. I concetti sono gli stessi di ogni linguaggio; qui impari la sintassi precisa del C++ e le sue piccole trappole (le graffe, l'= per errore).

Sezione 4.1if / else if / else

Il costrutto fondamentale: if esegue un blocco se la condizione è vera, else if aggiunge casi intermedi, else copre tutto il resto.

if / else if / else condizioni in cascata
voto.cpp
int voto = 7;

if (voto >= 9) {
    std::cout << "Ottimo!";
} else if (voto >= 6) {
    std::cout << "Promosso.";
} else {
    std::cout << "Bocciato.";
}
if (condizione) { … }
la condizione va tra parentesi tonde ( ) (obbligatorie in C++!), e il blocco tra graffe { }. Se la condizione è vera, esegue il blocco.
else if ed else
else if aggiunge un altro controllo; else (senza condizione) cattura «tutti gli altri casi». Python usa elif; in C++ è else if (due parole).
si ferma alla prima vera
come in ogni linguaggio: Python controlla dall'alto, esegue il primo blocco vero e salta il resto. L'ordine conta: metti i controlli dal più specifico al più generale.
if / else if / else: quale ramo? cambia il voto

Sezione 4.2Le graffe e i blocchi

In C++ un blocco di codice si delimita con le graffe { }. A differenza di Python, l'indentazione è solo estetica: conta ciò che sta tra le graffe. Ma alcune sottigliezze meritano attenzione.

⚠ Due trappole con le graffe

1. Graffe «opzionali» ingannevoli. Se il blocco ha una sola istruzione, le graffe si possono omettere: if (x > 0) std::cout << "pos";. Ma è una fonte di bug: se poi aggiungi una seconda riga pensando sia «dentro» l'if, in realtà è fuori! Consiglio: metti sempre le graffe, anche per una riga sola — più sicuro e più chiaro.

2. Il temuto = al posto di ==. Scrivere if (x = 5) (assegnazione) invece di if (x == 5) (confronto) compila in C++: assegna 5 a x, e la condizione risulta «vera» (5 è diverso da zero)! Un bug classico e insidioso. Il compilatore spesso ti avvisa («suggested parentheses») — non ignorarlo. Alcuni scrivono if (5 == x) apposta: così un = per errore darebbe errore di compilazione.

💡 L'indentazione resta importante (per gli umani)

In C++ l'indentazione non ha significato per il compilatore, ma è fondamentale per chi legge. Un codice indentato bene si capisce a colpo d'occhio; uno indentato a caso, pur compilando, è un incubo. Indenta sempre con cura — è cortesia verso te stesso e verso chi leggerà il tuo codice. Gli editor moderni lo fanno quasi da soli (e c'è chi usa strumenti di formattazione automatica come clang-format).

Sezione 4.3switch

Quando devi scegliere tra molti valori di una stessa variabile (un menu, un codice), una catena di else if diventa pesante. Il costrutto switch è più pulito per questi casi.

Il costrutto switch un valore, molti casi
menu.cpp
int scelta = 2;

switch (scelta) {
    case 1:
        std::cout << "Nuova partita";
        break;                 // IMPORTANTE: esce dallo switch
    case 2:
        std::cout << "Carica partita";
        break;
    default:                     // "tutti gli altri casi" (come else)
        std::cout << "Scelta non valida";
}
switch (variabile) { case … }
confronta la variabile con ogni case: quando trova quello giusto, esegue da lì. Funziona con interi e caratteri (non con testi o decimali).
il break; è cruciale
senza break, l'esecuzione «cade» nel caso successivo e continua (fall-through)! È il bug più comune dello switch: ricordati un break; alla fine di ogni case. (A volte il fall-through si usa apposta, ma di norma vuoi il break.)
default:
il caso «tutti gli altri», come l'else. Opzionale ma consigliato, per gestire l'imprevisto.
Quando usare switch e quando if? Lo switch brilla quando confronti una variabile con molti valori fissi (un menu numerico, un codice): è più leggibile di tanti else if. L'if/else if è più flessibile: gestisce condizioni complesse, intervalli (voto >= 6), combinazioni logiche. Menu e codici → switch; condizioni e intervalli → if.

Sezione 4.4L'operatore ternario

Per le decisioni semplici che scelgono tra due valori, esiste una scorciatoia elegante: l'operatore ternario ? :. Una decisione in una riga.

L'operatore ternario condizione ? seVero : seFalso
esempio.cpp
int eta = 20;

// forma lunga con if/else:
std::string stato;
if (eta >= 18) stato = "adulto";
else            stato = "minore";

// la stessa cosa col ternario, in una riga:
std::string stato2 = (eta >= 18) ? "adulto" : "minore";
Si legge: «se eta >= 18 allora "adulto" altrimenti "minore"». La struttura è condizione ? valore_se_vero : valore_se_falso. È perfetto quando devi scegliere un valore tra due (assegnare, stampare), ma non abusarne: se la logica è complessa o annidata, un normale if/else è molto più leggibile. Come sempre, l'eleganza serve la chiarezza.

Sezione 4.5Laboratorio: un classificatore

Costruiamo un programma che legge un voto numerico e restituisce un giudizio in lettere — mettendo insieme cin, confronti e if/else if/else.

Il programma giudizio.cpp leggi · classifica · stampa
giudizio.cpp
#include <iostream>

int main() {
    int voto;
    std::cout << "Voto (0-10): ";
    std::cin >> voto;

    if (voto < 0 || voto > 10) {   // controllo di validità (OR logico)
        std::cout << "Voto non valido.";
    } else if (voto >= 9) {
        std::cout << "Eccellente";
    } else if (voto >= 6) {
        std::cout << "Sufficiente";
    } else {
        std::cout << "Insufficiente";
    }
    std::cout << std::endl;
    return 0;
}
Nota la prima condizione: voto < 0 || voto > 10 usa l'OR logico per validare l'input prima di classificarlo — un'ottima abitudine (controllare i casi assurdi per primi). Poi la cascata classifica per intervalli. L'ordine è essenziale: se mettessimo voto >= 6 prima di voto >= 9, anche un 10 finirebbe in «Sufficiente». Compila con g++ giudizio.cpp -o giudizio e prova voti diversi, inclusi quelli fuori scala.
🔑 Quello che devi portarti dietro dal Capitolo 4
  1. if (cond) { … } else if (cond) { … } else { … }: condizione tra tonde, blocco tra graffe; si ferma al primo vero, l'ordine conta (§4.1).
  2. Le graffe delimitano i blocchi (non l'indentazione). Mettile sempre, e attento a = vs == nell'if (§4.2).
  3. switch per confrontare una variabile con molti valori fissi: ricorda il break; in ogni case e il default (§4.3).
  4. L'operatore ternario cond ? a : b sceglie un valore tra due, in una riga — per casi semplici (§4.4).

I tuoi programmi ora scelgono. Ma sanno fare le cose una volta sola: per elaborare tanti dati — i voti di una classe, le righe di un file — copiare il codice sarebbe assurdo. Serve ripetere. Nel Capitolo 5 arrivano i cicli del C++: while e do-while, il potente for classico (con la sua terna inizializzazione-condizione-incremento), e il moderno for range-based per scorrere le collezioni con eleganza. Uniti alle decisioni, i cicli sbloccano il lavoro vero su grandi quantità di dati.

Sezione 4.7Soluzioni

Soluzione 4.1
meteo.cpp
if (temp > 25) {
    std::cout << "Caldo";
} else {
    std::cout << "Fresco";
}
Soluzione 4.2

if (x = 5) usa un solo = (assegnazione): assegna 5 a x, e poiché 5 è diverso da zero la condizione risulta sempre vera. Non è un errore di sintassi, quindi compila e il bug passa inosservato. Volevi == (confronto). Lo eviti prestando attenzione ai warning del compilatore, o scrivendo if (5 == x) (così un = darebbe errore).

Soluzione 4.3

Senza break;, l'esecuzione «cade» nel case successivo e continua a eseguirlo (fall-through): di solito un bug. Lo switch conviene quando confronti una variabile con molti valori fissi (un menu, un codice): è più leggibile di tanti else if. Per condizioni complesse o intervalli, meglio if/else if.

Soluzione 4.4

std::string parita = (n % 2 == 0) ? "pari" : "dispari"; — «se n è pari, "pari", altrimenti "dispari"», in una riga.
