Capitolo 13

Template e la STL

Hai già usato std::vector<int> e vector<string>: lo stesso vector, diversi tipi di contenuto. Come? Grazie ai template — il meccanismo con cui si scrive codice generico, che funziona per qualsiasi tipo, scritto una volta sola. E i template sono la base della STL (Standard Template Library): la straordinaria collezione di contenitori (vector, map, set) e algoritmi (sort, find, count…) già pronti del C++. La STL è la parte più produttiva del linguaggio: ti fa fare in una riga ciò che a mano richiederebbe pagine, con codice testato ed efficiente. In questo capitolo scopri come scrivere template tuoi e, soprattutto, come sfruttare l'immenso arsenale che il C++ ti regala.

🗺 Dove siamo

Il codice generico coi template (§13.1), i contenitori della STL (§13.2), map e set (§13.3), gli algoritmi (§13.4), iteratori e lambda (§13.5). Nel laboratorio (§13.6) risolvi un problema con la STL.

🔑 Perché ci tocca: non reinventare la ruota

La STL è il motivo per cui il C++ moderno è produttivo: quasi ogni struttura dati e algoritmo che ti serve è già lì, testato e velocissimo. Ordinare, cercare, contare, associare chiavi a valori — tutto pronto. Saper usare la STL ti fa scrivere meno codice, più corretto e più efficiente. E i template, che la rendono possibile, sono un'idea potente che incontrerai ovunque nel codice C++ serio. È il capitolo che ti fa passare da «so la sintassi» a «so essere produttivo».

Sezione 13.1Codice generico coi template

Immagina di voler scrivere una funzione «massimo tra due valori». La scriveresti per int, poi di nuovo per double, poi per string… I template ti fanno scriverla una volta, per qualsiasi tipo.

Una funzione template template <typename T>
esempio.cpp
template <typename T>          // T è un tipo "generico", deciso all'uso
T massimo(T a, T b) {
    return (a > b) ? a : b;   // funziona per ogni tipo confrontabile con >
}

std::cout << massimo(3, 7);          // 7   (T = int)
std::cout << massimo(2.5, 1.8);      // 2.5 (T = double)
std::cout << massimo(std::string("a"), std::string("z"));  // z   (T = string!)
template <typename T>
dichiara un parametro di tipo T: un «tipo segnaposto» che verrà deciso quando usi la funzione. Nel corpo, usi T come se fosse un tipo qualsiasi.
il compilatore «genera» il codice
quando chiami massimo(3, 7), il compilatore crea al volo la versione con T = int; con massimo(2.5, 1.8), quella con T = double. Scrivi una funzione, ne ottieni infinite, una per ogni tipo che usi. Ed è tutto risolto in compilazione: nessun costo a runtime.
ecco com'è fatto vector
std::vector<T> è una classe template: lo stesso codice funziona per vector<int>, vector<string>, ecc. Ora sai il meccanismo dietro le strutture che usi.
Un template, tanti tipi chiama massimo con tipi diversi

Sezione 13.2I contenitori della STL

La STL offre molte strutture dati pronte, ognuna adatta a un uso. Conosci già vector; ecco le altre che userai di più.

I contenitori principali ognuno per il suo scopo
std::vector<T> — sequenza dinamica
l'array che cresce (Cap. 8): il tuttofare. Accesso rapido per indice, aggiunta in fondo. La scelta di default per una lista di elementi.
std::map<K, V> — chiave → valore
associa chiavi a valori (come un dizionario): map<string, int> per nome→età. Ricerca per chiave velocissima (§13.3).
std::set<T> — insieme senza duplicati
una collezione di elementi unici, ordinati automaticamente. Per «quali elementi ci sono», test di appartenenza rapidi (§13.3).
altri utili
std::array (array a dimensione fissa ma sicuro), std::pair (una coppia di valori), std::unordered_map (come map ma ancora più veloce, senza ordine). Ce n'è per ogni esigenza.
La bellezza: tutti condividono un'interfaccia coerente (.size(), .empty(), il for range-based, gli iteratori) — impari uno, sai muoverti in tutti. E tutti gestiscono la memoria per te (RAII, Cap. 12): nessun new/delete. Scegliere il contenitore giusto per il problema è metà del lavoro di un buon programmatore C++.

Sezione 13.3map e set

Due contenitori così utili da meritare un focus. La map associa chiavi a valori (il «dizionario» del C++); il set tiene elementi unici.

std::map chiave → valore
esempio.cpp
#include <map>

std::map<std::string, int> eta;   // mappa nome → età
eta["Anna"] = 25;               // inserisci/aggiorna con la chiave
eta["Marco"] = 30;

std::cout << eta["Anna"];        // 25   (accesso per chiave)

for (auto& coppia : eta) {        // scorri le coppie
    std::cout << coppia.first << ": " << coppia.second << "\n";
}
map<string, int>
due tipi tra angolari: il tipo della chiave e quello del valore. Accesso con eta["Anna"], come un dizionario Python. Le chiavi sono uniche e ordinate.
.first e .second
scorrendo una map ottieni coppie (std::pair): .first è la chiave, .second il valore. (Con C++ moderno puoi anche fare for (auto& [nome, anni] : eta) — lo «structured binding», più leggibile.)
il set
std::set<int> s = {3, 1, 3, 2}; tiene solo {1, 2, 3} (niente duplicati, ordinati). s.count(2) dice se c'è. Per raccolte di elementi unici.

Sezione 13.4Gli algoritmi (sort, find…)

Il vero superpotere della STL: decine di algoritmi pronti che operano sui contenitori. Ordinare, cercare, contare, sommare — tutto in una riga, testato ed efficiente.

Algoritmi comuni #include <algorithm>
esempio.cpp
#include <algorithm>
#include <vector>

std::vector<int> v = {7, 2, 9, 4};

std::sort(v.begin(), v.end());         // ordina: {2, 4, 7, 9}
int m = *std::max_element(v.begin(), v.end());   // 9
bool c = std::count(v.begin(), v.end(), 4);       // quante volte c'è il 4
std::sort(v.begin(), v.end())
ordina il contenitore (in ordine crescente di default). v.begin() e v.end() indicano «dall'inizio alla fine» (sono iteratori, §13.5). Un solo comando al posto di un intero algoritmo di ordinamento scritto a mano!
altri utilissimi
std::find (cerca un valore), std::count (conta occorrenze), std::max_element/min_element, std::accumulate (somma, da <numeric>), std::reverse, std::unique… decine di algoritmi.
ordinare a modo tuo
std::sort(v.begin(), v.end(), criterio) accetta un criterio (spesso una lambda, §13.5) per ordinare come vuoi — es. decrescente, o per un campo di una struct.
La filosofia della STL: contenitori + iteratori + algoritmi. Gli algoritmi non sono legati a un contenitore specifico: lavorano su «un intervallo» (da begin a end), quindi lo stesso std::sort ordina un vector, un array e molto altro. È un design elegante e potente. La regola del bravo programmatore C++: prima di scrivere un ciclo, chiediti «esiste un algoritmo STL?» — quasi sempre sì.

Sezione 13.5Iteratori e lambda

Due concetti che rendono la STL flessibile: gli iteratori (il modo uniforme di «puntare» agli elementi) e le lambda (funzioni «al volo» da passare agli algoritmi).

📐 Iteratori: puntatori «generalizzati»

Un iteratore è un oggetto che «indica» un elemento di un contenitore e sa avanzare al successivo — come un puntatore generalizzato. v.begin() indica il primo elemento, v.end() la posizione dopo l'ultimo. Gli algoritmi lavorano su intervalli di iteratori (begin, end), ed è questo che li rende universali: lo stesso codice funziona su qualunque contenitore che fornisca iteratori. Il for range-based (Cap. 5) è, sotto, zucchero sintattico sugli iteratori: non devi maneggiarli a mano spesso, ma capirli spiega come funziona la STL.

📐 Lambda: funzioni senza nome, al volo
esempio.cpp
std::vector<int> v = {5, 2, 8, 1};

// ordina in modo DECRESCENTE usando una lambda come criterio:
std::sort(v.begin(), v.end(), [](int a, int b) {
    return a > b;   // "a viene prima di b se è maggiore" → decrescente
});   // v ora è {8, 5, 2, 1}

Una lambda è una funzione anonima definita sul posto: [](int a, int b) { return a > b; }. Le quadre [] (la «capture list») aprono la lambda, poi ci sono parametri e corpo come una normale funzione. Sono comodissime per passare un «pezzo di logica» a un algoritmo senza scrivere una funzione separata: ordina così, filtra cosà. È l'equivalente C++ delle lambda di Python. Non serve padroneggiarle subito; sappi riconoscerle, perché la STL le usa tantissimo.

Sezione 13.6Laboratorio: usare la STL

Risolviamo un problema reale — contare le parole in un testo — sfruttando la STL: una map per i conteggi. Vedrai quanto codice ti risparmia.

Il programma conta_parole.cpp map al lavoro
conta_parole.cpp
#include <iostream>
#include <map>
#include <string>

int main() {
    std::map<std::string, int> conteggio;
    std::string parola;

    // legge parole finché ci sono (Ctrl+D / Ctrl+Z per finire)
    while (std::cin >> parola) {
        conteggio[parola]++;   // map: se la chiave non c'è, la crea a 0, poi incrementa
    }

    for (auto& [p, n] : conteggio) {   // structured binding (C++ moderno)
        std::cout << p << ": " << n << "\n";
    }
    return 0;
}
La riga magica è conteggio[parola]++: se la parola non è ancora nella map, [] la crea automaticamente con valore 0, poi ++ la porta a 1; se c'è già, la incrementa. Contare occorrenze — che a mano richiederebbe un ciclo di ricerca — diventa una riga, grazie alla map. E le chiavi escono già ordinate alfabeticamente (proprietà della map). Compila con g++ -std=c++17 conta_parole.cpp -o conta (il -std=c++17 abilita lo structured binding), poi digita del testo e finisci con Ctrl+Z (Windows) o Ctrl+D (Mac/Linux). Questa è la STL: risolvi problemi veri con pochissimo codice.
🔑 Quello che devi portarti dietro dal Capitolo 13
  1. I template (template <typename T>) scrivono codice generico, valido per ogni tipo, risolto in compilazione. È così che nasce vector<T> (§13.1).
  2. Contenitori STL: vector (sequenza), map (chiave→valore), set (unici); interfaccia coerente e memoria gestita (RAII) (§13.2-13.3).
  3. Gli algoritmi (sort, find, count, max_element…) operano su intervalli (begin, end): una riga al posto di un ciclo scritto a mano (§13.4).
  4. Iteratori = puntatori generalizzati agli elementi; lambda = funzioni al volo per dare un criterio agli algoritmi (§13.5). Regola: prima di scrivere un ciclo, cerca l'algoritmo STL (§13.6).

Sei ormai un programmatore C++ capace: sintassi, OOP, memoria, e la potenza della STL. Restano due tasselli pratici prima del gran finale. I programmi vanno a male: input sbagliati, file mancanti, situazioni impreviste. Nel Capitolo 14 vediamo la gestione degli errori in C++ — le eccezioni (try/catch/throw) — e quando conviene usarle. È il meccanismo che, insieme a RAII (Cap. 12!), permette al C++ di gestire gli imprevisti in modo pulito e sicuro. Poi, file e I/O, e infine un progetto completo che riunisce tutto.

Sezione 13.8Soluzioni

Soluzione 13.1

Un template permette di scrivere codice generico, valido per qualsiasi tipo, una volta sola. Il compilatore, alle due chiamate, genera due versioni della funzione: una con T = int (per massimo(3, 7)) e una con T = double (per massimo(2.5, 1.8)). Tutto in compilazione, senza costi a runtime.

Soluzione 13.2

(a) std::vector<int> (sequenza, ordinabile con sort); (b) std::map<std::string, std::string> (chiave→valore); (c) std::set<std::string> (elementi unici).

Soluzione 13.3

Ordinare: std::sort(v.begin(), v.end());. Massimo: *std::max_element(v.begin(), v.end()); (l'asterisco perché max_element restituisce un iteratore all'elemento, da dereferenziare). Entrambi richiedono #include <algorithm>.

Soluzione 13.4

Perché conteggio[parola] su una map, con una chiave non presente, la crea automaticamente con valore 0; quindi ++ la porta a 1 (prima occorrenza). Se la chiave c'è già, ++ incrementa il conteggio. Così, parola dopo parola, la map accumula quante volte compare ciascuna — in una sola riga.
