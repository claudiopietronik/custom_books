Capitolo 9

Le stringhe

Il testo è dappertutto — nomi, messaggi, comandi, contenuti. In C++ le stringhe hanno una doppia natura: le vecchie stringhe «grezze» ereditate dal C (array di char, spinose e insidiose) e la moderna std::string, comoda, sicura e ricca di metodi. La lezione di questo capitolo è la stessa del precedente con std::vector: usa std::string e il testo in C++ diventa quasi piacevole come nei linguaggi ad alto livello. Impariamo a concatenare, confrontare, estrarre sottostringhe, usare i metodi utili, e — dettaglio pratico importante — a leggere righe intere con gli spazi. Con vector e string, il C++ moderno mostra il suo volto amichevole.

🗺 Dove siamo

std::string vs char* (§9.1), concatenare e confrontare (§9.2), indici, lunghezza, sottostringhe (§9.3), i metodi utili (§9.4), e leggere righe intere con getline (§9.5). Nel laboratorio (§9.6) analizzi del testo.

🔑 Perché ci tocca: il testo, senza dolore

Ogni programma tocca del testo, e in C++ farlo «bene» (con std::string) invece che «alla vecchia» (con char*) è la differenza tra codice sicuro e codice pieno di trappole. Le vecchie stringhe C sono la fonte storica di innumerevoli falle di sicurezza (buffer overflow); std::string le elimina gestendo la memoria per te. Sapere che char* esiste ti serve per leggere codice legacy, ma per il tuo codice: std::string, sempre.

Sezione 9.1std::string vs char*

Due modi di rappresentare il testo, uno moderno e uno storico. Capire la differenza spiega perché il C++ ha fama di essere «difficile col testo» — e perché oggi non deve esserlo.

📐 Il vecchio e il nuovo
  • La stringa C «grezza»: un array di char che finisce con un carattere speciale «zero» ('\0'). Il nome della variabile è, di fatto, un puntatore al primo carattere (char*). Nessuna gestione della lunghezza, nessun controllo: facilissimo sforare la memoria (il famigerato buffer overflow). Spinosa e pericolosa.
  • std::string (moderna): un oggetto che gestisce il testo per te — memoria, lunghezza, crescita — e offre decine di metodi comodi. Si comporta come le stringhe di Python: la concateni con +, la confronti con ==, ne chiedi la lunghezza. Richiede #include <string>.
Creare una std::string un oggetto, non un array
esempio.cpp
#include <string>

std::string nome = "Anna";      // una std::string, non un array di char
std::cout << nome.length();   // 4  (conosce la sua lunghezza!)
Nota il doppio vs singolo apice, come dal Cap. 2: "Anna" (virgolette) è testo (usato per inizializzare la string); 'A' (apici) è un singolo char. Una std::string è una sequenza di char, gestita in modo sicuro.

Sezione 9.2Concatenare e confrontare

Con std::string, unire e confrontare testo è intuitivo — proprio come ci si aspetterebbe, e come nei linguaggi ad alto livello.

+ unisce, == confronta testo naturale
esempio.cpp
std::string nome = "Anna";
std::string saluto = "Ciao " + nome + "!";   // concatenazione con +
std::cout << saluto;      // Ciao Anna!

if (nome == "Anna") {      // confronto diretto con ==  (comodo!)
    std::cout << "Ciao Anna";
}
saluto += " Benvenuta";   // += aggiunge in fondo
Bello e naturale: + concatena, += aggiunge, == e != confrontano il contenuto. Con le vecchie char* nulla di tutto ciò funziona: == confronterebbe gli indirizzi, non il testo, e concatenare richiederebbe funzioni apposite e attenzione alla memoria. Ecco, in una riga, perché std::string ha vinto. Puoi anche confrontare con </> (ordine alfabetico) — utile per ordinare.

Sezione 9.3Indici, lunghezza, sottostringhe

Una stringa è una sequenza di caratteri: puoi accedere a ogni carattere per indice (da 0), chiederne la lunghezza ed estrarne pezzi.

Accedere e sezionare [] · length · substr
esempio.cpp
std::string s = "Programmare";

std::cout << s[0];         // 'P'  (indice da 0, come stringhe/vector)
std::cout << s.length();  // 11  (numero di caratteri)
std::cout << s.substr(0, 4);  // "Prog"  (da indice 0, lunghezza 4)
std::cout << s.substr(3);     // "grammare"  (da indice 3 alla fine)
s[i] — un carattere
accede al carattere all'indice i (un char). Da 0, come vector e come le stringhe di ogni linguaggio.
s.length() (o s.size())
il numero di caratteri. I due nomi sono equivalenti.
s.substr(inizio, lunghezza)
estrae una sottostringa: da inizio, per lunghezza caratteri. Omettendo la lunghezza, va fino alla fine. Nota: in C++ il secondo argomento è la lunghezza, non l'indice finale (attenzione se vieni da Python, dove lo slicing usa l'indice finale).

Sezione 9.4Metodi utili

std::string ha molti metodi pronti per le operazioni comuni sul testo. Ecco quelli che userai più spesso.

Metodi comuni find, replace, empty…
s.find("gram")
trova la posizione della sottostringa (l'indice dove inizia), o un valore speciale (std::string::npos) se non c'è. Per «contiene?»: if (s.find("x") != std::string::npos).
s.empty()
true se la stringa è vuota (""). Più chiaro di s.length() == 0.
s.push_back('!') e +=
aggiungono un carattere/testo in fondo (la stringa cresce, come un vector).
std::to_string(42) e std::stoi("42")
convertono numero → stringa (to_string) e stringa → intero (stoi). Utilissimi per costruire messaggi o leggere numeri da testo.
Nota culturale: std::string non ha un metodo .upper() «pronto» come Python — per trasformare in maiuscolo si usa un piccolo ciclo con std::toupper, o algoritmi della STL (Cap. 13). Il C++ ti dà i mattoni, a volte devi comporli tu: un filo meno «batterie incluse» di Python sul testo, ma con std::string il grosso è comodo e sicuro.

Sezione 9.5Leggere righe intere

Un dettaglio pratico che confonde molti principianti: std::cin >> s legge una sola parola (si ferma al primo spazio). Per leggere un'intera riga con gli spazi serve std::getline.

cin >> vs getline una parola vs una riga
esempio.cpp
std::string riga;

// cin >> riga;   →  se scrivi "Mario Rossi", legge solo "Mario"!

std::cout << "Nome completo: ";
std::getline(std::cin, riga);   // legge TUTTA la riga, spazi inclusi
std::cout << "Ciao " << riga;   // "Ciao Mario Rossi"
std::cin >> s
legge una parola: si ferma al primo spazio o a-capo. Va bene per un nome singolo, un numero — ma tronca le frasi con spazi.
std::getline(std::cin, s)
legge tutta la riga fino all'invio, spazi compresi, e la mette in s. È ciò che serve per nomi completi, frasi, indirizzi.
la trappola: mischiare i due
usare cin >> e poi getline nella stessa esecuzione crea un problema classico (il >> lascia un a-capo «in sospeso» che getline raccoglie come riga vuota). Se ti capita, sappi che serve «pulire» il buffer — un dettaglio noto, cercabile all'occorrenza.

Sezione 9.6Laboratorio: analizzare del testo

Costruiamo un programma che legge una frase intera e ne ricava informazioni: lunghezza, prima e ultima lettera, e se contiene una certa parola. Mette insieme getline, indici e find.

Il programma analizza.cpp getline + metodi
analizza.cpp
#include <iostream>
#include <string>

int main() {
    std::string frase;
    std::cout << "Scrivi una frase: ";
    std::getline(std::cin, frase);

    std::cout << "Lunghezza: " << frase.length() << "\n";
    if (!frase.empty()) {
        std::cout << "Prima lettera: " << frase[0] << "\n";
        std::cout << "Ultima lettera: " << frase[frase.length() - 1] << "\n";
    }
    if (frase.find("C++") != std::string::npos) {
        std::cout << "Parli di C++! 🎉\n";
    }
    return 0;
}
Nota frase[frase.length() - 1] per l'ultima lettera: gli indici vanno da 0 a length-1, quindi l'ultimo è a length() - 1 (in C++ non c'è l'indice negativo [-1] di Python). E find("C++") != std::string::npos è l'idioma per «contiene»: npos è il valore «non trovato». Compila con g++ analizza.cpp -o analizza e prova frasi diverse. Vedi convergere getline (leggere la riga), i metodi (length, empty, find) e gli indici — il testo, gestito con comodità.
🔑 Quello che devi portarti dietro dal Capitolo 9
  1. Usa std::string (#include <string>), non le char* grezze del C: gestisce memoria e lunghezza, ed è sicura (§9.1).
  2. + concatena, += aggiunge, ==/!= confrontano il contenuto (§9.2). s[i], s.length(), s.substr(inizio, lunghezza) (la lunghezza, non l'indice finale!) (§9.3).
  3. Metodi: find (+ npos per «contiene»), empty; conversioni to_string/stoi (§9.4).
  4. cin >> s legge una parola; std::getline(std::cin, s) legge tutta la riga con gli spazi (§9.5).

Con vector e string, hai visto il lato «comodo» del C++ moderno — e non è un caso: entrambi sono classi della libreria standard, oggetti che uniscono dati e metodi. È ora di imparare a costruire i tuoi tipi così. Nel Capitolo 10 arriva la programmazione a oggetti del C++: le struct (dati insieme) e le class (dati + metodi), l'incapsulamento con public e private, i costruttori che preparano gli oggetti e i distruttori che li puliscono. È il passo che ti fa passare da «usare le classi degli altri» a «progettare le tue» — il cuore del C++ come linguaggio a oggetti.

Sezione 9.8Soluzioni

Soluzione 9.1

Perché std::string gestisce la memoria e la lunghezza automaticamente (niente buffer overflow), permette operazioni naturali (+, ==, metodi comodi) ed è sicura. Le char* grezze richiedono di gestire manualmente memoria e terminatore '\0', non si confrontano/concatenano con gli operatori normali, e sono la causa storica di molte falle di sicurezza.

Soluzione 9.2

s.length() → 10 (10 caratteri, spazio incluso); s[0] → 'C'; s.substr(5, 5) → "mondo" (da indice 5, per 5 caratteri).

Soluzione 9.3

Perché cin >> nome legge solo una parola (si ferma al primo spazio): otterresti solo «Mario». Per leggere l'intera riga con lo spazio si usa std::getline(std::cin, nome).

Soluzione 9.4
email.cpp
if (email.find('@') != std::string::npos) {
    std::cout << "valida";
} else {
    std::cout << "manca la @";
}
