Capitolo 15

File e I/O

Tutto ciò che un programma elabora svanisce alla chiusura, a meno di salvarlo su disco. Dare ai programmi una memoria che dura è il compito dell'input/output su file. E c'è una bella sorpresa: in C++, leggere e scrivere file usa la stessa idea che conosci fin dal Capitolo 1 — gli stream. std::cout è uno stream verso lo schermo, std::cin uno stream dalla tastiera; un file è semplicemente un altro stream. Impari ofstream per scrivere e ifstream per leggere, con gli stessi operatori << e >> di sempre — e grazie a RAII (Cap. 12) i file si chiudono da soli. È l'ultimo strumento pratico prima di mettere tutto insieme nel progetto finale.

🗺 Dove siamo

Il concetto di stream (§15.1), scrivere su file con ofstream (§15.2), leggere con ifstream (§15.3), leggere riga per riga (§15.4). Nel laboratorio (§15.5) salvi e ricarichi dei dati.

🔑 Perché ci tocca: memoria persistente

Salvare e caricare dati è indispensabile in quasi ogni programma reale: impostazioni, archivi, log, risultati. Senza file, un programma dimentica tutto ad ogni avvio. In C++ farlo è sorprendentemente semplice grazie all'astrazione degli stream, coerente con l'I/O che già usi. E imparerai un'ultima, elegante applicazione di RAII: i file si aprono con un oggetto e si chiudono automaticamente quando quell'oggetto muore.

Sezione 15.1Gli stream

Il C++ tratta ogni sorgente/destinazione di dati come uno stream: un «flusso» da cui si legge o su cui si scrive, sempre con la stessa interfaccia. Schermo, tastiera, file: tutti stream.

📐 Un'unica idea per tutto l'I/O

Uno stream è un flusso ordinato di dati. Ne conosci già due: std::cout (stream di output verso lo schermo, si usa con <<) e std::cin (stream di input dalla tastiera, con >>). La bellezza: un file è solo un altro stream. Per scrivere su file usi un std::ofstream («output file stream») con lo stesso <<; per leggere, un std::ifstream («input file stream») con lo stesso >>. Impari l'I/O su schermo (Cap. 1-2), e sai già l'I/O su file: cambia solo «dove va» il flusso. È un design elegante che rende il codice uniforme. Richiede #include <fstream>.

Sezione 15.2Scrivere su file (ofstream)

Per salvare dati, apri un ofstream su un nome di file e ci «spingi» dentro il testo con <<, esattamente come faresti con cout.

Scrivere con ofstream come cout, ma su file
scrivi.cpp
#include <fstream>

std::ofstream file("dati.txt");   // apre (crea/sovrascrive) il file in scrittura

file << "Prima riga\n";         // scrivi con << (come cout!)
file << "Numero: " << 42 << "\n";

// niente file.close(): si chiude DA SOLO quando "file" esce dall'ambito (RAII!)
std::ofstream file("dati.txt")
crea uno stream verso il file dati.txt, aprendolo in scrittura. Se il file non esiste lo crea; se esiste, lo sovrascrive (come la modalità "w" di Python). Per aggiungere in fondo: std::ofstream file("dati.txt", std::ios::app).
scrivere con <<
identico a cout: file << testo << numero. Puoi mischiare testo e valori, usare "\n" per andare a capo.
chiusura automatica (RAII!)
quando l'oggetto file esce dall'ambito (fine funzione/blocco), il suo distruttore chiude il file da solo — svuotando i dati sul disco. Non devi ricordarti close(): è RAII (Cap. 12) in azione. (Puoi chiuderlo prima con file.close() se serve.)

Sezione 15.3Leggere da file (ifstream)

Speculare: apri un ifstream e ne «estrai» i dati con >>, come da cin. Ma attento a controllare che il file si sia aperto davvero.

Leggere con ifstream come cin, ma da file
leggi.cpp
#include <fstream>
#include <iostream>

std::ifstream file("dati.txt");   // apre il file in lettura

if (!file) {                     // controlla: si è aperto? (magari il file non c'è)
    std::cout << "Impossibile aprire il file\n";
    return 1;
}

std::string parola;
while (file >> parola) {          // legge parola per parola finché ce ne sono
    std::cout << parola << "\n";
}
std::ifstream file("dati.txt")
apre il file in lettura. Come per ofstream, si chiude da solo (RAII).
controlla sempre l'apertura
if (!file) verifica se l'apertura è fallita (file inesistente, permessi…). Saltare questo controllo è un errore classico: se il file non c'è e non te ne accorgi, leggeresti dati inesistenti.
while (file >> parola)
legge dallo stream finché ci sono dati: l'espressione file >> parola è «vera» finché la lettura riesce, «falsa» a fine file. È l'idioma per scorrere tutto il contenuto parola per parola. Come cin, >> si ferma agli spazi.
ofstream scrive, ifstream rilegge scrivi delle righe e rileggile dal "file"

Sezione 15.4Leggere riga per riga

>> legge una «parola» alla volta (si ferma agli spazi). Per leggere righe intere — il caso più comune con i file di testo — si usa di nuovo std::getline (Cap. 9).

getline su un file una riga alla volta
righe.cpp
std::ifstream file("testo.txt");
std::string riga;

while (std::getline(file, riga)) {   // legge una RIGA intera per volta
    std::cout << "> " << riga << "\n";
}
std::getline(file, riga) legge fino all'a-capo, spazi inclusi, e mette il tutto in riga. Il while continua finché ci sono righe (getline «diventa falso» a fine file). Questo è il modo standard di leggere un file di testo riga per riga — per elaborare un log, un CSV, un elenco. La stessa getline del Cap. 9, applicata a uno stream di file invece che a cin: ecco di nuovo l'eleganza degli stream, un'unica idea per schermo, tastiera e file.

Sezione 15.5Laboratorio: salvare dati

Uniamo scrittura e lettura in una mini-app che salva una lista di nomi su file e poi la rilegge — lo schema «persisti e ricarica» che sta dietro moltissimi programmi.

Il programma rubrica.cpp scrivi, poi rileggi
rubrica.cpp
#include <iostream>
#include <fstream>
#include <vector>
#include <string>

int main() {
    std::vector<std::string> nomi = {"Anna", "Marco", "Bea"};

    // SCRIVE i nomi, uno per riga
    {
        std::ofstream out("nomi.txt");
        for (const std::string& n : nomi) {
            out << n << "\n";
        }
    }   // "out" esce dall'ambito qui → file chiuso e salvato (RAII)

    // RILEGGE e stampa
    std::ifstream in("nomi.txt");
    std::string riga;
    std::cout << "Rubrica:\n";
    while (std::getline(in, riga)) {
        std::cout << "- " << riga << "\n";
    }
    return 0;
}
Output: «Rubrica:» seguito da «- Anna / - Marco / - Bea» — dati letti dal file appena scritto. Nota il blocco { } attorno alla scrittura: serve a far uscire out dall'ambito prima di rileggere, così RAII chiude e salva il file (garantendo che i dati siano su disco prima della rilettura). Questo schema «scrivi → chiudi → rileggi» è il fondamento della persistenza. Compila con g++ rubrica.cpp -o rubrica. Estendilo: leggi i nomi dall'utente prima di salvarli, o aggiungi (modalità app) invece di sovrascrivere.
🔑 Quello che devi portarti dietro dal Capitolo 15
  1. Uno stream è un flusso di dati; cout/cin e i file usano la stessa interfaccia (<</>>). Serve #include <fstream> (§15.1).
  2. std::ofstream file("nome") scrive (con <<), sovrascrivendo (o std::ios::app per aggiungere) (§15.2). std::ifstream legge (con >>); controlla sempre if (!file) (§15.3).
  3. Leggi righe intere con std::getline(file, riga) in un while (§15.4).
  4. I file si chiudono da soli (RAII) a fine ambito; lo schema scrivi → chiudi → rileggi dà persistenza (§15.5).

Sezione 15.7Soluzioni

Soluzione 15.1

Sono tutti stream (flussi di dati): l'astrazione unica del C++ per l'I/O. Si usano con gli operatori << (output, «spingi dentro lo stream») e >> (input, «estrai dallo stream»). Cambia solo la destinazione/sorgente (schermo, tastiera, file), non il modo di usarli.

Soluzione 15.2

Perché l'apertura può fallire: il file potrebbe non esistere, o mancare i permessi. if (!file) rileva l'apertura fallita, così puoi avvisare/uscire invece di proseguire leggendo dati inesistenti (che darebbe risultati sbagliati). È un controllo essenziale prima di leggere.

Soluzione 15.3

file >> parola legge una parola alla volta (si ferma agli spazi e agli a-capo); std::getline(file, riga) legge un'intera riga (fino all'a-capo), spazi inclusi. Per file di testo «a righe» (log, elenchi) si usa getline; per token separati da spazi, >>.

Soluzione 15.4
log.cpp
std::ofstream log("log.txt", std::ios::app);
log << "avvio\n";

L'append (std::ios::app) aggiunge in fondo senza cancellare le righe precedenti: per un log vuoi conservare lo storico. La scrittura normale (senza app) sovrascriverebbe il file, perdendo tutto ciò che c'era.
