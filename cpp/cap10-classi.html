Capitolo 10

Struct e classi

Hai usato le classi degli altri — std::string, std::vector sono oggetti che uniscono dati e metodi. Ora impari a costruire i tuoi tipi. La programmazione a oggetti unisce i dati e i comportamenti che li riguardano in un unico «oggetto», modellato su un progetto (la classe). Il C++ offre due strumenti quasi identici — struct e class — e aggiunge un concetto potente: l'incapsulamento, cioè decidere cosa è public (accessibile da fuori) e cosa private (protetto all'interno). E introduce i costruttori (che preparano ogni oggetto) e i distruttori (che lo puliscono) — un meccanismo che porterà, tra due capitoli, all'idea più elegante del C++: gli oggetti che gestiscono da soli le proprie risorse.

🗺 Dove siamo

La struct per tenere dati insieme (§10.1), la class con dati + metodi (§10.2), l'incapsulamento con public e private (§10.3), i costruttori (§10.4) e il distruttore (§10.5). Nel laboratorio (§10.6) modelli qualcosa di reale.

🔑 Perché ci tocca: progettare i propri tipi

Le classi sono il modo in cui è organizzato tutto il software C++ serio. Ti permettono di creare tipi «su misura» per il tuo problema — un giocatore, un conto, una connessione — con i loro dati e le loro operazioni, protetti e coerenti. L'incapsulamento (public/private), in particolare, è ciò che rende il codice robusto: nasconde i dettagli e impedisce usi scorretti. E costruttori/distruttori sono la base di RAII, l'idea che rende il C++ moderno sicuro sulla memoria (Cap. 12). È il capitolo che ti fa progettare, non solo usare.

Sezione 10.1struct: dati insieme

La struct è il modo più semplice per raggruppare dati correlati sotto un unico nome. Un «pacchetto» di variabili che vanno insieme.

Una struct dati correlati in un tipo
esempio.cpp
struct Punto {        // definisce un nuovo tipo "Punto"
    int x;
    int y;
};                    // nota il ; finale dopo la graffa!

Punto p;              // crea un Punto
p.x = 3;             // accede ai campi col punto
p.y = 5;
std::cout << p.x << ", " << p.y;   // 3, 5
struct Punto { … };
definisce un nuovo tipo che racchiude dei campi (le variabili membro). Attenzione al ; dopo la graffa di chiusura — dimenticarlo è un errore classico. Da lì, Punto è un tipo come int, con cui creare variabili.
accesso ai campi col .
p.x, p.y: il punto (l'operatore, non la geometria!) accede ai membri. Come frase.length() sulla string.
a cosa serve
invece di portarti dietro tre variabili sciolte (x, y, magari colore), le tieni unite in un tipo dal significato chiaro. Puoi passare un intero Punto a una funzione, metterlo in un vector, ecc.

Sezione 10.2class: dati + metodi

La class fa un passo in più: unisce ai dati anche i metodi — le funzioni che operano su quei dati. È l'oggetto completo: dati + comportamento, insieme.

Una class dati + metodi in un oggetto
conto.cpp
class Conto {
public:
    double saldo;             // dato membro

    void deposita(double importo) {   // metodo
        saldo += importo;             // accede direttamente al campo saldo
    }
    void preleva(double importo) {
        if (importo <= saldo) saldo -= importo;
    }
};

Conto c;
c.saldo = 100;
c.deposita(50);     // chiama il metodo sull'oggetto → saldo 150
class Conto { … };
come la struct, ma pensata per contenere anche metodi. Dentro, i dati membro e le funzioni membro.
i metodi accedono ai campi direttamente
dentro un metodo, scrivi saldo e ti riferisci al campo dell'oggetto su cui il metodo è chiamato (l'equivalente del self di Python, qui implicito). Ogni oggetto ha i suoi dati: c.deposita(50) modifica il saldo di c.
chiamare un metodo
c.deposita(50): come vettore.push_back(9) o stringa.length(). Ora sai cosa c'è dietro quei metodi che usi: sono funzioni membro di una classe.
Un oggetto Conto: dati e metodi chiama i metodi, guarda il campo saldo
💡 struct o class? (quasi la stessa cosa)

Tecnicamente, in C++ struct e class sono quasi identiche: l'unica differenza è il default di visibilità (nella struct i membri sono public di default, nella class private). Convenzione diffusa: usa struct per semplici «sacchetti di dati» pubblici (come Punto), e class quando c'è comportamento e incapsulamento (dati da proteggere, metodi). Non è una regola ferrea, ma un'abitudine utile.

Sezione 10.3public e private

L'incapsulamento è una delle idee più potenti dell'OOP: decidere cosa è accessibile da fuori la classe (public) e cosa resta protetto all'interno (private). Protegge i dati da usi scorretti.

Incapsulamento nascondi i dati, esponi i metodi
conto.cpp
class Conto {
private:                     // accessibile SOLO dai metodi della classe
    double saldo = 0;

public:                      // accessibile da fuori
    void deposita(double x) {
        if (x > 0) saldo += x;    // controllo: solo importi positivi
    }
    double get_saldo() {         // un "getter" per leggere il saldo
        return saldo;
    }
};

Conto c;
c.deposita(50);        // OK (metodo public)
// c.saldo = -1000000;    →  ERRORE! saldo è private, non accessibile da fuori
std::cout << c.get_saldo();   // 50
private: — nascosto
i membri sotto private sono accessibili solo dai metodi della classe stessa, non da fuori. Rende saldo «intoccabile» dall'esterno: nessuno può metterlo a un valore assurdo direttamente.
public: — esposto
i membri sotto public sono l'interfaccia della classe: ciò che il mondo esterno può usare. Qui i metodi deposita e get_saldo.
perché è potente
tenendo i dati privati e esponendo solo metodi controllati, garantisci che l'oggetto resti sempre in uno stato valido: deposita rifiuta importi negativi, quindi il saldo non può essere corrotto. Nascondi i dettagli, esponi un'interfaccia sicura: è il cuore dell'incapsulamento, e il motivo per cui il codice a oggetti ben fatto è robusto.

Sezione 10.4Costruttori

Un costruttore è un metodo speciale che prepara ogni nuovo oggetto: viene chiamato automaticamente alla creazione e inizializza i dati. Garantisce che un oggetto nasca sempre «pronto».

Il costruttore stesso nome della classe, nessun tipo di ritorno
conto.cpp
class Conto {
private:
    double saldo;
    std::string titolare;
public:
    Conto(std::string nome, double iniziale) {   // COSTRUTTORE
        titolare = nome;
        saldo = iniziale;
    }
    double get_saldo() { return saldo; }
};

Conto c("Anna", 100);   // il costruttore parte in automatico: titolare="Anna", saldo=100
std::cout << c.get_saldo();   // 100
Conto(std::string nome, double iniziale)
il costruttore ha lo stesso nome della classe e nessun tipo di ritorno (nemmeno void). Riceve i valori iniziali e li assegna ai campi.
chiamata automatica
Conto c("Anna", 100); crea l'oggetto ed esegue il costruttore con quegli argomenti. Garantisce che ogni Conto nasca con un titolare e un saldo: niente campi «spazzatura» (Cap. 2). È il modo giusto per inizializzare gli oggetti.
overloading dei costruttori
puoi avere più costruttori con parametri diversi (Cap. 6): es. uno che parte da saldo 0. Il compilatore sceglie quello giusto.

Sezione 10.5Il distruttore

Il gemello opposto del costruttore: il distruttore viene chiamato automaticamente quando un oggetto viene distrutto (esce dal suo ambito o viene liberato). Serve a fare le «pulizie».

📐 Nascita e morte di un oggetto
esempio.cpp
class Risorsa {
public:
    Risorsa() { std::cout << "Creata\n"; }        // costruttore
    ~Risorsa() { std::cout << "Distrutta\n"; }    // DISTRUTTORE: ~ + nome
};

int main() {
    Risorsa r;        // stampa "Creata"
    std::cout << "Uso r\n";
}                     // alla fine del blocco, r viene distrutto → stampa "Distrutta" (automatico!)

Il distruttore ha lo stesso nome della classe preceduto da una tilde ~, e viene chiamato automaticamente quando l'oggetto «muore» (qui, alla fine di main). Perché è importante? Perché è il posto giusto per rilasciare risorse: chiudere un file, liberare memoria, chiudere una connessione. Se un oggetto ha «preso» qualcosa nel costruttore, lo «restituisce» nel distruttore — automaticamente, con certezza, anche se il programma esce per un errore. Questa coppia costruttore/distruttore è la base dell'idea più elegante del C++, RAII, che affronteremo nel Capitolo 12. Per ora, tieni l'immagine: gli oggetti C++ hanno una nascita e una morte precise, ed entrambe possono «fare» qualcosa in automatico.

Sezione 10.6Laboratorio: modellare qualcosa

Costruiamo una classe completa — un Libro — con dati privati, un costruttore, metodi che «fanno qualcosa» e l'incapsulamento. È l'esempio-tipo di come si modella un'entità del mondo reale in C++.

La classe Libro tutto insieme
libro.cpp
#include <iostream>
#include <string>

class Libro {
private:
    std::string titolo;
    int pagine;
    int letta = 0;
public:
    Libro(std::string t, int p) {   // costruttore
        titolo = t;
        pagine = p;
    }
    void leggi(int n) {
        letta += n;
        if (letta > pagine) letta = pagine;   // non superare la fine
    }
    int progresso() {
        return letta * 100 / pagine;   // percentuale
    }
};

int main() {
    Libro b("C++", 200);
    b.leggi(50);
    std::cout << b.progresso() << "%\n";   // 25%
    return 0;
}
Guarda com'è tutto coeso e protetto: i dati (titolo, pagine, letta) sono privati — nessuno può manometterli da fuori; l'unico modo di interagire è tramite i metodi pubblici (leggi, progresso), che mantengono l'oggetto sempre coerente (letta non supera mai pagine). Tutto ciò che riguarda un libro sta in un posto solo, e in uno stato sempre valido: questa è la potenza dell'OOP con l'incapsulamento. Compila con g++ libro.cpp -o libro. Estendilo: aggiungi un metodo finito() che restituisce true se hai letto tutto.
🔑 Quello che devi portarti dietro dal Capitolo 10
  1. La struct raggruppa dati (campi), accesso col .; occhio al ; dopo la graffa (§10.1). La class aggiunge i metodi: dati + comportamento in un oggetto (§10.2).
  2. L'incapsulamento: private nasconde i dati (protetti), public espone l'interfaccia (metodi). Nascondi i dettagli, esponi operazioni sicure (§10.3).
  3. Il costruttore (nome della classe, nessun tipo di ritorno) prepara ogni oggetto alla creazione, garantendo dati validi (§10.4).
  4. Il distruttore (~Classe()) è chiamato automaticamente alla «morte» dell'oggetto: il posto per le pulizie. È la base di RAII (Cap. 12) (§10.5).

Sai progettare tipi tuoi. Ma spesso più tipi hanno molto in comune: un Cane e un Gatto sono entrambi Animale; un Cerchio e un Quadrato sono entrambi Forma. Riscrivere il codice comune sarebbe uno spreco. Nel Capitolo 11 vediamo l'ereditarietà (una classe che si basa su un'altra) e il polimorfismo — la capacità, tramite i metodi virtuali, di trattare oggetti diversi con un'interfaccia comune, lasciando che ognuno risponda «a modo suo». È una delle idee più potenti dell'OOP, e in C++ ha un legame profondo con i puntatori che hai già imparato.

Sezione 10.8Soluzioni

Soluzione 10.1

Tecnicamente differiscono solo per il default di visibilità: nella struct i membri sono public di default, nella class private. Convenzione: struct per semplici sacchetti di dati pubblici (es. Punto), class quando c'è comportamento e incapsulamento (dati da proteggere, metodi).

Soluzione 10.2

Rendere un dato private impedisce di modificarlo direttamente dall'esterno, costringendo a passare per metodi controllati. Esempio: se saldo fosse public, chiunque potrebbe scrivere c.saldo = -1000000; rendendolo private ed esponendo solo deposita/preleva (che controllano gli importi), l'oggetto resta sempre in uno stato valido.

Soluzione 10.3

Un costruttore è un metodo speciale che inizializza un oggetto alla creazione. Si riconosce perché ha lo stesso nome della classe e nessun tipo di ritorno (nemmeno void). Viene chiamato automaticamente quando crei un oggetto (Conto c("Anna", 100);), garantendo che nasca con dati validi.

Soluzione 10.4
rettangolo.cpp
class Rettangolo {
private:
    int base, altezza;
public:
    Rettangolo(int b, int h) { base = b; altezza = h; }
    int area() { return base * altezza; }
};

Rettangolo r(4, 3);
std::cout << r.area();   // 12
