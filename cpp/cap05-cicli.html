Capitolo 5

I cicli

Il superpotere dell'automazione: far ripetere al computer la stessa operazione, migliaia di volte, senza sbagliare. In C++ i cicli sono uno strumento centrale e potente — dal while «finché», al celebre for classico con la sua terna inizializzazione-condizione-incremento (compatta e onnipresente in tutto il codice C++), fino al moderno for range-based che scorre le collezioni con eleganza. Uniti alle decisioni del capitolo scorso, i cicli sbloccano il lavoro vero: elaborare liste, righe, dati in massa con poche righe di codice. Il for del C++, in particolare, è così caratteristico che imparerai a leggerlo a colpo d'occhio — lo troverai ovunque.

🗺 Dove siamo

I cicli while e do-while (§5.1), il for classico con le sue tre parti (§5.2), il moderno for range-based (§5.3), break e continue (§5.4). Nel laboratorio (§5.5) generi tabelline e figure di testo.

🔑 Perché ci tocca: fare tanto con poco

La ripetizione è l'essenza dell'automazione: tutto ciò che va fatto «per ognuno» o «più volte» è un ciclo. Insieme all'if, il for è lo strumento che userai in ogni programma non banale. Il for classico del C++, con la sua struttura compatta a tre parti, è anche una «firma» del linguaggio: saperlo leggere e scrivere fluentemente è indispensabile per capire qualsiasi codice C++ tu incontri.

Sezione 5.1while e do-while

Il ciclo più semplice: while ripete un blocco finché una condizione è vera. La variante do-while garantisce almeno un'esecuzione.

while e do-while finché · almeno una volta
esempio.cpp
int conto = 3;
while (conto > 0) {
    std::cout << conto << " ";
    conto--;               // FONDAMENTALE: avvicinare la fine (o ciclo infinito!)
}
// stampa: 3 2 1

do {
    std::cout << "Eseguito almeno una volta";
} while (false);   // do-while controlla la condizione DOPO
while (condizione) { … }
controlla la condizione prima di ogni giro: se vera esegue, poi ricontrolla. Se falsa dall'inizio, il blocco non viene mai eseguito.
do { … } while (condizione);
controlla dopo: quindi il blocco viene eseguito almeno una volta, poi ripete finché la condizione è vera. Utile per menu e input («chiedi, poi verifica»). Nota il ; finale.
il pericolo: il ciclo infinito
dentro il while devi far avvicinare la condizione alla fine (qui conto--). Dimenticarlo = ciclo che non finisce mai (fermalo con Ctrl+C). L'errore più temuto del while.

Sezione 5.2Il for classico

La firma del C++. Il for classico raccoglie tre cose in una riga — inizializza, controlla, incrementa — rendendo compatto ed esplicito il ciclo «ripeti N volte». Vale la pena capirlo a fondo.

Il for a tre parti for (init; condizione; incremento)
esempio.cpp
for (int i = 0; i < 5; i++) {
    std::cout << i << " ";
}
// stampa: 0 1 2 3 4
1. inizializzazione: int i = 0
eseguita una sola volta, all'inizio: crea la variabile del ciclo e le dà il valore di partenza.
2. condizione: i < 5
controllata prima di ogni giro: se vera, esegue il corpo; se falsa, il ciclo finisce. Qui: continua finché i è minore di 5.
3. incremento: i++
eseguito alla fine di ogni giro: fa avanzare la variabile. Qui aumenta i di 1.
il flusso completo
init → (condizione? → corpo → incremento) → (condizione? → corpo → incremento) → … finché la condizione diventa falsa. Con i < 5 partendo da 0: gira per i = 0, 1, 2, 3, 4 (cinque volte), poi i diventa 5, la condizione è falsa, si esce.
Il for classico è perfetto quando conosci il numero di ripetizioni o lavori con gli indici. Le tre parti in una riga tengono tutta la «logica di conteggio» in un colpo d'occhio — ecco perché i programmatori C++ lo amano. Il widget qui sotto lo esegue passo passo: guarda le tre parti al lavoro.

Sezione 5.3Il for range-based

Il C++ moderno (dal 2011) offre un for più semplice per scorrere una collezione senza gestire indici: il for range-based. Assomiglia molto al for di Python.

Il for range-based for (tipo x : collezione)
esempio.cpp
#include <vector>

std::vector<int> voti = {7, 8, 6, 9};   // una lista (Cap. 8)

for (int v : voti) {          // "per ogni v in voti"
    std::cout << v << " ";
}
// stampa: 7 8 6 9
for (int v : voti)
si legge «per ogni v nella collezione voti»: ad ogni giro v assume l'elemento successivo. Niente indici da gestire: più semplice e meno soggetto a errori quando ti serve solo scorrere.
for (auto v : voti)
spesso si usa auto per far dedurre il tipo (Cap. 2): comodo con collezioni di tipi complessi.
per modificare gli elementi: il riferimento
for (int& v : voti) (con la &) permette di modificare gli elementi originali (i riferimenti sono nel Cap. 7). Senza &, v è una copia e modificarla non tocca la collezione.
Quando usare quale for? Il range-based quando devi solo scorrere una collezione (più leggibile). Il classico quando ti servono gli indici (es. accedere per posizione, andare a passi, contare) o quando non hai una collezione ma vuoi «ripetere N volte». Nel dubbio, se hai una collezione da scorrere, usa il range-based.

Sezione 5.4break e continue

Due parole per controllare un ciclo dall'interno, identiche in significato a tanti linguaggi: break lo interrompe, continue salta al giro successivo.

break e continue interrompi · salta
esempio.cpp
for (int i = 1; i <= 10; i++) {
    if (i == 5) break;        // esce dal ciclo quando i vale 5
    std::cout << i << " ";      // stampa: 1 2 3 4
}

for (int i = 1; i <= 6; i++) {
    if (i % 2 == 0) continue;  // salta i pari
    std::cout << i << " ";      // stampa: 1 3 5
}
break abbandona subito tutto il ciclo (utile: «ho trovato ciò che cercavo»); continue interrompe solo il giro corrente e passa al successivo (utile: «questo elemento non m'interessa»). Nel for, dopo un continue, l'incremento viene comunque eseguito. Usali con parsimonia: troppi salti rendono un ciclo difficile da seguire.

Sezione 5.5Laboratorio: tabelline e figure

Due classici che mostrano la potenza dei cicli: una tabellina e un triangolo di asterischi con cicli annidati. Poche righe, risultati che a mano richiederebbero pagine.

Tabellina e triangolo for singolo e annidato
figure.cpp
#include <iostream>

int main() {
    int n = 5;

    // tabellina del 5
    for (int i = 1; i <= 10; i++) {
        std::cout << n << " x " << i << " = " << n * i << "\n";
    }

    // triangolo di asterischi (cicli ANNIDATI)
    for (int riga = 1; riga <= 4; riga++) {
        for (int j = 0; j < riga; j++) {   // interno: 'riga' asterischi
            std::cout << "*";
        }
        std::cout << "\n";
    }
    return 0;
}
Il triangolo esce così: * / ** / *** / ****. Sono cicli annidati: per ogni riga del ciclo esterno, quello interno stampa riga asterischi. È lo schema per tutto ciò che ha due dimensioni (griglie, tabelle). Nota anche l'accumulatore implicito della tabellina: n * i ricalcolato ogni giro. Compila con g++ figure.cpp -o figure, poi divertiti a modificare le dimensioni o a disegnare altre figure.
🔑 Quello che devi portarti dietro dal Capitolo 5
  1. while (cond) ripete finché vera (controlla prima); do…while esegue almeno una volta (controlla dopo). Attento al ciclo infinito (§5.1).
  2. Il for classico for (init; cond; incr): init una volta, cond prima di ogni giro, incr alla fine. Per contare e usare gli indici (§5.2).
  3. Il for range-based for (auto x : coll) scorre una collezione senza indici; con & per modificarla (§5.3).
  4. break esce, continue salta al giro dopo (§5.4). I cicli si annidano per le griglie (§5.5).

Con variabili, decisioni e cicli hai i tre pilastri della logica: puoi già scrivere programmi non banali. Ma man mano che crescono, riscriverai gli stessi pezzi. Serve organizzare il codice in blocchi riutilizzabili con un nome. Nel Capitolo 6 arrivano le funzioni del C++: come si dichiarano e definiscono, il tipo di ritorno e i parametri, e una distinzione fondamentale che il C++ rende esplicita — il passaggio degli argomenti per valore o per riferimento, che cambia se la funzione può o no modificare i dati che riceve. È un concetto che ci porterà dritti al cuore del C++: i puntatori.

Sezione 5.7Soluzioni

Soluzione 5.1

Init int i = 0 (una volta, all'inizio); condizione i < 5 (prima di ogni giro); incremento i++ (alla fine di ogni giro). Gira 5 volte (i = 0,1,2,3,4). L'incremento avviene dopo il corpo, prima di ricontrollare la condizione.

Soluzione 5.2

while controlla la condizione prima: se falsa all'inizio, il blocco non gira mai. do-while controlla dopo: quindi il blocco viene eseguito almeno una volta, poi ripete finché la condizione è vera. Il do-while esegue sempre almeno una volta.

Soluzione 5.3
stampa.cpp
for (int x : v) {
    std::cout << x << "\n";
}
Soluzione 5.4
somma.cpp
#include <iostream>
int main() {
    int totale = 0;
    for (int i = 1; i <= 100; i++) {
        totale += i;
    }
    std::cout << totale << std::endl;   // 5050
    return 0;
}
