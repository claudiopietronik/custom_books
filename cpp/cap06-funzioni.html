Capitolo 6

Le funzioni

Le funzioni danno un nome a un blocco di codice per riusarlo a comando: sono lo strumento di organizzazione per eccellenza, in C++ come ovunque. Ma il C++ aggiunge una distinzione che rende esplicita una cosa fondamentale — quando passi un dato a una funzione, glielo dai in copia o glielo presti? È la differenza tra passaggio per valore e per riferimento, e decide se la funzione può o no modificare i dati che riceve. Capirla è il ponte diretto verso il cuore del C++: i puntatori (prossimo capitolo). In questo capitolo vediamo come si dichiarano e definiscono le funzioni, i parametri, l'overloading, e come organizzarle in file header.

🗺 Dove siamo

Dichiarare e definire una funzione (§6.1), tipo di ritorno e parametri (§6.2), il passaggio per valore vs per riferimento (§6.3), overloading e valori di default (§6.4), e come si organizzano le dichiarazioni negli header (§6.5). Nel laboratorio (§6.6) costruisci una libreria di funzioni.

🔑 Perché ci tocca: organizzare, e capire la memoria

Le funzioni scompongono i problemi in pezzi gestibili e riutilizzabili — la competenza che trasforma «script» in programmi ordinati. Ma in C++ le funzioni insegnano anche qualcosa di più profondo: il passaggio per valore/riferimento è il primo assaggio di come il C++ ragiona sulla memoria — copie vs prestiti, efficienza, effetti collaterali. È il concetto che apre la porta a puntatori e riferimenti, il tratto più caratteristico e potente del linguaggio.

Sezione 6.1Dichiarare e definire

In C++ una funzione si scrive indicando il tipo di ritorno, il nome, i parametri (con i loro tipi) e il corpo. Tutto è tipizzato, come le variabili.

Definire una funzione tipo nome(parametri) { … return … }
esempio.cpp
#include <iostream>

int area_rettangolo(int base, int altezza) {   // tipo ritorno, nome, parametri tipati
    return base * altezza;
}

int main() {
    int a = area_rettangolo(4, 3);   // chiamata: base=4, altezza=3
    std::cout << a << std::endl;    // 12
    return 0;
}
int area_rettangolo(int base, int altezza)
il tipo di ritorno (int) davanti, poi nome e parametri con il loro tipo. Ogni parametro dichiara tipo e nome, separati da virgole.
return valore;
restituisce un valore del tipo dichiarato e termina la funzione. Il compilatore verifica che il tipo combaci: restituire un tipo sbagliato è un errore.
l'ordine conta: definire prima di usare
in C++ una funzione va dichiarata prima del punto in cui la chiami. Per questo area_rettangolo sta sopra main. (In alternativa si mette una «dichiarazione anticipata» in cima — §6.5.)
Una funzione che «fa qualcosa» ma non restituisce nulla ha tipo di ritorno void: void saluta() { std::cout << "Ciao"; }. void = «nessun valore di ritorno» — per funzioni che compiono un'azione (stampare, salvare) senza «dare indietro» un risultato.

Sezione 6.2Tipo di ritorno e parametri

Ogni funzione è un piccolo contratto tipizzato: dichiara cosa riceve (i parametri, coi loro tipi) e cosa restituisce (il tipo di ritorno). Il compilatore fa rispettare il contratto.

📐 Il contratto della funzione

La «firma» di una funzione (nome + tipi dei parametri + tipo di ritorno) è un contratto: chi la chiama deve passare argomenti dei tipi giusti, e riceverà un risultato del tipo dichiarato. Se sbagli — passi un testo dove serve un numero, o usi male il valore restituito — il compilatore te lo dice subito, prima dell'esecuzione. È di nuovo la sicurezza della tipizzazione statica (Cap. 2), applicata alle funzioni. Un parametro può essere di qualsiasi tipo: int, double, std::string (Cap. 9), o strutture più complesse. E una funzione può non avere parametri: int main() ne è l'esempio.

Sezione 6.3Passaggio per valore e per riferimento

Ecco il concetto chiave del capitolo, tipico del C++. Quando passi un argomento, per default ne viene fatta una copia (per valore). Ma puoi anche passare l'originale (per riferimento), permettendo alla funzione di modificarlo.

Per valore vs per riferimento copia o originale?
esempio.cpp
void raddoppia_copia(int x) {     // PER VALORE: x è una copia
    x = x * 2;                  // modifica solo la copia locale
}

void raddoppia_rif(int& x) {     // PER RIFERIMENTO: la & !  x è l'originale
    x = x * 2;                  // modifica la variabile del chiamante
}

int n = 5;
raddoppia_copia(n);    // n resta 5  (modificata solo la copia)
raddoppia_rif(n);      // n diventa 10  (modificato l'originale)
per valore (default): int x
la funzione riceve una copia dell'argomento. Modificarla non tocca la variabile originale del chiamante. Sicuro, ma per dati grandi costa (copiare).
per riferimento: int& x (la &!)
la & dopo il tipo significa «riferimento»: la funzione lavora sull'originale, non su una copia. Modificarla cambia la variabile del chiamante. Potente — e da usare con consapevolezza.
riferimento const: const int& x
il meglio dei due mondi per i dati grandi: passi l'originale (niente copia costosa) ma const impedisce di modificarlo. È la scelta idiomatica per passare oggetti pesanti (stringhe, vettori) in sola lettura, in modo efficiente.
Copia o originale? per valore vs per riferimento chiama le due funzioni e guarda n
💡 Perché è importante

Il passaggio per riferimento serve a due scopi: modificare i dati del chiamante (es. una funzione ordina(vettore) che riordina la lista originale), e evitare copie costose (passare un grosso oggetto per riferimento const, invece di copiarlo). In Python questa distinzione è nascosta; il C++ la mette in mano a te — più potere, ma anche più responsabilità. E la & che hai appena visto è imparentata con la stessa & dei puntatori (Cap. 7): stai già mettendo le mani sulla memoria.

Sezione 6.4Overloading e default

Due comodità del C++: puoi avere più funzioni con lo stesso nome ma parametri diversi (overloading), e dare ai parametri dei valori di default.

Overloading e default stesso nome · parametri opzionali
esempio.cpp
// OVERLOADING: stesso nome, tipi di parametri diversi
int somma(int a, int b) { return a + b; }
double somma(double a, double b) { return a + b; }
// somma(2, 3) usa la prima; somma(2.5, 1.5) usa la seconda: il compilatore sceglie

// VALORI DI DEFAULT: parametri opzionali
void saluta(std::string nome, std::string saluto = "Ciao") {
    std::cout << saluto << " " << nome;
}
saluta("Anna");              // Ciao Anna  (usa il default)
saluta("Marco", "Salve");   // Salve Marco
L'overloading permette funzioni con lo stesso nome ma «firme» diverse (tipi/numero di parametri): il compilatore sceglie quella giusta in base agli argomenti. Utile per operazioni concettualmente uguali su tipi diversi. I valori di default rendono un parametro opzionale (i parametri con default vanno alla fine). Sono le stesse idee che, «da fuori», rendono comode tante funzioni della libreria standard.

Sezione 6.5Dichiarazioni e header

Nei progetti veri, il codice si divide in più file. Il C++ usa una distinzione tra dichiarazione (la «firma», senza corpo) e definizione (col corpo), e i file header per condividere le dichiarazioni.

📐 Prototipo, header, definizione

Una dichiarazione (o «prototipo») dice al compilatore che una funzione esiste, con quale firma, senza darne il corpo: int somma(int a, int b); (nota il ;, niente graffe). Serve per poter chiamare una funzione prima che sia definita, o definita in un altro file. La definizione è quella completa, col corpo. Nei progetti, le dichiarazioni si mettono in un file .h (o .hpp, l'header) che gli altri file #includeono, e le definizioni in un file .cpp. Così un file può usare le funzioni di un altro conoscendone solo le firme. È lo stesso #include <iostream> del Capitolo 1: stai includendo l'header della libreria di I/O! Per ora non ti serve organizzare tutto così, ma sapere cos'è un header ti farà capire la struttura dei progetti C++.

Sezione 6.6Laboratorio: una libreria di funzioni

Costruiamo alcune funzioni utili che collaborano — una che chiama l'altra — e usiamo il passaggio per riferimento per uno scopo concreto: scambiare due valori.

Funzioni che collaborano e uno scambio per riferimento
utili.cpp
#include <iostream>

bool e_pari(int n) {
    return n % 2 == 0;
}

// scambia due valori: SERVE il passaggio per riferimento!
void scambia(int& a, int& b) {
    int tmp = a;
    a = b;
    b = tmp;
}

int main() {
    int x = 3, y = 8;
    scambia(x, y);          // ora x=8, y=3
    std::cout << "x=" << x << " y=" << y << "\n";
    std::cout << "8 pari? " << e_pari(8) << std::endl;   // 1 (true)
    return 0;
}
La funzione scambia è l'esempio perfetto del perché serve il riferimento: se a e b fossero copie (per valore), scambieremmo le copie e le variabili originali x, y resterebbero invariate — la funzione sarebbe inutile! Con int& lavoriamo sugli originali, e lo scambio funziona davvero. Nota che e_pari(8) stampa 1, non «true»: cout mostra i bool come 1/0 (per vedere true/false serve un piccolo trucco, std::boolalpha). Compila con g++ utili.cpp -o utili.
🔑 Quello che devi portarti dietro dal Capitolo 6
  1. Una funzione: tipo nome(parametri) { … return … }, tutto tipizzato; void se non restituisce nulla. Va dichiarata prima dell'uso (§6.1-6.2).
  2. Per valore (default, int x): la funzione riceve una copia, l'originale non cambia. Per riferimento (int& x): lavora sull'originale e può modificarlo. const int& per passare grandi dati senza copiarli, in sola lettura (§6.3).
  3. Overloading: più funzioni con lo stesso nome, parametri diversi. Default: parametri opzionali (§6.4).
  4. Dichiarazione (firma, con ;) vs definizione (col corpo); gli header .h condividono le firme tra file (§6.5).

La & del passaggio per riferimento non è un caso isolato: è la punta dell'iceberg del tratto più distintivo — e temuto — del C++. Come fa un riferimento a «essere» la variabile originale? Dove vivono davvero i dati in memoria, e come li si può raggiungere direttamente? Nel Capitolo 7 affrontiamo i puntatori e i riferimenti: cos'è un indirizzo di memoria, gli operatori & (indirizzo di) e * (contenuto di), e perché i puntatori sono al tempo stesso il super-potere e il tallone d'Achille del C++. È il capitolo che, una volta capito, ti fa vedere come funziona davvero un computer.

Sezione 6.7Esercizi

Esercizio 6.1

Scrivi una funzione int quadrato(int n) che restituisce n al quadrato, e chiamala in main per stampare il quadrato di 7.

Esercizio 6.2

Qual è la differenza tra void f(int x) e void f(int& x)? In quale dei due una modifica a x dentro la funzione si vede fuori?

Esercizio 6.3

Perché la funzione scambia deve ricevere i parametri per riferimento? Cosa accadrebbe con il passaggio per valore?

Esercizio 6.4

(Pratico) Scrivi void incrementa(int& x, int quanto = 1) che aumenta x di quanto (default 1). Provala chiamandola con e senza il secondo argomento.

Sezione 6.8Soluzioni

Soluzione 6.1
quadrato.cpp
int quadrato(int n) { return n * n; }

int main() {
    std::cout << quadrato(7) << std::endl;   // 49
    return 0;
}
Soluzione 6.2

void f(int x) passa per valore: x è una copia, e modificarla non si vede fuori. void f(int& x) passa per riferimento: x è l'originale, e una modifica si vede nel chiamante. La modifica «esce» solo nel secondo caso (con la &).

Soluzione 6.3

Perché deve modificare le variabili originali del chiamante. Con il passaggio per valore, a e b sarebbero copie: la funzione scambierebbe le copie, mentre x e y resterebbero invariate — lo scambio sarebbe inutile. Il riferimento (int&) permette di agire sugli originali.

Soluzione 6.4
incrementa.cpp
void incrementa(int& x, int quanto = 1) {
    x += quanto;
}

int n = 10;
incrementa(n);       // n = 11  (default +1)
incrementa(n, 5);    // n = 16  (+5)
