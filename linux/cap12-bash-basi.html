Capitolo 12

Bash scripting: le basi

Fin qui hai dato i comandi uno per uno. Ma il vero superpotere del terminale è l'automazione: mettere una sequenza di comandi in un file e farli eseguire tutti insieme, tante volte, con un solo comando. Quel file è uno script bash — e scriverne uno è più semplice di quanto pensi, perché è fatto degli stessi comandi che già conosci, messi in fila. Ciò che fai a mano una volta, uno script lo fa per te per sempre: un backup, una pulizia, un rapporto, un'installazione. È il passaggio da «usare Linux» a «far lavorare Linux al posto tuo». In questo capitolo muoviamo i primi passi: cos'è uno script e perché è potente (§12.1), la riga magica iniziale (shebang) e come eseguire uno script (§12.2), le variabili per dare nomi ai valori (§12.3), e come far ricevere allo script dei dati in ingresso (§12.4). Se hai fatto il libro di Python, molte idee ti saranno familiari: uno script è «programmare con i comandi del terminale».

🗺 Dove siamo

Cos'è uno script (§12.1), lo shebang e l'esecuzione (§12.2), variabili ed echo (§12.3), leggere input e argomenti (§12.4). Nel laboratorio (§12.5) scrivi il tuo primo script.

🔑 Perché ci tocca: automatizzare è moltiplicare il tuo tempo

Ogni compito ripetitivo che automatizzi è tempo (e errori) risparmiato per sempre. Uno script scritto una volta lavora all'infinito: fa il backup ogni notte, controlla lo stato di un servizio, prepara un ambiente. È la competenza che distingue chi «usa» il terminale da chi lo «comanda». E sui server, gli script sono il pane quotidiano dell'amministrazione.

Sezione 12.1Cos'è uno script

Uno script bash è semplicemente un file di testo che contiene una sequenza di comandi, uno per riga. Quando lo esegui, la shell (bash) legge il file ed esegue i comandi in ordine, come se li avessi digitati tu uno dopo l'altro.

📖 Comandi in fila, in un file
saluto.sh — uno script semplicissimo
#!/bin/bash
echo "Ciao!"
echo "Oggi è:"
date
echo "I file qui sono:"
ls

Ecco tutto: uno script è una lista di comandi (gli stessi che daresti a mano) in un file, di solito con estensione .sh. Eseguendolo, bash fa scorrere i comandi dall'alto in basso: stampa «Ciao!», poi la data, poi «I file qui sono:», poi l'elenco dei file. La potenza: invece di digitare cinque comandi ogni volta, ne scrivi uno (lanciare lo script) e vengono eseguiti tutti. Moltiplica questo per compiti di venti, cinquanta comandi, e capisci perché gli script sono così preziosi. Le righe che iniziano con # (tranne lo shebang, §12.2) sono commenti: note per te, ignorate da bash.

Sezione 12.2Lo shebang e l'esecuzione

Due dettagli fanno di un file di testo uno script eseguibile: la prima riga speciale, lo shebang, e il permesso di esecuzione (il Cap. 6 torna utile!).

terminale — da file a script eseguibile
# 1) la prima riga dello script deve essere lo "shebang":
#    #!/bin/bash   -> dice al sistema "esegui questo con bash"

# 2) rendi lo script eseguibile (aggiungi il permesso x, Cap. 6)
$ chmod +x saluto.sh

# 3) eseguilo indicando il percorso (./ = "in questa cartella")
$ ./saluto.sh
Ciao!
Oggi è:
Sat Jul 19 ...
I due requisiti shebang · chmod +x · ./
lo shebang: #!/bin/bash
la prima riga dello script. Dice al sistema quale interprete usare per eseguirlo (qui bash). I caratteri #! sono lo «shebang»; senza questa riga, il sistema non saprebbe come eseguire il file.
chmod +x saluto.sh
rende lo script eseguibile (aggiunge il permesso x, Cap. 6). Senza, otterresti «Permission denied» provando a lanciarlo — l'errore che ora sai risolvere.
./saluto.sh
per eseguire uno script nella cartella corrente, premetti ./ («in questa cartella»). Serve perché, per sicurezza, la shell non esegue automaticamente i programmi della cartella corrente: devi indicarne esplicitamente il percorso.
Il flusso completo, da ricordare: scrivi lo script (con lo shebang in cima) → chmod +x → lancialo con ./nome.sh. In alternativa puoi eseguirlo senza renderlo eseguibile passandolo a bash: bash saluto.sh (utile per una prova veloce). Ma per uno script che userai spesso, conviene renderlo eseguibile una volta e lanciarlo con ./. L'estensione .sh è una convenzione (aiuta a riconoscerlo), non un obbligo.

Sezione 12.3Variabili ed echo

Come in ogni linguaggio, negli script puoi usare variabili: contenitori con un nome per conservare un valore e riusarlo. E echo è il comando per stampare — testo e valori delle variabili.

esempio.sh — variabili
#!/bin/bash
NOME="Anna"          # assegna (NIENTE spazi attorno all'=!)
CITTA="Roma"

echo "Ciao, $NOME!"          # $NOME "espande" il valore
echo "Vivi a ${CITTA}."      # ${...} è più esplicito, equivalente
l'output
Ciao, Anna!
Vivi a Roma.
Variabili in bash NOME=valore · $NOME
assegnare: NOME="Anna"
dai un valore a una variabile. Attenzione: niente spazi attorno all'=! NOME = "Anna" (con spazi) dà errore — è una delle trappole più comuni. La convenzione è nominare le variabili in MAIUSCOLO.
usare: $NOME o ${NOME}
il simbolo $ davanti al nome «espande» il valore della variabile. echo "Ciao $NOME" stampa «Ciao Anna». La forma ${NOME} (con graffe) è equivalente e più esplicita (utile per evitare ambiguità).
echo — stampare
stampa testo e valori. Usa le virgolette doppie "..." per far espandere le variabili dentro il testo; le virgolette singole '...' le lasciano letterali ($NOME resta $NOME). Nel dubbio, doppie.
Se hai fatto Python o JavaScript, l'idea di variabile ti è familiare — ma la sintassi di bash ha le sue peculiarità: niente spazi attorno all'= in assegnazione, e il $ per leggere il valore. Queste due regole coprono la maggior parte degli errori dei principianti. Le variabili rendono gli script flessibili: definisci un valore in cima (es. una cartella) e lo riusi ovunque, cambiandolo in un solo punto. Prova il widget qui sotto.

Sezione 12.4Leggere input e argomenti


  

    Uno script diventa davvero utile quando può ricevere dati dall'esterno, invece di avere tutto «fisso».
    Due modi: gli argomenti (dati passati al lancio) e l'input interattivo (chiesto durante l'esecuzione).
  


  

    
saluta.sh — argomenti e input

    
#!/bin/bash
# $1 è il PRIMO argomento passato allo script
echo "Ciao, $1!"

# read: chiedi un valore all'utente durante l'esecuzione
echo "Come stai?"
read RISPOSTA
echo "Hai detto: $RISPOSTA"

  


  

    
terminale — eseguirlo con un argomento

    
$ ./saluta.sh Luca       # "Luca" diventa $1
Ciao, Luca!
Come stai?
$ bene                    # (l'utente digita)
Hai detto: bene

  


  

    
Ricevere dati $1 $2 ... · read

    

      

        
gli argomenti: $1, $2, …

        
i valori passati allo script al lancio: ./script.sh ciao mondo mette «ciao» in
            $1 e «mondo» in $2. $0 è il nome dello script, $#
            il numero di argomenti, $@ tutti gli argomenti. Il modo standard di parametrizzare uno
            script.

        
l'input interattivo: read VARIABILE

        
ferma lo script e aspetta che l'utente digiti qualcosa, mettendolo nella variabile. Per
            script che «dialogano» con chi li usa.

        
quale usare

        
argomenti per dati noti al lancio (comodi per l'automazione: nessuna interazione); read
            quando serve chiedere qualcosa durante l'esecuzione. Gli argomenti rendono lo script
            «riutilizzabile» con valori diversi ogni volta.

      

    

    

      Gli argomenti ($1, $2…) sono ciò che rende uno script un vero strumento: lo
      stesso saluta.sh funziona per «Luca», «Anna», chiunque — passando un argomento diverso. È lo
      stesso principio dei parametri di una funzione (se hai fatto Python/C++): un blocco di logica riutilizzabile
      con input diversi. Nel prossimo capitolo, con condizioni e cicli, questi script diventeranno capaci di
      «ragionare».
    

  



  
  
Sezione 12.5Laboratorio: il primo script


  

    Scriviamo, rendiamo eseguibile e lanciamo un vero primo script — che usa variabili e un argomento.
  


  

    
terminale + file — dal nulla a uno script funzionante

    
# 1) crea lo script con nano (o il tuo editor)
$ nano info.sh

  


  

    
info.sh (scrivi questo dentro nano)

    
#!/bin/bash
UTENTE=$(whoami)          # $(...) cattura l'output di un comando
echo "Ciao $1, sei loggato come $UTENTE."
echo "Ti trovi in: $(pwd)"
echo "Ci sono $(ls | wc -l) elementi qui."

  


  

    
terminale — rendilo eseguibile e lancialo

    
# 2) rendilo eseguibile
$ chmod +x info.sh

# 3) lancialo passando il tuo nome come argomento
$ ./info.sh Marco
Ciao Marco, sei loggato come anna.
Ti trovi in: /home/anna
Ci sono 7 elementi qui.

  


  

    
✅ Cosa hai costruito

    

      Il tuo primo script completo: usa un argomento ($1, il nome), variabili
      (UTENTE), e — bonus — la sostituzione di comando $(...), che cattura l'output di
      un comando in una variabile o in una stringa (qui whoami, pwd, la pipe
      ls | wc -l). Nota quanto è potente $(comando): mette il risultato
      di un comando dentro il testo. Hai messo insieme tutto il libro finora — comandi, pipe, variabili — in un file
      riutilizzabile. Questo è scripting. Nel prossimo capitolo aggiungiamo il «cervello»: condizioni e
      cicli, per far prendere decisioni e ripetere azioni ai tuoi script.

Sezione 12.7Soluzioni


  

    
Soluzione 12.1

    

      Uno script bash è un file di testo che contiene una sequenza di comandi (gli stessi del terminale),
      eseguiti in ordine dalla shell. I tre passi per ciao.sh: (1) scriverlo con un editor (con
      #!/bin/bash in cima); (2) renderlo eseguibile con chmod +x ciao.sh; (3)
      lanciarlo con ./ciao.sh.
    

  


  

    
Soluzione 12.2

    

      La riga #!/bin/bash (lo shebang) dice al sistema quale interprete usare per eseguire lo
      script — qui bash. Senza, il sistema non saprebbe come farlo girare. Serve poi chmod +x per dare al
      file il permesso di esecuzione (Cap. 6): senza quel permesso, tentando di lanciarlo otterresti «Permission
      denied». In breve: lo shebang dice «come» eseguirlo, il permesso x dà il «diritto» di eseguirlo.
    

  


  

    
Soluzione 12.3

    

      L'errore sono gli spazi attorno all'=: in bash l'assegnazione non li ammette. Corretto:
      NOME="Luca" (nessuno spazio). Per stampare «Ciao Luca» usando la variabile:
      echo "Ciao $NOME" (con virgolette doppie, così $NOME viene espanso al suo valore).
    

  


  

    
Soluzione 12.4

    

      $1 contiene il primo argomento passato allo script al momento del lancio. Lanciando
      ./script.sh mela pera: $1 vale mela e $2 vale
      pera. (In più: $0 è il nome dello script, $# il numero di argomenti = 2,
      $@ tutti gli argomenti.)
