Capitolo 11

Installare software

Su Windows o macOS, per installare un programma scarichi un file dal web e ci clicchi sopra. Su Linux funziona diversamente — e molto meglio: c'è un sistema centralizzato e ordinato chiamato gestore di pacchetti. Invece di cercare installer sparsi per il web (con i rischi di sicurezza che comporta), chiedi al gestore di installare un programma dal nome, e lui lo scarica da un archivio ufficiale (il «repository»), risolve tutto ciò che serve, e lo configura — con un comando. Lo stesso gestore aggiorna e rimuove il software, tenendo il sistema pulito e sicuro. È uno dei grandi vantaggi di Linux. C'è un dettaglio: ogni famiglia di distribuzioni ha il suo gestore (il Cap. 1: è l'unica cosa che cambia davvero tra distro). Il più comune è apt (Debian/Ubuntu). In questo capitolo: cos'è un gestore di pacchetti (§11.1), apt nel dettaglio (§11.2), gli altri gestori (§11.3), e come aggiornare il sistema (§11.4). È la chiave che ti dà accesso a migliaia di programmi, in modo pulito.

🗺 Dove siamo

I gestori di pacchetti (§11.1), apt per Debian/Ubuntu (§11.2), altri gestori: dnf, pacman (§11.3), aggiornare il sistema (§11.4). Nel laboratorio (§11.5) installi un programma.

🔑 Perché ci tocca: il software, in modo pulito e sicuro

Installare software è un'operazione costante: strumenti, librerie, aggiornamenti. Farlo col gestore di pacchetti — invece di scaricare eseguibili a caso — è più semplice, più pulito e molto più sicuro (i pacchetti vengono da fonti verificate). Ed è indispensabile sui server, dove installi e aggiorni tutto da terminale. Un concetto semplice con un impatto enorme sulla sicurezza e sull'ordine del sistema.

Sezione 11.1I gestori di pacchetti

Un gestore di pacchetti è un sistema che installa, aggiorna e rimuove software in modo centralizzato, attingendo a repository ufficiali: archivi online di programmi verificati e pronti all'uso.

Come funziona pacchetto · repository · dipendenze
un pacchetto
un programma (o una libreria) confezionato per l'installazione, con un nome (es. htop, git). Chiedi il pacchetto per nome, non cerchi un file da scaricare.
i repository
gli archivi online, ufficiali e curati, da cui il gestore scarica i pacchetti. Fonte verificata e sicura: non installi «roba a caso» presa dal web, ma software controllato dalla tua distribuzione.
le dipendenze, risolte in automatico
un programma spesso ne richiede altri per funzionare. Il gestore li installa automaticamente per te (come pip per Python, se hai fatto quel libro): tu chiedi htop, lui prende tutto ciò che serve.
i vantaggi in una riga
un comando per installare, aggiornare, rimuovere; software sicuro (fonti ufficiali); dipendenze automatiche; sistema pulito (nessun file «orfano» sparso). Molto meglio del «scarica ed esegui».
Il concetto è universale, ma il comando cambia con la distribuzione (Cap. 1): le distro basate su Debian/Ubuntu usano apt; quelle basate su Fedora/Red Hat usano dnf; Arch usa pacman. Imparato uno, gli altri sono varianti dello stesso schema (installa, rimuovi, aggiorna, cerca): cambiano le parole, non l'idea. Ci concentriamo su apt, il più diffuso, e vediamo la corrispondenza con gli altri.

Sezione 11.2apt (Debian/Ubuntu)

apt è il gestore di pacchetti di Debian, Ubuntu e delle loro derivate — le distribuzioni più comuni, specie sui server. Poche operazioni coprono quasi tutto. (Sono azioni di sistema, quindi con sudo, Cap. 7.)

terminale — le operazioni di apt
# aggiorna l'elenco dei pacchetti disponibili (fallo PRIMA di installare)
$ sudo apt update

# installa un programma (per nome)
$ sudo apt install htop

# rimuovi un programma
$ sudo apt remove htop

# cerca un pacchetto (se non ricordi il nome esatto)
$ apt search editor

# aggiorna TUTTI i programmi installati alle ultime versioni
$ sudo apt upgrade
Le operazioni di apt update · install · remove · search · upgrade
apt update
aggiorna l'elenco dei pacchetti disponibili (non i programmi, l'elenco!). Va fatto prima di installare, per avere le informazioni aggiornate. Non installa nulla, solo «rinfresca» il catalogo.
apt install nome
installa il pacchetto (con le sue dipendenze). Puoi installarne più insieme: apt install git htop curl.
apt remove nome
disinstalla un programma. (apt purge rimuove anche i file di configurazione.)
apt search parola
cerca tra i pacchetti disponibili quelli che riguardano una parola. Utile quando non sai il nome esatto. (Non serve sudo: è solo una ricerca.)
apt upgrade
aggiorna tutti i programmi installati alle ultime versioni. La coppia apt update && apt upgrade è la manutenzione classica per tenere il sistema aggiornato e sicuro.
Nota la differenza sottile ma importante tra update e upgrade: update aggiorna l'elenco (il catalogo), upgrade aggiorna i programmi. Si usano insieme, in quest'ordine: prima rinfreschi il catalogo, poi aggiorni. È una delle prime cose che si fanno su un sistema nuovo (l'hai visto nel deploy del libro «Backend»: apt update && apt upgrade).

Sezione 11.3Altri gestori: dnf, pacman

Se usi una distribuzione non basata su Debian, il gestore cambia — ma le operazioni sono le stesse, con parole diverse. Ecco la corrispondenza per le altre due famiglie principali.

Lo stesso concetto, tre dialetti apt · dnf · pacman
la tabella di corrispondenza
operazione        apt (Debian/Ubuntu)   dnf (Fedora)       pacman (Arch)
-----------       -------------------   ---------------    -------------
installa          apt install X         dnf install X      pacman -S X
rimuovi           apt remove X          dnf remove X       pacman -R X
aggiorna elenco   apt update            (automatico)       pacman -Sy
aggiorna tutto    apt upgrade           dnf upgrade        pacman -Su
cerca             apt search X          dnf search X       pacman -Ss X
dnf (Fedora, Red Hat, CentOS)
molto simile ad apt nella logica e nei nomi (install, remove, upgrade). Se sai apt, dnf è quasi immediato.
pacman (Arch e derivate)
usa opzioni con lettere (-S installa, -R rimuove). Più criptico ma potente. La logica è la stessa: installa/rimuovi/aggiorna/cerca.
come sapere quale hai?
dipende dalla tua distribuzione. Se non sai quale distro usi: prova cat /etc/os-release (lo mostra), o semplicemente prova il comando (se apt non esiste, probabilmente hai dnf o pacman).
Il messaggio da portare a casa: non «imparare tutti i gestori», ma capire il modello (installa/rimuovi/aggiorna/cerca da repository ufficiali) e sapere che ogni distro ha il suo comando. Quando ti trovi su un sistema nuovo, chiedi (o cerca) «qual è il gestore di pacchetti?» e applichi lo schema che già conosci. La tabella qui sopra è un utile promemoria. Sezione 11.4Aggiornare il sistema

Tenere il software aggiornato non è solo per avere le ultime funzionalità: è soprattutto una questione di sicurezza. Gli aggiornamenti correggono vulnerabilità, e un sistema non aggiornato è un bersaglio.

🔒 Aggiornare = mettere in sicurezza
terminale — la manutenzione classica (Debian/Ubuntu)
# rinfresca l'elenco, poi aggiorna tutti i programmi
$ sudo apt update && sudo apt upgrade

Questo comando (update per aggiornare il catalogo, poi upgrade per aggiornare i programmi) è la manutenzione di base di ogni sistema Linux. Gli aggiornamenti includono le patch di sicurezza: una volta che una vulnerabilità è nota, chi non aggiorna resta esposto (l'hai visto nel libro «Backend», parlando di sicurezza dei server). Farlo con regolarità — sul tuo PC e ancor più sui server — è una delle abitudini più importanti. Su un server si possono anche configurare aggiornamenti automatici per le patch di sicurezza critiche. Ricorda: un software che non si aggiorna diventa, col tempo, vulnerabile.

Sezione 11.5Laboratorio: installare un programma

Installiamo htop (il monitor dei processi «colorato» del Cap. 8) e vediamo il gestore di pacchetti all'opera. (Serve una distribuzione Debian/Ubuntu; adatta il comando alla tua se diversa.)

terminale — installare htop (Debian/Ubuntu)
# 1) rinfresca l'elenco dei pacchetti
$ sudo apt update

# 2) installa htop
$ sudo apt install htop
... htop installato con successo

# 3) usalo! (monitor dei processi migliorato)
$ htop          # q per uscire

# 4) (facoltativo) rimuovilo, se vuoi
$ sudo apt remove htop
✅ Cosa hai imparato a fare

Hai installato un programma dal gestore di pacchetti (con update + install), lo hai usato, e sai come rimuoverlo. Nota quanto è pulito e semplice: nessun file da scaricare a mano, nessun installer, nessuna dipendenza da cercare — solo il nome del programma. Questo è il modo «Linux» di gestire il software, ed è uno dei suoi punti di forza. Con questo, hai completato gli strumenti di base per usare e amministrare un sistema: navigare, gestire file e permessi, processi, comporre comandi, installare software. Nel prossimo capitolo facciamo il grande salto verso l'automazione: scrivere i tuoi script per far eseguire al computer sequenze di comandi da solo.

🔑 Quello che devi portarti dietro dal Capitolo 11
  1. Un gestore di pacchetti installa/aggiorna/rimuove software da repository ufficiali (sicuri), risolvendo le dipendenze in automatico. Meglio del «scarica ed esegui» (§11.1).
  2. apt (Debian/Ubuntu): update (rinfresca elenco), install, remove, search, upgrade. Con sudo (§11.2).
  3. Altri gestori: dnf (Fedora, simile ad apt), pacman (Arch, con -S/-R). Stesso schema, parole diverse (§11.3).
  4. Aggiornare è sicurezza: sudo apt update && sudo apt upgrade con regolarità. Le patch chiudono le vulnerabilità (§11.4).

Hai tutti gli strumenti di base di Linux. È ora del salto che rende il terminale davvero potente: l'automazione. Finora hai dato comandi uno per uno; ma puoi metterli in un file — uno script — e farli eseguire tutti insieme, ripetutamente, da soli. Nel Capitolo 12 impariamo le basi del bash scripting: cos'è uno script, come si scrive e si esegue, le variabili — i primi passi per far lavorare il computer al posto tuo.

Sezione 11.7Soluzioni

Soluzione 11.1

Due vantaggi: (1) sicurezza — i pacchetti provengono da repository ufficiali e verificati, non da file «a caso» presi dal web (che potrebbero contenere malware); (2) dipendenze automatiche — il gestore installa da solo tutto ciò che il programma richiede, e tiene traccia di cosa è installato (così puoi rimuoverlo pulito). In più: un solo comando, aggiornamenti centralizzati, sistema ordinato.

Soluzione 11.2

apt update aggiorna l'elenco dei pacchetti disponibili (il catalogo), senza installare nulla; apt upgrade aggiorna i programmi installati alle ultime versioni. Si usano in quest'ordine: prima update (per avere il catalogo aggiornato), poi upgrade (per applicare gli aggiornamenti) — spesso in un colpo: sudo apt update && sudo apt upgrade.

Soluzione 11.3
terminale
$ sudo apt install curl
$ sudo apt remove curl

Serve sudo perché installare e rimuovere software modifica il sistema (per tutti gli utenti), quindi richiede i poteri di amministratore (Cap. 7).

Soluzione 11.4

Su una distribuzione con pacman (come Arch), l'equivalente di apt install X è sudo pacman -S X (l'opzione -S = «sync/install»). Il concetto — installare il pacchetto X dai repository — è identico; cambia solo la sintassi del comando.
