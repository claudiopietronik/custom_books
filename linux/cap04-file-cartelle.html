Capitolo 4

File e cartelle

Ora che sai orientarti nell'albero, è ora di agire su di esso: creare, copiare, spostare e cancellare file e cartelle. Sono i comandi che userai ogni giorno per organizzare il tuo lavoro, e sono pochi e logici: mkdir e touch per creare, cp e mv per copiare e spostare, rm per cancellare. C'è però una differenza importante rispetto all'interfaccia grafica, che va detta subito e con chiarezza: il terminale non ha un cestino. Quando cancelli con rm, il file sparisce davvero, senza possibilità di recupero facile. Non è un motivo per aver paura — è un motivo per capire bene cosa fai. In questo capitolo: creare con mkdir/touch (§4.1), copiare e spostare con cp/mv (§4.2), cancellare con rm (§4.3), e i comodissimi caratteri jolly per agire su tanti file insieme (§4.4). Con questi comandi diventi padrone dei tuoi file, con la responsabilità che ne consegue.

🗺 Dove siamo

Creare con mkdir/touch (§4.1), copiare/spostare con cp/mv (§4.2), cancellare con rm/rmdir (§4.3), i caratteri jolly * e ? (§4.4). Nel laboratorio (§4.5) gestisci i file.

🔑 Perché ci tocca: organizzare (e non perdere) i dati

Gestire i file è l'operazione più quotidiana su qualsiasi sistema. Farlo dal terminale è veloce e potente (specie coi jolly), ma richiede attenzione: senza cestino, un comando avventato può cancellare cose importanti. Imparare questi comandi e le loro precauzioni ti rende efficiente senza farti rischiare disastri.

Sezione 4.1Creare: mkdir, touch

Due comandi per creare: mkdir per le cartelle, touch per i file (vuoti).

terminale — creare cartelle e file
# crea una cartella
$ mkdir progetto

# crea più cartelle annidate in un colpo (-p = crea i genitori mancanti)
$ mkdir -p progetto/src/utils

# crea un file vuoto
$ touch note.txt

# crea più file insieme
$ touch a.txt b.txt c.txt
Creare, smontato mkdir · mkdir -p · touch
mkdir nome — «make directory»
crea una cartella con quel nome nella posizione corrente (o al percorso indicato). Errore se esiste già (te lo dice).
mkdir -p a/b/c
l'opzione -p («parents») crea anche le cartelle intermedie mancanti. Senza -p, se a/b non esiste, mkdir a/b/c fallisce.
touch nome
crea un file vuoto con quel nome. Utile per «preparare» un file che riempirai dopo (con un editor, Cap. 10). (Tecnicamente touch aggiorna anche la data di un file esistente, ma l'uso comune è creare file vuoti.)
Nota che puoi passare più nomi a touch (e a molti comandi): crea tutti i file elencati. Questa capacità di agire su più cose in un colpo — potenziata dai jolly (§4.4) — è una delle ragioni per cui il terminale è più veloce dei clic. Suggerimento: usa il Tab (Cap. 3) per completare i nomi ed evitare errori di battitura.

Sezione 4.2Copiare e spostare: cp, mv

cp copia (l'originale resta), mv sposta (l'originale sparisce dalla vecchia posizione). Entrambi prendono una sorgente e una destinazione.

terminale — copiare e spostare
# COPIA un file (l'originale resta)
$ cp note.txt note-backup.txt

# copia una CARTELLA (serve -r, "ricorsivo", per il contenuto)
$ cp -r progetto progetto-copia

# SPOSTA un file in un'altra cartella
$ mv note.txt documenti/

# mv serve anche per RINOMINARE (spostare = cambiare nome/posizione)
$ mv vecchio.txt nuovo.txt
cp e mv smontati sorgente → destinazione
cp sorgente destinazione
copia: crea un duplicato. L'originale resta. Per copiare una cartella col suo contenuto serve -r (ricorsivo).
mv sorgente destinazione
sposta: l'originale sparisce dalla vecchia posizione e compare nella nuova. Non serve -r per le cartelle (spostare è un'operazione «leggera»).
mv = anche rinominare
«spostare» un file allo stesso posto con un nome diverso = rinominarlo. In Linux non c'è un comando «rename»: si usa mv. mv vecchio.txt nuovo.txt rinomina.
la destinazione conta
se la destinazione è una cartella esistente, il file ci finisce dentro; se è un nome nuovo, il file viene copiato/spostato con quel nome. Attenzione: se la destinazione è un file esistente, viene sovrascritto (vedi trappola sotto).
⚠ cp e mv sovrascrivono senza chiedere

Se la destinazione esiste già, cp e mv la sovrascrivono silenziosamente, senza avviso. Un file importante può sparire così. Precauzione: usa l'opzione -i («interactive») — cp -i, mv -i — che chiede conferma prima di sovrascrivere. Prendi l'abitudine di controllare con ls cosa c'è nella destinazione prima di un'operazione importante.

Sezione 4.3Cancellare: rm, rmdir

rm cancella file, rmdir cancella cartelle vuote. È qui che serve la massima attenzione: nel terminale, cancellare è definitivo.

terminale — cancellare (con cautela)
# cancella un file
$ rm note.txt

# cancella una cartella e TUTTO il suo contenuto (-r = ricorsivo)
$ rm -r vecchia-cartella

# chiedi conferma prima di ogni cancellazione (-i)
$ rm -i importante.txt
rm: remove regular file 'importante.txt'? 

# cancella una cartella VUOTA
$ rmdir cartella-vuota
⚠ rm non ha cestino: cancella per sempre

Quando cancelli con rm, il file non va nel cestino: sparisce, e recuperarlo è difficile o impossibile. Non c'è un «annulla». Regole di sicurezza: (1) controlla sempre cosa stai per cancellare (con ls); (2) usa rm -i per farti chiedere conferma su cose importanti; (3) temi come la peste il comando rm -rf (ricorsivo + forzato, senza domande): cancella tutto ciò che indichi, in silenzio. In particolare, rm -rf / o percorsi sbagliati con i jolly possono distruggere il sistema. È uno dei modi più famosi di «farsi male» con Linux: pensa due volte prima di premere Invio su un rm.

Sezione 4.4I caratteri jolly (*, ?)

Ecco un superpotere che rende il terminale molto più veloce dei clic: i caratteri jolly (wildcard). Ti permettono di indicare molti file insieme con un solo comando, in base a uno schema.

I jolly principali * (qualsiasi) · ? (un carattere)
* — qualsiasi sequenza di caratteri
*.txt = «tutti i file che finiscono in .txt»; foto* = «tutto ciò che inizia con foto»; * da solo = «tutto». Il jolly più usato.
? — un singolo carattere qualsiasi
file?.txt = file1.txt, fileA.txt… (un carattere al posto del ?), ma non file10.txt (sono due caratteri). Più preciso di *.
si combinano coi comandi
ls *.jpg (elenca tutte le immagini), cp *.txt backup/ (copia tutti i .txt), rm temp* (cancella tutto ciò che inizia con temp). Un comando, tanti file.
terminale — i jolly in azione
$ ls *.txt            # tutti i file .txt della cartella
note.txt  todo.txt  lista.txt
$ cp *.jpg foto/       # copia tutte le immagini jpg in foto/
$ rm temp?.log        # cancella temp1.log, temp2.log... (un carattere)
⚠ I jolly + rm = molta potenza, molto rischio

I jolly sono comodissimi, ma con rm vanno maneggiati con cura: rm * cancella tutti i file della cartella; un * messo dove non doveva può cancellare più del previsto. Trucco di sicurezza: prova prima con ls (es. ls temp*) per vedere esattamente quali file corrisponderebbero, e solo dopo, se sono quelli giusti, sostituisci ls con rm. Così «guardi prima di sparare».

Sezione 4.5Laboratorio: gestire i file


  

    Un giro completo di creazione, organizzazione e pulizia — in una cartella di prova, così puoi sperimentare senza
    rischi.
  


  

    
terminale — un giro sicuro (in una cartella di prova)

    
# crea una cartella di prova ed entra (così non tocchi altro)
$ mkdir prova && cd prova

# crea alcuni file e una struttura
$ touch uno.txt due.txt tre.txt
$ mkdir backup

# copia tutti i .txt nella cartella backup (jolly!)
$ cp *.txt backup/
$ ls backup
uno.txt  due.txt  tre.txt

# rinomina e cancella (con cautela)
$ mv uno.txt primo.txt
$ ls *.txt         # GUARDA prima cosa c'è
$ rm due.txt          # poi cancella con consapevolezza

# pulizia finale: torna su e cancella tutta la cartella di prova
$ cd ..
$ rm -r prova          # ricorsivo: cartella + contenuto

  


  

    
✅ Cosa hai praticato

    

      Hai fatto il giro completo: creato (mkdir, touch), copiato con i jolly
      (cp *.txt), rinominato/spostato (mv) e cancellato (rm,
      rm -r). Nota la buona abitudine: prima di rm, hai guardato con
      ls cosa c'era. E hai lavorato in una cartella di prova, così nessun errore poteva far danni
      veri — un'ottima strategia quando impari. Ora sei padrone dei tuoi file. Nel prossimo capitolo impareremo a
      guardarne il contenuto e a cercare testo e file — perché organizzare i file è metà del lavoro;
      l'altra metà è ritrovarci dentro le informazioni.

Sezione 4.7Soluzioni


  

    
Soluzione 4.1

    

      
terminale

      
$ mkdir lavoro
$ mkdir -p lavoro/2026/gennaio

    

    
In realtà basta il secondo comando: mkdir -p crea anche
      lavoro e 2026 se non esistono. L'opzione -p crea tutta la catena di
      cartelle mancanti.

  


  

    
Soluzione 4.2

    

      cp copia (crea un duplicato, l'originale resta); mv sposta (l'originale
      sparisce dalla vecchia posizione). Per rinominare si usa mv: rinominare equivale a
      «spostare» il file nella stessa cartella con un nome diverso (mv vecchio.txt nuovo.txt). In Linux
      non serve un comando «rename» separato perché cambiare nome e cambiare posizione sono, concettualmente, la
      stessa operazione — entrambe gestite da mv.
    

  


  

    
Soluzione 4.3

    

      Perché rm non usa un cestino: cancella in modo definitivo, senza possibilità di
      «annulla» facile. Due precauzioni: (1) guardare sempre con ls cosa si sta per cancellare
      (specie coi jolly) prima di eseguire; (2) usare rm -i, che chiede conferma prima di
      ogni cancellazione. In generale, evitare rm -rf se non si è assolutamente certi.
    

  


  

    
Soluzione 4.4

    

      (a) *.log: tutti i file che finiscono in .log. (b) report?.pdf: i file
      tipo report1.pdf, reportA.pdf (un singolo carattere al posto del ?),
      ma non report10.pdf. (c) foto*: tutto ciò che inizia con foto. Conviene
      provare prima con ls perché così vedi esattamente quali file corrisponderebbero: se sono
      quelli giusti, sostituisci ls con rm; eviti così di cancellare per errore più file del
      previsto.
