Capitolo 15

La libreria standard, in pratica

Python arriva «con le batterie incluse»: una vasta collezione di moduli già pronti che risolvono al volo problemi ricorrenti, senza installare nulla. Non serve conoscerli a memoria — serve sapere che esistono e cosa cercare. In questo capitolo facciamo una passeggiata tra i più utili: gestire date e orari, generare numeri casuali, fare matematica, usare collezioni specializzate, e persino parlare col web con poche righe. È il capitolo che ti fa scoprire quanto lavoro puoi evitare: prima di scrivere una funzione complicata, quasi sempre esiste già uno strumento pronto che fa il lavoro per te.

🗺 Dove siamo

datetime per date e orari (§15.1), random per la casualità (§15.2), math per la matematica (§15.3), collections per collezioni comode (§15.4), e requests per il web (§15.5). Nel laboratorio (§15.6) costruisci un piccolo strumento utile combinandoli.

🔑 Perché ci tocca: la regola del «esiste già?»

Il segno di un programmatore maturo non è scrivere tanto codice, ma scriverne poco riusando ciò che c'è. Ogni riga che non scrivi è una riga che non può avere bug. Conoscere la libreria standard ti fa risolvere in due righe cose che altrimenti costerebbero un pomeriggio — e ti insegna dove cercare per tutto il resto. È anche un ponte verso l'ecosistema esterno: gli stessi meccanismi (import, funzioni, oggetti) valgono per i pacchetti che userai nel libro di Backend Development.

Sezione 15.1Date e tempo (datetime)

Lavorare con date e orari a mano è un incubo (mesi di lunghezza diversa, anni bisestili…). Il modulo datetime fa tutto per te.

Il modulo datetime oggi, differenze, formattazione
esempio.py
from datetime import date, timedelta

oggi = date.today()
print(oggi)                 # 2025-07-19

tra_una_settimana = oggi + timedelta(days=7)   # fai i conti sulle date!
print(tra_una_settimana)

compleanno = date(2025, 12, 25)
mancano = compleanno - oggi       # differenza tra date
print(f"Mancano {mancano.days} giorni")
Le operazioni chiave: date.today() (oggi), timedelta (una «durata» da sommare/ sottrarre alle date), e la sottrazione tra date che dà i giorni di differenza. Non devi mai più preoccuparti di quanti giorni ha febbraio: datetime conosce il calendario. Per data e ora insieme c'è datetime.now(), e per formattare le date come vuoi c'è .strftime().

Sezione 15.2Numeri casuali (random)

Per giochi, simulazioni, scelte casuali: il modulo random. Poche funzioni coprono quasi ogni esigenza di casualità.

Il modulo random dadi, scelte, mescolate
esempio.py
import random

print(random.randint(1, 6))             # un dado: intero a caso 1-6
print(random.choice(["testa", "croce"]))   # un elemento a caso

mazzo = [1, 2, 3, 4, 5]
random.shuffle(mazzo)                # mescola la lista SUL POSTO
print(mazzo)                          # es. [3, 1, 5, 2, 4]
Le più usate: randint(a, b) (intero a caso, estremi inclusi), choice(lista) (un elemento a caso), shuffle(lista) (mescola sul posto), random() (decimale tra 0 e 1), sample(lista, k) (k elementi distinti a caso). Prova il widget qui sotto: lancia un dado tante volte e guarda le frequenze livellarsi — la casualità nel breve è imprevedibile, ma su tanti lanci si distribuisce equamente (è la legge dei grandi numeri, dal libro di Statistica!).
random.randint(1, 6): il dado, tante volte lancia e guarda le frequenze

Sezione 15.3Matematica (math)

Oltre agli operatori base (Cap. 3), il modulo math offre le funzioni matematiche «serie»: radici, arrotondamenti, costanti, trigonometria.

Il modulo math le funzioni matematiche
Shell interattiva
import math
>>> math.sqrt(144)     12.0       # radice quadrata
>>> math.floor(3.7)     3          # arrotonda per DIFETTO
>>> math.ceil(3.2)      4          # arrotonda per ECCESSO
>>> math.pi            3.14159...  # la costante π
Utili: sqrt (radice), floor/ceil (arrotondamenti in giù/su), pi ed e (costanti), factorial, le trigonometriche (sin/cos/tan). Nota: per arrotondare «normalmente» c'è la funzione base round(x, n) (senza import), mentre floor/ceil forzano giù/su. Se un giorno ti serviranno array e calcolo scientifico veloce, il pacchetto esterno numpy (pip install) è lo standard — ma per la matematica quotidiana, math basta e avanza.

Sezione 15.4Collezioni utili (collections)

Il modulo collections offre versioni «potenziate» delle strutture dati. La più utile per iniziare: Counter, che conta le occorrenze in un colpo solo.

Counter contare, senza sforzo
esempio.py
from collections import Counter

voti = ["A", "B", "A", "C", "A", "B"]
conteggio = Counter(voti)
print(conteggio)              # Counter({'A': 3, 'B': 2, 'C': 1})
print(conteggio.most_common(1))  # [('A', 3)]  → il più frequente
Counter conta automaticamente quante volte compare ogni elemento — un'operazione comunissima (parole in un testo, voti, click) che a mano richiederebbe un ciclo e un dizionario con .get. Con most_common(n) ottieni i più frequenti, già ordinati. Altri utili in collections: defaultdict (un dizionario che crea da solo i valori mancanti) e deque (una coda efficiente). Sono esattamente il tipo di strumento che, una volta scoperto, ti fa dire «peccato non averlo saputo prima».

Sezione 15.5Il web dal terminale (requests)

Ricordi HTTP dal libro di Reti? Con il pacchetto requests (esterno, ma diffusissimo) puoi fare richieste HTTP in poche righe e ricevere dati da internet — spesso in JSON, che ora sai maneggiare.

Il pacchetto requests pip install requests
meteo.py — dopo: pip install requests
import requests

r = requests.get("https://api.github.com")   # una richiesta GET (Cap. 11 di Reti!)
print(r.status_code)   # 200  → il codice di stato HTTP

dati = r.json()       # il JSON diventa un dizionario Python!
print(dati["current_user_url"])
Guarda come tutto si connette: requests.get(url) fa una richiesta HTTP GET (il metodo del Cap. 11 di Reti!), r.status_code è il codice di stato (200 = ok), e r.json() trasforma la risposta JSON in un dizionario Python (Cap. 12), pronto da usare. Con questo puoi leggere dati da migliaia di servizi web (API): meteo, cambio valute, notizie… È anche l'anticamera del libro di Backend Development, dove starai dall'altra parte a rispondere a queste richieste.

Sezione 15.6Laboratorio: un piccolo strumento utile

Combiniamo alcuni moduli in uno strumentino vero: un «estrattore» che sceglie a caso dei vincitori da una lista e stampa data e ora dell'estrazione. Piccolo, ma completo.

Il programma estrazione.py random + datetime
estrazione.py
import random
from datetime import datetime

partecipanti = ["Ada", "Bea", "Cid", "Dan", "Eva"]
quanti = int(input("Quanti vincitori? "))

vincitori = random.sample(partecipanti, quanti)   # k a caso, senza ripetizioni
ora = datetime.now().strftime("%d/%m/%Y %H:%M")   # formatta data e ora

print(f"🎉 Estrazione del {ora}")
for i, nome in enumerate(vincitori, 1):   # enumerate da 1 (Cap. 14)
    print(f"{i}° premio: {nome}")
Esecuzione:
Terminale
Quanti vincitori? 2
🎉 Estrazione del 19/07/2025 14:30
1° premio: Cid
2° premio: Ada
Un solo programmino, e già tre idee del libro al lavoro: random.sample (scelte casuali senza ripetizioni), datetime con strftime (data formattata), enumerate con inizio a 1 (Cap. 14). Questo è programmare davvero: comporre strumenti pronti per risolvere un problema. Estendilo: leggi i partecipanti da un file (Cap. 12), o salva il risultato dell'estrazione in un log.
🔑 Quello che devi portarti dietro dal Capitolo 15
  1. datetime: date.today(), timedelta, differenze tra date, strftime per formattare (§15.1).
  2. random: randint, choice, shuffle, sample (§15.2). math: sqrt, floor/ceil, pi (§15.3).
  3. collections.Counter conta le occorrenze e dà i più frequenti (most_common) (§15.4).
  4. requests (esterno) fa richieste HTTP: .get(), .status_code, .json() → dizionario (§15.5). Regola d'oro: prima di scrivere, chiediti «esiste già?».

Hai tutto: le basi, lo stile e la conoscenza degli strumenti pronti. È arrivato il momento di mettere tutto insieme. Nel Capitolo 16, il gran finale, costruiremo dall'inizio alla fine un progetto completo — non un esempio isolato, ma un piccolo programma vero, dalla progettazione al codice funzionante, passando per la struttura, le prove e le rifiniture. Vedrai come tutti i pezzi che hai imparato — variabili, funzioni, file, classi, gestione errori — collaborano in un'unica applicazione. È lì che «saper programmare» diventa davvero tuo.

Sezione 15.8Soluzioni

Soluzione 15.1

mancano = date(2025, 12, 25) - date.today(), poi mancano.days. La sottrazione tra due date restituisce un oggetto timedelta (una durata), da cui leggi .days.

Soluzione 15.2

(a) random.randint(1, 6); (b) random.choice(nomi); (c) random.shuffle(mazzo) (mescola sul posto).

Soluzione 15.3

from collections import Counter poi c = Counter(parole) conta tutto in una riga. La più frequente: c.most_common(1) (restituisce una lista con la coppia (parola, conteggio) del primo).

Soluzione 15.4
moneta.py
import random
teste = 0
for _ in range(10):
    esito = random.choice(["Testa", "Croce"])
    print(esito)
    if esito == "Testa":
        teste += 1
print(f"Teste: {teste}/10")

Nota for _ in range(10): l'underscore _ è la convenzione per «una variabile del ciclo che non mi serve» (ripeto 10 volte, ma non uso il numero).
