Capitolo 9

Le funzioni

Man mano che i programmi crescono, ti accorgi di riscrivere gli stessi pezzi: la stessa validazione, lo stesso calcolo, la stessa formattazione. Le funzioni risolvono questo: danno un nome a un blocco di codice, così puoi riusarlo a comando tutte le volte che vuoi, con dati diversi. È il passo che trasforma uno «script» in un vero programma organizzato: pezzi piccoli, con un nome chiaro, che fanno una cosa sola e bene, e che componi come mattoncini. Imparare a scrivere funzioni è imparare a pensare per problemi più piccoli — la competenza che separa chi «sa buttare giù codice» da chi sa costruire programmi che si capiscono e si mantengono.

🗺 Dove siamo

Perché servono le funzioni (§9.1), come definirle con def, i parametri e il return (§9.2), gli argomenti e i valori di default (§9.3), l'ambito delle variabili (§9.4), e le docstring e buone abitudini (§9.5). Nel laboratorio (§9.6) costruisci una tua piccola libreria di funzioni.

🔑 Perché ci tocca: dividere per conquistare

Le funzioni sono lo strumento di organizzazione per eccellenza: spezzano un problema grande in pezzi piccoli e gestibili, ognuno con un nome che dice cosa fa. Rendono il codice riutilizzabile, leggibile e testabile, ed evitano di ripetere le stesse righe (il principio «non ripeterti»). Sono anche il modo in cui è fatta tutta la libreria di Python e di ogni framework: quando userai FastAPI, scriverai funzioni che rispondono alle richieste. Padroneggiarle è il vero salto verso la programmazione «adulta».

Sezione 9.1Perché le funzioni

Hai già usato funzioni degli altri: print(), len(), input(). Ora impari a crearne di tue. L'idea: dare un nome a un compito, per poterlo richiamare quando serve.

💡 L'analogia: un elettrodomestico

Pensa a una funzione come a un frullatore: ci metti dentro degli ingredienti (gli argomenti), lui fa il suo lavoro (il codice dentro), e ti restituisce un risultato (il valore di return). Non ti importa come funziona dentro: ti basta sapere cosa mettere e cosa ottieni. Puoi usarlo cento volte con ingredienti diversi, sempre con lo stesso gesto. È esattamente così che userai le tue funzioni: le definisci una volta, poi le chiami dove vuoi, dando ogni volta dati diversi. Dare un nome a un pezzo di lavoro e «premere un bottone» per eseguirlo: questa è la potenza.

Sezione 9.2def, parametri, return

Una funzione si definisce con def, riceve dei dati nei parametri, e restituisce un risultato con return. Vediamola pezzo per pezzo.

Definire e chiamare una funzione def nome(parametri): … return
esempio.py
def area_rettangolo(base, altezza):
    a = base * altezza
    return a              # restituisce il risultato a chi ha chiamato

risultato = area_rettangolo(4, 3)   # CHIAMATA: base=4, altezza=3
print(risultato)                        # 12
def nome(parametri): — la definizione
def, il nome della funzione, i parametri tra parentesi, i due punti. Sotto, il blocco indentato col codice. Definire una funzione non la esegue: la «insegna» a Python, pronta all'uso.
i parametri (base, altezza)
sono variabili «segnaposto» che riceveranno i valori al momento della chiamata. Dentro la funzione li usi come normali variabili.
return valore — il risultato
restituisce un valore a chi ha chiamato la funzione, e termina la funzione. Il valore restituito si può salvare in una variabile o usare direttamente.
la chiamata: area_rettangolo(4, 3)
qui la funzione viene eseguita: 4 va in base, 3 in altezza, e il return «diventa» 12. Gli argomenti (4, 3) sono i valori concreti; i parametri (base, altezza) i nomi interni.
Distinzione utile: una funzione con return produce un valore (come len); una senza return fa qualcosa ma restituisce None (come print, che mostra ma non «dà indietro» nulla). Se ti serve il risultato per proseguire, usa return; se la funzione deve solo «compiere un'azione» (stampare, salvare), può non averlo.
La funzione come «macchina»: input → return cambia gli argomenti e guarda il risultato

Sezione 9.3Argomenti e valori di default

Le funzioni possono ricevere gli argomenti in modi flessibili, e avere valori predefiniti per i parametri che spesso restano uguali. Piccole comodità che rendono le funzioni molto più pratiche.

Default e argomenti con nome valori predefiniti · per nome
esempio.py
def saluta(nome, saluto="Ciao"):   # saluto ha un valore di default
    return f"{saluto} {nome}!"

print(saluta("Anna"))                 # Ciao Anna!   (usa il default)
print(saluta("Marco", "Buongiorno"))   # Buongiorno Marco!
print(saluta(saluto="Salve", nome="Ada"))  # per nome: l'ordine non conta
valori di default — saluto="Ciao"
un parametro con un valore predefinito diventa opzionale: se non lo passi, usa il default. Perfetto per opzioni che raramente cambiano. I parametri con default vanno dopo quelli senza.
argomenti per nome — nome="Ada"
puoi passare gli argomenti specificando il nome del parametro: così l'ordine non conta e la chiamata è più leggibile (utile con molti parametri).
Hai già visto queste cose «da fuori»: sorted(lista, reverse=True) (Cap. 5) usa proprio un argomento con nome e valore di default! Ora capisci come funzionano le funzioni che usi ogni giorno — e puoi costruirne di altrettanto comode.

Sezione 9.4Ambito delle variabili (scope)

Le variabili create dentro una funzione vivono solo lì: nascono quando la funzione parte e spariscono quando finisce. È l'ambito (scope), e capirlo evita molta confusione.

📐 Dentro resta dentro
esempio.py
def calcola():
    segreto = 42       # variabile LOCALE: esiste solo dentro calcola()
    return segreto

calcola()
print(segreto)         # ERRORE! "segreto" non esiste qui fuori

La variabile segreto è locale: esiste solo durante l'esecuzione della funzione, e fuori non è visibile (dà NameError). È una cosa positiva: ogni funzione ha il suo «spazio di lavoro» privato, e non può rovinare per sbaglio le variabili di un'altra parte del programma. È il motivo per cui puoi riusare nomi comuni (i, totale, x) in funzioni diverse senza conflitti. Per far uscire un risultato da una funzione, si usa return: è la porta ufficiale, non le variabili interne.

Sezione 9.5Docstring e buone abitudini

Una funzione ben scritta non è solo corretta: è comprensibile. Poche abitudini fanno la differenza tra codice che si capisce a colpo d'occhio e codice che è un enigma.

📐 Le regole d'oro di una buona funzione
  • Un nome che dice cosa fa, con un verbo: calcola_media, valida_email, trova_massimo. Leggendo la chiamata si deve capire cosa succede.
  • Una funzione, un compito: se una funzione fa troppe cose, spezzala. Piccola e focalizzata è più facile da capire, riusare e correggere.
  • Una docstring per spiegarla: una stringa (spesso tripla) subito sotto il def, che descrive cosa fa la funzione.
esempio.py
def calcola_media(voti):
    """Restituisce la media di una lista di voti."""   # docstring
    return sum(voti) / len(voti)

La docstring è documentazione «viva»: gli editor la mostrano quando usi la funzione, e con help(calcola_media) la leggi dalla shell. Scrivere una riga che spiega la funzione ti costringe anche a chiarirti le idee su cosa deve fare. È un'abitudine dei professionisti che vale la pena prendere subito.

Sezione 9.6Laboratorio: una libreria di funzioni

Costruiamo alcune funzioni utili e vediamo come si combinano — una che chiama l'altra. È il modo in cui si costruiscono i programmi veri: tanti piccoli mattoni che collaborano.

Il file utili.py funzioni che collaborano
utili.py
def celsius_a_fahrenheit(c):
    """Converte gradi Celsius in Fahrenheit."""
    return c * 9 / 5 + 32

def e_febbre(temp_c):
    """True se la temperatura corporea indica febbre (> 37.5 °C)."""
    return temp_c > 37.5

def referto(temp_c):
    """Compone un referto, usando le altre due funzioni."""
    f = celsius_a_fahrenheit(temp_c)   # chiama un'altra funzione!
    stato = "FEBBRE" if e_febbre(temp_c) else "normale"
    return f"{temp_c}°C ({f}°F) → {stato}"

print(referto(38.2))   # 38.2°C (100.76°F) → FEBBRE
print(referto(36.6))   # 36.6°C (97.88°F) → normale
Guarda la composizione: referto non rifà i calcoli, chiama le altre due funzioni. Ogni funzione fa una cosa sola e bene; insieme risolvono un problema più grande. È il modo di pensare che userai sempre: scomponi il problema in pezzi, scrivi una funzione per ciascuno, poi le combini. Estendilo: aggiungi fahrenheit_a_celsius, o una funzione che dato un elenco di temperature restituisca quante indicano febbre (un ciclo + e_febbre!).
🔑 Quello che devi portarti dietro dal Capitolo 9
  1. Una funzione si definisce con def nome(parametri): e restituisce con return; definirla non la esegue, la chiamata sì (§9.2).
  2. I parametri possono avere valori di default (opzionali) e ricevere argomenti per nome (§9.3).
  3. Le variabili dentro una funzione sono locali: non esistono fuori. Per far uscire un risultato, usa return (§9.4).
  4. Buone abitudini: nome chiaro col verbo, un solo compito, una docstring. Le funzioni si compongono: una chiama l'altra (§9.5-9.6).

Sai organizzare il codice in funzioni riutilizzabili. Ma finora hai scritto tutto in un unico file, usando solo ciò che Python offre «di serie». I programmi veri sono fatti di più file e usano codice scritto da altri — migliaia di «pacchetti» pronti per fare quasi tutto, dal disegnare grafici al parlare col web. Nel Capitolo 10 impariamo i moduli e i pacchetti: come dividere il codice in più file, come usare la ricchissima libreria standard di Python, e come installare pacchetti esterni con pip dentro un ambiente virtuale. È il momento in cui il tuo Python si connette all'immenso ecosistema là fuori.

Esercizio 9.3

Scrivi saluta(nome, lingua="it") che restituisce «Ciao nome» se lingua è "it" e «Hello nome» se è "en". Mostra come chiamarla in entrambi i modi.

Esercizio 9.4

(Pratico) Scrivi e_primo(n) che restituisce True se n è primo (suggerimento: un ciclo che prova i divisori da 2 a n-1; se ne trovi uno, non è primo). Provala su 7 e su 9.

Sezione 9.8Soluzioni

Soluzione 9.1
quadrato.py
def quadrato(n):
    return n ** 2

print(quadrato(7))   # 49
Soluzione 9.2

Con return la funzione produce un valore che puoi salvare in una variabile e riusare (x = quadrato(5); print(x * 2)). Con solo print, la funzione mostra ma non «dà indietro» nulla (restituisce None): il risultato appare a schermo ma non è disponibile per altri calcoli. Per costruire programmi, serve return.

Soluzione 9.3
saluta.py
def saluta(nome, lingua="it"):
    if lingua == "en":
        return f"Hello {nome}"
    return f"Ciao {nome}"

print(saluta("Ada"))              # Ciao Ada  (default it)
print(saluta("Ada", "en"))        # Hello Ada
Soluzione 9.4
primo.py
def e_primo(n):
    if n < 2:
        return False
    for d in range(2, n):
        if n % d == 0:   # trovato un divisore
            return False
    return True

print(e_primo(7), e_primo(9))   # True False

Nota come il return False «di mezzo» esce subito appena trova un divisore: se il ciclo finisce senza trovarne, il numero è primo (return True).
