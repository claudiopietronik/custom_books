Capitolo 10

Moduli, pacchetti e ambienti

Nessun programmatore scrive tutto da zero. La forza di Python è il suo immenso ecosistema: una ricchissima libreria standard già inclusa, e centinaia di migliaia di pacchetti pronti, scritti da altri, per fare quasi tutto — grafici, siti web, intelligenza artificiale, giochi. In questo capitolo impari a importare questo codice altrui, a organizzare il tuo in più file, a installare pacchetti esterni con pip, e a tenere ordinati i progetti con gli ambienti virtuali. È il capitolo che collega il tuo piccolo Python all'oceano di strumenti là fuori — il momento in cui smetti di reinventare la ruota e cominci a costruire davvero.

🗺 Dove siamo

import per usare codice altrui (§10.1), la libreria standard «inclusa» (§10.2), pip per installare pacchetti esterni (§10.3), gli ambienti virtuali per non fare confusione (§10.4), e come creare un tuo modulo (§10.5). Nel laboratorio (§10.6) costruisci un progetto su più file.

🔑 Perché ci tocca: non reinventare la ruota

Saper usare i moduli è ciò che rende Python super-produttivo: quasi ogni problema che incontrerai ha già una soluzione in un pacchetto. Vuoi parlare col web? C'è requests. Fare grafici? matplotlib. Un sito? FastAPI (il libro dedicato!). Padroneggiare import, pip e i venv è la competenza pratica che ti apre tutto l'ecosistema — e gli ambienti virtuali, in particolare, ti eviteranno un mare di problemi quando lavorerai a più progetti.

Sezione 10.1import: usare il codice altrui

Un modulo è un file di codice Python (di solito funzioni e strumenti pronti). Con import lo «porti dentro» il tuo programma e puoi usarne le funzioni.

Le forme di import portare dentro strumenti
esempio.py
import math                 # importa tutto il modulo math
print(math.sqrt(16))       # 4.0  → si usa col prefisso: math.qualcosa
print(math.pi)             # 3.14159...

from random import randint   # importa SOLO randint da random
print(randint(1, 6))        # un numero a caso 1-6 (senza prefisso)
import math
importa l'intero modulo; per usarne le cose, le prefissi col nome: math.sqrt, math.pi. Il prefisso rende chiaro da dove viene ogni strumento.
from random import randint
importa solo un pezzo specifico, che poi usi senza prefisso (randint(...)). Comodo quando ti serve solo quello.
dove va l'import
gli import si mettono in cima al file, tutti insieme: è la convenzione, e permette di vedere a colpo d'occhio da cosa dipende il programma.
import: cosa «porti dentro» scegli una forma di import

Sezione 10.2La libreria standard

Python arriva «con le batterie incluse»: una vasta libreria standard di moduli già pronti, senza bisogno di installare nulla. Conoscerne alcuni ti fa risparmiare un sacco di lavoro.

Moduli standard utilissimi già inclusi in Python
math
funzioni matematiche: sqrt (radice), pi, floor/ceil (arrotondamenti), sin/cos…
random
casualità: randint(a, b) (intero a caso), choice(lista) (elemento a caso), shuffle (mescola). Per giochi, simulazioni, campionamenti.
datetime
date e orari: la data di oggi, differenze tra date, formattazione (Cap. 15).
os e pathlib
interagire col sistema: file, cartelle, percorsi (Cap. 12).
json
leggere e scrivere dati in formato JSON — il formato del web (Cap. 12).
Non serve conoscerli a memoria: l'importante è sapere che esistono e sapere dove cercarli (la documentazione ufficiale di Python è ottima). La regola del bravo programmatore: prima di scrivere una funzione complicata, chiediti «esiste già?» — molto spesso la risposta è sì, in un modulo standard.

Sezione 10.3pip: installare pacchetti

Oltre alla libreria standard, esiste un universo di pacchetti esterni creati dalla comunità, raccolti su PyPI (il «negozio» dei pacchetti Python). Si installano con pip, dal terminale.

Il comando pip installa dal terminale, non dalla shell Python
Terminale (non la REPL!)
C:\> pip install requests     # installa il pacchetto "requests"
C:\> pip list                  # elenca i pacchetti installati
pip install nome
scarica e installa il pacchetto da PyPI. Dopo, lo puoi importare nei tuoi programmi come un modulo qualsiasi.
attenzione: pip va nel terminale
non dentro la shell Python (>>>)! È un comando di sistema. Errore tipico del principiante: scrivere pip install dentro la REPL — dà SyntaxError.
esempi famosi
requests (parlare col web), numpy/pandas (dati), fastapi (siti), pillow (immagini). Migliaia di altri per ogni esigenza.

Sezione 10.4Ambienti virtuali (venv)

Se installi tutti i pacchetti «globalmente», progetti diversi finiscono per pestarsi i piedi (uno vuole la versione 1 di un pacchetto, un altro la 2). La soluzione professionale: un ambiente virtuale per ogni progetto.

📐 Una «scatola» pulita per progetto

Un ambiente virtuale (venv) è una cartella isolata con la sua copia di Python e i suoi pacchetti, separata dal resto del sistema. Ogni progetto ha il suo: così le dipendenze di un progetto non interferiscono con quelle di un altro. È una delle prime cose che si fanno iniziando un progetto serio.

Terminale — nella cartella del progetto
C:\progetto> python -m venv .venv     # crea l'ambiente (cartella .venv)
C:\progetto> .venv\Scripts\activate       # lo ATTIVA (Windows)
(.venv) C:\progetto> pip install requests  # ora installa QUI dentro
# su Mac/Linux si attiva con:  source .venv/bin/activate

Quando l'ambiente è attivo, vedi (.venv) all'inizio del prompt: tutto ciò che installi con pip finisce lì dentro, non nel sistema. Per uscirne: deactivate. Non preoccuparti se ora sembra macchinoso: diventerà un gesto automatico, e nei libri su server e deploy sarà pane quotidiano. La regola d'oro: un venv per progetto, attivato prima di lavorarci.

Sezione 10.5Creare un tuo modulo

Anche il tuo codice può diventare un modulo: basta metterlo in un file .py e importarlo da un altro. È così che si organizza un programma grande in più file, ognuno con il suo compito.

📐 Un file che importa un altro
matematica.py
def quadrato(n):
    return n ** 2

def cubo(n):
    return n ** 3
main.py — nella stessa cartella
from matematica import quadrato, cubo   # importa dal tuo file!

print(quadrato(5))   # 25
print(cubo(3))       # 27

from matematica import ... importa dal tuo file matematica.py (senza l'estensione), esattamente come importeresti un modulo standard. Dividere il codice in più file, per argomento, è come organizzare i documenti in cartelle: rende un progetto grande gestibile. Un programma reale ha spesso decine di file, ognuno con funzioni affini, e un main.py che li mette insieme.

Sezione 10.6Laboratorio: un progetto con più file

Costruiamo un mini-progetto diviso in due file: un modulo di utilità e un programma principale che lo usa. È lo scheletro di ogni progetto Python organizzato.

Un progetto in due file modulo + main
testo.py — il modulo di utilità
def conta_parole(frase):
    """Conta le parole in una frase."""
    return len(frase.split())

def inverti(frase):
    """Restituisce la frase con le parole in ordine inverso."""
    return " ".join(reversed(frase.split()))
main.py
import testo

frase = input("Scrivi una frase: ")
print(f"Parole: {testo.conta_parole(frase)}")
print(f"Invertita: {testo.inverti(frase)}")
Esecuzione (python main.py) con «il gatto dorme»:
Terminale
Scrivi una frase: il gatto dorme
Parole: 3
Invertita: dorme gatto il
Nota " ".join(...): il metodo join unisce una lista di parole in una stringa mettendo un separatore tra loro (l'opposto di split). Il punto chiave è l'organizzazione: la logica del testo sta in testo.py, il programma principale in main.py la usa. Aggiungi funzioni a testo.py e falle usare da main: hai la struttura di un progetto vero.
🔑 Quello che devi portarti dietro dal Capitolo 10
  1. import modulo (uso con prefisso modulo.cosa) o from modulo import cosa (senza prefisso): porti dentro codice altrui (§10.1).
  2. Python ha una ricca libreria standard «inclusa» (math, random, datetime, json, os…): prima di scrivere, chiediti «esiste già?» (§10.2).
  3. pip install nome (nel terminale!) installa pacchetti esterni da PyPI (§10.3). Un ambiente virtuale (python -m venv .venv + attivazione) isola le dipendenze di ogni progetto (§10.4).
  4. Il tuo codice diventa un modulo: mettilo in un .py e importalo — così organizzi un progetto su più file (§10.5-10.6).

Ora sai attingere all'intero ecosistema Python e organizzare progetti su più file. Ma i programmi, per quanto ben scritti, possono rompersi: un input inatteso, un file che manca, una divisione per zero. Finora un errore fermava tutto con un messaggio spaventoso. È ora di imparare a prevederli e gestirli con grazia, così che un imprevisto non faccia crollare il programma. Nel Capitolo 11 affrontiamo gli errori e le eccezioni: come leggere un messaggio d'errore (il «traceback»), e come usare try/except per rendere i tuoi programmi robusti e a prova di sorpresa.

Sezione 10.7Esercizi

Esercizio 10.1

Qual è la differenza tra import math e from math import sqrt? Come useresti sqrt(25) in ciascun caso?

Esercizio 10.2

Vuoi un numero casuale tra 1 e 100. Quale modulo importi e quale funzione usi?

Esercizio 10.3

Un amico scrive pip install requests dentro la shell Python (>>>) e ottiene un errore. Perché? Dove va scritto quel comando?

Esercizio 10.4

A cosa serve un ambiente virtuale? Perché è meglio averne uno per progetto invece di installare tutto globalmente?

Sezione 10.8Soluzioni

Soluzione 10.1

import math importa tutto il modulo: usi math.sqrt(25) (col prefisso). from math import sqrt importa solo sqrt: usi sqrt(25) (senza prefisso). Il primo tiene chiaro da dove viene la funzione; il secondo è più conciso se usi solo quella.

Soluzione 10.2

Modulo random, funzione randint: from random import randint poi randint(1, 100) (oppure import random e random.randint(1, 100)).

Soluzione 10.3

pip è un comando di sistema, va scritto nel terminale (cmd/PowerShell), non dentro la shell Python. Nella REPL, pip install requests è codice Python non valido → SyntaxError. Esci dalla REPL (exit()) e lancialo dal terminale.

Soluzione 10.4

Un ambiente virtuale isola i pacchetti (e la versione di Python) di un progetto in una cartella dedicata. Averne uno per progetto evita che le dipendenze si scontrino: un progetto può usare la versione 1 di un pacchetto e un altro la versione 2, senza conflitti. Tiene anche il sistema «pulito».
