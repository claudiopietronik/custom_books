Sezione 14.1Iterare bene (enumerate, zip)

Due funzioni che rendono i cicli più eleganti: enumerate quando ti serve anche l'indice, e zip per scorrere due liste in parallelo.

enumerate e zip indice + valore · due liste insieme
esempio.py
nomi = ["Ada", "Bea", "Cid"]

# enumerate: indice E valore insieme (invece di gestire un contatore a mano)
for i, nome in enumerate(nomi):
    print(f"{i}: {nome}")     # 0: Ada / 1: Bea / 2: Cid

# zip: scorre due liste in parallelo
voti = [7, 9, 6]
for nome, voto in zip(nomi, voti):
    print(f"{nome}: {voto}")         # Ada: 7 / Bea: 9 / Cid: 6
Senza enumerate, per avere l'indice dovresti gestire un contatore a mano (i = 0 … i += 1): brutto e soggetto a errori. Senza zip, dovresti usare gli indici per accoppiare gli elementi. Questi strumenti eliminano codice noioso e fonte di bug, esprimendo direttamente l'intenzione: «per ogni elemento con il suo indice», «per ogni coppia».

Sezione 14.2Comprehension di liste e dizionari

Il gioiello della corona. Una list comprehension costruisce una lista in una sola riga, al posto di un ciclo con append. Una volta capita, non ne farai più a meno.

Le comprehension [espressione for x in collezione]
esempio.py
# il modo "lungo" con un ciclo:
quadrati = []
for n in range(5):
    quadrati.append(n ** 2)

# la stessa cosa, in UNA riga (comprehension):
quadrati = [n ** 2 for n in range(5)]   # [0, 1, 4, 9, 16]

# con un filtro (if): solo i pari
pari = [n for n in range(10) if n % 2 == 0]  # [0, 2, 4, 6, 8]
la struttura: [espressione for x in collezione]
si legge «l'espressione, per ogni x nella collezione». Costruisce una nuova lista applicando l'espressione a ogni elemento. È un ciclo + append condensati in una riga.
con filtro: [x for x in coll if condizione]
l'if finale seleziona quali elementi includere. «Prendi x, per ogni x, se la condizione è vera».
anche dict comprehension
{n: n**2 for n in range(4)} costruisce un dizionario allo stesso modo: {0:0, 1:1, 2:4, 3:9}.
Da ciclo a comprehension confronta le due forme e il risultato
⚠ Con moderazione

Le comprehension sono bellissime, ma se diventano troppo complesse (filtri multipli, cicli annidati, espressioni contorte) perdono il loro pregio: la leggibilità. Regola pratica: se una comprehension non si legge in un colpo d'occhio, torna al buon vecchio ciclo — più lungo ma chiaro. L'eleganza serve la leggibilità, non il contrario. Una riga «furba» ma indecifrabile è peggio di tre righe ovvie.

Sezione 14.3Generatori e yield

E se dovessi elaborare milioni di elementi, o una sequenza infinita? Costruire una lista enorme sprecherebbe memoria. I generatori producono i valori uno alla volta, su richiesta: memoria minima, potenza massima.

📐 Produrre a richiesta con yield
esempio.py
def conta_fino_a(n):
    i = 1
    while i <= n:
        yield i       # "restituisce" i, ma si mette in PAUSA (non finisce!)
        i += 1

for numero in conta_fino_a(5):   # produce 1,2,3,4,5 uno alla volta
    print(numero)

Un generatore è una funzione che usa yield invece di return. La differenza cruciale: yield «restituisce» un valore ma mette in pausa la funzione, che riprende da dove era rimasta quando le si chiede il prossimo valore. Così i valori si producono pigramente, uno per volta, senza mai costruire l'intera sequenza in memoria. Perfetto per file enormi, flussi di dati, sequenze lunghissime. Esiste anche la forma compatta, come una comprehension ma con le tonde: (n**2 for n in range(1000000)) — un generatore che non occupa memoria finché non lo scorri.

Sezione 14.4Funzioni come valori (lambda, map)

In Python le funzioni sono valori come gli altri: puoi metterle in variabili, passarle ad altre funzioni. Da qui le lambda (mini-funzioni al volo) e strumenti come map e sorted con chiave.

lambda e funzioni-argomento funzioni usa e getta
esempio.py
# una lambda: una funzione anonima, in una riga
doppio = lambda x: x * 2
print(doppio(5))          # 10   (equivale a def doppio(x): return x*2)

# passare una funzione come "chiave" per ordinare
parole = ["casa", "a", "albero", "bo"]
print(sorted(parole, key=len))   # ordina per LUNGHEZZA: ['a','bo','casa','albero']
Una lambda è una funzione «usa e getta» senza nome, comoda quando serve una funzioncina al volo, spesso come argomento di un'altra. L'esempio più utile è sorted(..., key=...): passi una funzione che dice secondo cosa ordinare. sorted(persone, key=lambda p: p["eta"]) ordina una lista di dizionari per età. Il fatto che le funzioni siano valori è ciò che rende Python così flessibile — un'idea potente che ritroverai in molti framework.

Sezione 14.5Lo stile «pythonico»

«Pythonico» descrive il codice che sfrutta lo spirito del linguaggio: leggibile, semplice, esplicito. Non è solo estetica — è una filosofia che rende il codice migliore.

📐 Lo Zen di Python (l'essenziale)

Python ha un «manifesto» (scrivi import this nella REPL per leggerlo tutto!). I principi chiave:

  • «Il bello è meglio del brutto» e «il leggibile conta»: il codice si legge più di quanto si scriva.
  • «L'esplicito è meglio dell'implicito»: meglio chiaro che «furbo». Nomi parlanti, niente magie nascoste.
  • «Il semplice è meglio del complesso»: la soluzione più semplice che funziona, di solito, è la migliore.
  • «Dovrebbe esserci un modo ovvio per farlo»: Python predilige una via chiara e canonica.

In pratica: nomi chiari, funzioni piccole, poche righe che fanno molto senza trucchi, e sempre la domanda «un altro capirà questo codice tra sei mesi?». Gli idiomi di questo capitolo (comprehension, enumerate/zip) sono pythonici perché esprimono l'intenzione in modo diretto e leggibile — non perché sono corti.

Sezione 14.6Laboratorio: riscrivere in bello

Il modo migliore per assimilare lo stile pythonico è rifattorizzare: prendere codice funzionante ma «goffo» e riscriverlo elegante. Vediamo un esempio completo.

Prima e dopo stesso risultato, stile diverso
prima.py — funziona, ma goffo
voti = [7, 4, 9, 5, 8]
promossi = []
for i in range(len(voti)):
    if voti[i] >= 6:
        promossi.append(voti[i])
totale = 0
for v in promossi:
    totale = totale + v
print(totale / len(promossi))
dopo.py — pythonico
voti = [7, 4, 9, 5, 8]
promossi = [v for v in voti if v >= 6]   # comprehension con filtro
print(sum(promossi) / len(promossi))         # sum() invece del ciclo-accumulatore
Da 9 righe a 3, e molto più chiare: la comprehension esprime «i voti ≥ 6», sum() evita l'accumulatore a mano, e si scorre la lista direttamente invece che per indice. Stesso risultato, ma il «dopo» dice cosa fa a colpo d'occhio. Questo è lo stile pythonico: non trucchi per accorciare, ma esprimere l'intenzione nel modo più diretto. Prova tu: prendi un tuo vecchio programmino e cerca cicli con append da trasformare in comprehension, o accumulatori sostituibili con sum/max/min.

Sezione 14.7Esercizi

Esercizio 14.1

Riscrivi con una comprehension: un ciclo che costruisce la lista dei cubi dei numeri da 1 a 5 ([1, 8, 27, 64, 125]).

Esercizio 14.2

Data parole = ["ciao", "a", "mondo", "py"], scrivi una comprehension che tenga solo le parole di almeno 3 lettere.

Esercizio 14.3

Quando è meglio un generatore rispetto a una lista? Cosa fa yield di diverso da return?

Esercizio 14.4

(Pratico) Data persone = [{"nome": "Bea", "eta": 30}, {"nome": "Ada", "eta": 25}], ordinala per età usando sorted con key.

Sezione 14.8Soluzioni

Soluzione 14.1

cubi = [n ** 3 for n in range(1, 6)] → [1, 8, 27, 64, 125].

Soluzione 14.2

[p for p in parole if len(p) >= 3] → ['ciao', 'mondo'].

Soluzione 14.3

Un generatore è meglio quando i dati sono tanti (o infiniti) e non vuoi tenerli tutti in memoria: produce un valore alla volta, su richiesta. yield restituisce un valore ma mette in pausa la funzione (che riprende al prossimo valore), mentre return la termina restituendo un solo risultato.

Soluzione 14.4

sorted(persone, key=lambda p: p["eta"]) → prima Ada (25), poi Bea (30). La lambda dice «ordina guardando il valore della chiave 'eta' di ogni dizionario».
