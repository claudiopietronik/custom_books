Capitolo 12

File e dati

Finora tutto ciò che i tuoi programmi elaboravano svaniva alla chiusura: nessuna memoria oltre la singola esecuzione. Ma un'app che dimentica tutto ogni volta è di poca utilità. Serve salvare i dati su disco e ritrovarli dopo — dare ai programmi una memoria che dura. In questo capitolo impari a leggere e scrivere file, a gestire percorsi e cartelle, e a usare i due formati che incontrerai ovunque: CSV (dati a tabella, come un foglio Excel) e JSON (dati strutturati, la lingua con cui il web scambia informazioni). È il capitolo che rende i tuoi programmi capaci di ricordare — e di dialogare col resto del mondo digitale.

Sezione 12.1Leggere e scrivere file di testo

Un file di testo è una sequenza di caratteri salvata su disco. Python lo apre con open, poi puoi leggerlo o scriverci. La cosa da capire subito è la «modalità» di apertura.

open, modalità, read/write "r" leggi · "w" scrivi · "a" aggiungi
esempio.py
# SCRIVERE (modalità "w": crea/sovrascrive il file)
f = open("note.txt", "w")
f.write("Prima riga\n")   # \n = a capo
f.write("Seconda riga\n")
f.close()               # IMPORTANTE: chiudere il file

# LEGGERE (modalità "r", quella di default)
f = open("note.txt", "r")
contenuto = f.read()   # tutto il file in una stringa
f.close()
print(contenuto)
le modalità
"r" leggere (default), "w" scrivere (attenzione: cancella il contenuto esistente!), "a" aggiungere in fondo senza cancellare. Scegli con cura: "w" su un file pieno lo svuota.
.write(testo) e .read()
write scrive una stringa (il \n manda a capo); read restituisce tutto il contenuto come un'unica stringa. Per leggere riga per riga si usa spesso un ciclo (§12.2).
.close()
chiude il file, «liberandolo». Dimenticarlo può causare dati non salvati o file bloccati — ed è proprio il problema che il prossimo with risolve elegantemente.

Sezione 12.2with: aprire in sicurezza

Chiudere il file a mano è facile da dimenticare, specie se in mezzo c'è un errore. Python offre il costrutto with, che chiude automaticamente il file quando hai finito. È il modo giusto, sempre.

Il costrutto with apre, usa, chiude da solo
esempio.py
# SCRIVERE con with: niente close() da ricordare
with open("note.txt", "w") as f:
    f.write("Ciao mondo\n")
# qui fuori il file è GIÀ chiuso automaticamente

# LEGGERE riga per riga (il modo più comune)
with open("note.txt") as f:
    for riga in f:              # scorre una riga alla volta
        print(riga.strip())      # strip() toglie il \n finale
with open(...) as f:
apre il file e lo lega alla variabile f per la durata del blocco indentato. Alla fine del blocco — o anche se c'è un errore — il file viene chiuso automaticamente. Non serve (e non si deve) chiamare close().
leggere con un ciclo
for riga in f: è il modo più efficiente di leggere un file: una riga alla volta, senza caricarlo tutto in memoria (fondamentale per file enormi). Il .strip() pulisce il fastidioso \n a fine riga.
Regola: usa sempre with per aprire i file. È più corto, più sicuro, e ti libera dal ricordare la chiusura. Il vecchio stile con open/close manuale lo trovi in codice datato, ma per il tuo codice: with e basta.

Sezione 12.3Percorsi e cartelle (pathlib)

Dove sta un file? In quale cartella? Gestire i percorsi in modo pulito (e che funzioni su Windows, Mac e Linux) è compito del modulo pathlib, moderno ed elegante.

📐 Path: percorsi che funzionano ovunque
esempio.py
from pathlib import Path

cartella = Path("dati")
file = cartella / "note.txt"    # la "/" costruisce il percorso!

print(file.exists())            # il file c'è?
cartella.mkdir(exist_ok=True)   # crea la cartella (ok se esiste già)
file.write_text("contenuto")     # scorciatoia: scrivi tutto in un colpo

Con pathlib, un percorso è un oggetto Path e la barra / serve a comporre i percorsi in modo naturale (cartella / "file.txt"). Il grande vantaggio: funziona identico su Windows (che usa \) e su Mac/Linux (che usano /) — pathlib traduce da solo. Metodi utili: .exists(), .mkdir(), .read_text() / .write_text() (scorciatoie che aprono e chiudono da sole), .glob("*.txt") (trova i file per pattern). È il modo moderno e pulito di gestire file e cartelle.

Sezione 12.4CSV: dati a tabella

Il CSV (Comma-Separated Values) è il formato più semplice per i dati «a tabella»: righe e colonne, come un foglio Excel. Ogni riga è una linea di testo, i valori separati da virgole.

Un file CSV righe e colonne, separati da virgole
persone.csv
nome,eta,citta
Anna,25,Torino
Marco,30,Milano
leggi.py — col modulo csv
import csv

with open("persone.csv") as f:
    lettore = csv.DictReader(f)   # ogni riga → un dizionario!
    for riga in lettore:
        print(riga["nome"], "ha", riga["eta"], "anni")
# → Anna ha 25 anni
# → Marco ha 30 anni
Il modulo csv della libreria standard gestisce i dettagli noiosi (virgole, virgolette, a capo). DictReader è particolarmente comodo: trasforma ogni riga in un dizionario, usando la prima riga (l'intestazione) come chiavi — così accedi ai valori per nome di colonna (riga["nome"]) invece che per posizione. È il formato ideale per esportare/importare dati da e verso fogli di calcolo.

Sezione 12.5JSON: dati strutturati

Il JSON è il re dei formati per dati strutturati e annidati — ed è, in pratica, dizionari e liste scritti come testo. È la lingua con cui web, app e database si scambiano dati. Lo riconoscerai subito.

JSON ⇄ Python dict/list ⇄ testo
esempio.py
import json

persona = {"nome": "Anna", "eta": 25, "hobby": ["musica", "trekking"]}

# SALVARE un dizionario in un file JSON
with open("persona.json", "w") as f:
    json.dump(persona, f, indent=2)   # dict → testo JSON nel file

# CARICARE indietro (il testo JSON ridiventa un dizionario)
with open("persona.json") as f:
    dati = json.load(f)
print(dati["nome"])            # Anna  → di nuovo un dizionario Python!
json.dump(oggetto, file)
scrive un oggetto Python (dizionario, lista…) nel file come testo JSON. indent=2 lo rende leggibile e ordinato.
json.load(file)
legge il testo JSON dal file e lo ricostruisce come oggetto Python. Dizionario dentro, dizionario fuori: JSON è solo il «formato di viaggio» in mezzo.
varianti «stringa»
json.dumps/json.loads (con la «s») fanno lo stesso ma da/verso una stringa invece che un file — utile per mandare dati sul web.
Guarda il file persona.json prodotto: è quasi identico al dizionario Python! Piccole differenze (le chiavi sempre tra doppie virgolette, true/false/null minuscoli). Ecco perché dizionari e liste sono così importanti: sono la forma dei dati di tutto il web moderno. Quando nel libro di Backend Development riceverai una richiesta o interrogherai un'API, otterrai JSON — e ora sai già cosa farci.
Python ⇄ JSON: la stessa cosa, due forme osserva la corrispondenza

Sezione 12.6Laboratorio: salvare i tuoi dati

Uniamo tutto in una mini-app «lista di cose da fare» che ricorda tra un avvio e l'altro, salvando in JSON. È il pattern «carica → modifica → salva» che sta dietro moltissime applicazioni.

Il programma todo.py carica · aggiungi · salva
todo.py
import json
from pathlib import Path

FILE = Path("todo.json")

# 1. CARICA la lista esistente (o parti da vuota se il file non c'è)
if FILE.exists():
    cose = json.loads(FILE.read_text())
else:
    cose = []

# 2. AGGIUNGI una voce nuova
nuova = input("Cosa devi fare? ")
cose.append(nuova)

# 3. SALVA e mostra
FILE.write_text(json.dumps(cose, indent=2))
print(f"Hai {len(cose)} cose da fare: {cose}")
La magia: ogni volta che lo esegui, ricarica ciò che avevi salvato e ci aggiunge la nuova voce. La lista persiste tra un avvio e l'altro, perché vive nel file todo.json. Questo schema — carica lo stato, modificalo, risalvalo — è alla base di innumerevoli programmi. Estendilo: aggiungi la possibilità di rimuovere una voce (per indice), o di segnare come «fatto». Hai costruito la tua prima app con memoria persistente!

  3. CSV per dati a tabella (csv.DictReader → ogni riga un dizionario) (§12.4). JSON per dati strutturati: json.dump/load, ed è dizionari e liste come testo — la lingua del web (§12.5).
  4. Lo schema carica → modifica → salva (spesso in JSON) dà ai programmi una memoria persistente (§12.6).

Sezione 12.8Soluzioni

Soluzione 12.1

"w" sovrascrive: cancella il contenuto esistente e riparte da zero. "a" aggiunge in fondo, conservando ciò che c'era. Per un log si usa "a": ogni nuova riga si somma alle precedenti.

Soluzione 12.2

Perché with chiude il file automaticamente alla fine del blocco, anche se avviene un errore. Con open/close manuale è facile dimenticare la chiusura (o saltarla per un errore), lasciando dati non salvati o file bloccati. with è più sicuro e più conciso.

Soluzione 12.3

Ottieni un dizionario Python: {"nome": "Ada", "voti": [7, 9]}. Il primo voto si accede così: dati["voti"][0] → 7 (prima la chiave "voti" per la lista, poi l'indice 0).

Soluzione 12.4
saluti.py
nome = input("Nome ospite: ")
with open("ospiti.txt", "a") as f:
    f.write(nome + "\n")

print("Ospiti finora:")
with open("ospiti.txt") as f:
    for riga in f:
        print("-", riga.strip())
