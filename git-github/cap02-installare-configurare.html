Capitolo 2

Installare e configurare Git

Basta teoria: mettiamo Git sul tuo computer e prepariamolo all'uso. È un programma leggero che si installa in pochi minuti e vive nel terminale — la finestra dove si digitano comandi. Dopo l'installazione, c'è un passo di configurazione che tutti fanno una volta sola ma che è importante capire: dire a Git chi sei, cioè il tuo nome ed email. Perché? Perché Git «firma» ogni tua commit con questi dati, così nella storia di un progetto si sa sempre chi ha fatto cosa. In questo capitolo installiamo Git (§2.1), impariamo il comando di configurazione git config (§2.2), impostiamo la nostra identità (§2.3) e prendiamo confidenza con i comandi base del terminale (§2.4). Alla fine avrai un Git funzionante e configurato, pronto per creare il tuo primo repository nel prossimo capitolo.

🗺 Dove siamo

Installare Git (§2.1), git config e i livelli di configurazione (§2.2), impostare nome ed email (§2.3), i comandi base del terminale (§2.4). Nel laboratorio (§2.5) configuri la tua identità.

🔑 Perché ci tocca: la configurazione che ti segue ovunque

La configurazione di nome ed email si fa una volta e vale per tutti i tuoi progetti su quel computer. Farla bene fin da subito ti evita commit «anonime» o firmate con dati sbagliati — un fastidio comune di chi salta questo passo. E capire il terminale, dove Git vive, è la chiave per usarlo con sicurezza invece che a tentoni.

Sezione 2.1Installare Git

Git è gratuito e disponibile per Windows, macOS e Linux. L'installazione è semplice; il modo cambia leggermente a seconda del sistema operativo.

Installare Git, per sistema Windows · macOS · Linux
Windows
scarica l'installer da git-scm.com e installalo (vanno bene le opzioni di default). Ottieni anche «Git Bash», un terminale in stile Unix comodo per seguire il libro.
macOS
spesso Git è già presente. Altrimenti, installalo con brew install git (se hai Homebrew) o scaricandolo da git-scm.com.
Linux
dal gestore pacchetti: sudo apt install git (Debian/Ubuntu) o l'equivalente della tua distribuzione.
verifica l'installazione
apri un terminale e digita git --version. Se vedi un numero di versione, Git è installato e funziona. È il tuo primo comando Git!
terminale — verifica
$ git --version
git version 2.45.0          # il numero preciso non importa: basta che ci sia

Sezione 2.2La prima configurazione

Prima di usare Git, gli si danno alcune impostazioni con il comando git config. La più importante è la tua identità (§2.3); ma capiamo prima come funziona la configurazione, perché ha dei «livelli».

git config e i suoi livelli system · global · local
git config
il comando per leggere e impostare le opzioni di Git. La forma base è git config <livello> <chiave> <valore>.
--global — per te, su tutto il computer
l'impostazione vale per tutti i tuoi progetti sul tuo account. È il livello che userai quasi sempre, ad esempio per nome ed email: le imposti una volta e valgono ovunque.
--local — solo questo progetto
l'impostazione vale solo per il repository corrente, e sovrascrive quella globale. Utile, per esempio, per usare un'email diversa in un progetto specifico.
--system — per tutti gli utenti
vale per l'intero computer, tutti gli utenti. Si usa raramente; richiede privilegi di amministratore.
La regola di precedenza: local batte global, che batte system — dal più specifico al più generale. Per la tua identità userai --global: la imposti una volta e vale per ogni progetto. Puoi rivedere tutte le impostazioni con git config --list.

Sezione 2.3Nome ed email: la tua identità

Ecco il passo che conta: dire a Git chi sei. Git includerà questo nome ed email in ogni commit che farai, come «firma» dell'autore. Sono due comandi, da fare una volta sola.

terminale — imposta la tua identità (una volta sola)
$ git config --global user.name "Anna Rossi"
$ git config --global user.email "anna@esempio.com"

# verifica che siano stati impostati:
$ git config --global user.name
Anna Rossi
Perché nome ed email la firma di ogni commit
user.name
il nome che comparirà come autore di ogni tua commit. Metti il tuo nome vero (o lo pseudonimo con cui vuoi essere riconosciuto): apparirà nella storia del progetto.
user.email
l'email associata alle tue commit. Importante per GitHub (Cap. 10): usa la stessa email del tuo account GitHub, così il servizio collega le commit al tuo profilo.
compaiono in ogni commit
quando salvi uno snapshot (Cap. 4), Git vi allega automaticamente nome, email, data. Ecco perché la configurazione va fatta prima di iniziare: senza, Git si lamenta o firma con dati vuoti.
Nessuna password qui: nome ed email sono solo un'etichetta di autore, non un login. Non sono un segreto (finiscono pubblicamente nella storia dei progetti). Prova il widget qui sotto per vedere come diventano la «firma» di una commit.
La tua identità nelle commit imposta nome/email, guarda la firma

Sezione 2.4Il terminale e i comandi di base

Git si usa dal terminale (detto anche console, riga di comando, shell): una finestra dove scrivi comandi testuali. Se non lo hai mai usato, ecco i pochissimi comandi di «navigazione» che ti serviranno per muoverti tra le cartelle.

I comandi di navigazione essenziali muoversi tra le cartelle
pwd — dove sono?
«print working directory»: mostra la cartella in cui ti trovi ora. Il tuo «tu sei qui». (Su Windows in PowerShell funziona anche cd senza argomenti.)
ls — cosa c'è qui?
elenca i file e le cartelle della posizione corrente. (Su Windows/PowerShell anche dir.)
cd nome-cartella — entra in una cartella
«change directory»: ti sposti dentro una cartella. cd .. torna a quella superiore; cd da solo (o cd ~) torna alla tua home.
mkdir nome — crea una cartella
«make directory»: crea una nuova cartella. La userai per creare la cartella dei tuoi progetti.
terminale — un giro di navigazione
$ pwd
/Users/anna                    # sono nella mia home
$ mkdir progetti            # creo una cartella "progetti"
$ cd progetti               # ci entro
$ pwd
/Users/anna/progetti           # ora sono qui dentro
Non serve diventare esperti di terminale: questi quattro comandi (pwd, ls, cd, mkdir) bastano per tutto il libro. Su Windows, «Git Bash» (installato con Git) offre questi comandi in stile Unix, comodi per seguire gli esempi. Il concetto chiave: Git lavora nella cartella in cui ti trovi, quindi devi sempre sapere «dove sei» (pwd) prima di dare comandi Git.

Sezione 2.5Laboratorio: git config

Configura ora la tua identità: è un passo veloce e fondamentale, che farai una volta e ti servirà per tutto il resto del libro (e della tua vita da programmatore).

terminale — la ricetta completa
# 1) verifica che Git sia installato
$ git --version
git version 2.45.0

# 2) imposta nome ed email (usa i TUOI dati; per GitHub, la stessa email dell'account)
$ git config --global user.name "Il Tuo Nome"
$ git config --global user.email "tua@email.com"

# 3) (utile) imposta "main" come nome del branch iniziale
$ git config --global init.defaultBranch main

# 4) verifica tutto
$ git config --list
user.name=Il Tuo Nome
user.email=tua@email.com
init.defaultBranch=main
✅ Cosa hai fatto

Hai verificato Git, impostato la tua identità (che firmerà ogni commit) e scelto main come nome del branch principale (una convenzione moderna diffusa; vedrai i branch al Cap. 7). Con git config --list controlli sempre le tue impostazioni. Se un domani vorrai cambiare qualcosa, basta rieseguire il comando con il nuovo valore. Fatto questo, Git è pronto: nel prossimo capitolo creiamo il nostro primo repository e iniziamo a versionare davvero.

🔑 Quello che devi portarti dietro dal Capitolo 2
  1. Git si installa da git-scm.com (o dal gestore pacchetti); verifica con git --version (§2.1).
  2. git config imposta le opzioni; i livelli sono local (progetto) > global (tuo account) > system (tutti). Per l'identità usi --global (§2.2).
  3. Imposta nome ed email (user.name, user.email): firmano ogni commit; per GitHub, usa l'email dell'account (§2.3).
  4. Git vive nel terminale: naviga con pwd, ls, cd, mkdir. Sappi sempre «dove sei» (§2.4).

Git è installato e sa chi sei. È il momento di creare qualcosa di reale: nel Capitolo 3 trasformiamo una cartella qualsiasi in un repository Git con git init, e scopriamo il concetto più importante di tutto Git — le tre aree (working directory, staging, repository) — che spiega come una modifica diventa uno snapshot. È la base su cui poggia ogni comando successivo.

Sezione 2.7Soluzioni

Soluzione 2.1
terminale
$ git --version

Se stampa un numero di versione (es. git version 2.45.0), Git è installato e pronto.

Soluzione 2.2
terminale
$ git config --global user.name "Mario Bianchi"
$ git config --global user.email "mario@posta.it"

Le virgolette attorno al nome servono perché contiene uno spazio.

Soluzione 2.3

--global imposta l'opzione per tutti i progetti del tuo account sul computer; --local la imposta solo per il repository corrente. Se una chiave è definita in entrambi, vince quella local (la più specifica): la regola di precedenza è local > global > system. Così puoi avere un'impostazione generale e sovrascriverla in un singolo progetto quando serve.

Soluzione 2.4

user.name e user.email sono l'identità che Git allega come autore a ogni commit: servono a sapere, nella storia del progetto, chi ha fatto ogni modifica. Per GitHub conviene che l'email coincida con quella dell'account perché GitHub usa l'email delle commit per collegarle al tuo profilo: con l'email giusta, i tuoi contributi appaiono correttamente attribuiti a te (e nel tuo grafico dei contributi); con un'email diversa, GitHub non riesce ad associare le commit al tuo account.
