Programmazione · Versionare e collaborare

Git & GitHub

Ogni programmatore, prima o poi, incontra Git: lo strumento che tiene traccia di ogni modifica al codice, permette di tornare indietro senza paura, di lavorare su idee in parallelo e — con GitHub — di collaborare con altri in tutto il mondo. È una competenza indispensabile, spesso data per scontata e mai spiegata davvero. Questo libro la insegna da zero: ogni comando smontato pezzo per pezzo, widget interattivi per vedere commit e branch, laboratori guidati da fare nel terminale con l'output atteso. Dalla prima commit al flusso di lavoro con le pull request.

🧭 Come leggere questo libro

I capitoli sono in ordine di dipendenza: prima Git da solo (commit, storia, branch, merge), poi GitHub e la collaborazione (remoto, push/pull, pull request). Ogni capitolo chiude con un Laboratorio: comandi veri da provare, con l'output atteso accanto. Serve solo un terminale e Git installato (te lo spieghiamo al Capitolo 2). Nessun linguaggio di programmazione richiesto: Git funziona con qualsiasi file.

I riquadri colorati hanno un significato fisso: blu = definizione o concetto, verde = intuizione, rosso = trappola in cui cadono tutti, viola = da ricordare, oro = nota a margine.

Indice

I capitoli in grigio sono ancora da scrivere: il libro cresce un capitolo alla volta.
